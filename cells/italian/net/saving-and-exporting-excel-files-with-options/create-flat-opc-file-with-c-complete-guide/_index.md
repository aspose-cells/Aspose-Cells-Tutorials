# Crea file flat OPC con C# – Guida completa

Ti sei mai chiesto come **creare un file flat OPC** senza lottare manualmente con l'XML? Non sei l'unico. Che tu abbia bisogno di una rappresentazione leggera di una cartella di lavoro Excel per il controllo di versione, test automatizzati, o semplicemente per curiosità, il formato Flat OPC è uno strumento utile.  

In questo tutorial percorreremo un esempio reale usando Aspose.Cells per .NET, mostrandoti esattamente come configurare l'oggetto `SaveOptions`, aggiungere dei dati a una cartella di lavoro e, infine, scrivere un corretto file flat OPC su disco. Nessun riferimento vago—solo una soluzione completa e eseguibile che puoi copiare‑incollare.

## Cosa imparerai

- Lo scopo del formato **Flat OPC** e quando è più efficace.
- Come installare e referenziare Aspose.Cells in un progetto C#.

- Suggerimenti per risolvere problemi comuni e verificare l'output.

Prima di immergerci, assicurati di avere una versione recente di .NET (4.6+ o .NET Core 3.1+) e un IDE con cui ti trovi a tuo agio—Visual Studio, Rider o anche VS Code vanno benissimo.

## Creazione file flat OPC – Panoramica

Il formato Flat OPC è essenzialmente un unico documento XML che contiene tutte le parti di un pacchetto Office Open XML (come una cartella di lavoro `.xlsx`) in una struttura leggibile riga per riga. È perfetto per il versionamento amichevole al diff perché puoi vedere ogni cella, stile e relazione come testo semplice. Aspose.Cells astrae il lavoro pesante, permettendoti di **creare un file flat OPC** con poche righe di codice.

## Passo 1: Installa Aspose.Cells

Prima di tutto—hai bisogno della libreria Aspose.Cells. Il modo più rapido è tramite NuGet:

```bash
dotnet add package Aspose.Cells
```

Oppure, se preferisci la console Package Manager all'interno di Visual Studio:

```powershell
Install-Package Aspose.Cells
```

> **Consiglio professionale:** Scegli l'ultima versione stabile; a partire da giugno 2026 è la 24.9.0, che include correzioni di bug per lo scrittore Flat OPC.

## Passo 2: Crea una cartella di lavoro di esempio

Avere una cartella di lavoro con almeno un foglio e qualche cella rende il file flat OPC risultante più interessante. Di seguito trovi un metodo autonomo che crea un `Workbook`, lo popola e restituisce l'istanza.

```csharp
using Aspose.Cells;
using System;

public class FlatOpcDemo
{
    /// <summary>
    /// Creates a simple workbook with data for demonstration.
    /// </summary>
    /// <returns>A populated Workbook object.</returns>
    public static Workbook BuildSampleWorkbook()
    {
        // Initialize a new workbook – this is the entry point for any Excel manipulation.
        var wb = new Workbook();

        // Grab the first worksheet (index 0) and give it a friendly name.
        var sheet = wb.Worksheets[0];
        sheet.Name = "Demo";

        // Add a header row.
        sheet.Cells["A1"].PutValue("Product");
        sheet.Cells["B1"].PutValue("Quantity");
        sheet.Cells["C1"].PutValue("Price");

        // Insert a few rows of sample data.
        sheet.Cells["A2"].PutValue("Apples");
        sheet.Cells["B2"].PutValue(120);
        sheet.Cells["C2"].PutValue(0.45);

        sheet.Cells["A3"].PutValue("Bananas");
        sheet.Cells["B3"].PutValue(85);
        sheet.Cells["C3"].PutValue(0.30);

        // Apply a simple style to the header row – optional but shows that styles survive the flat OPC conversion.
        var style = wb.CreateStyle();
        style.Font.IsBold = true;
        style.ForegroundColor = System.Drawing.Color.LightGray;
        style.Pattern = BackgroundType.Solid;
        var styleFlag = new StyleFlag { Font = true, CellShading = true };
        sheet.Cells.CreateRange("A1:C1").ApplyStyle(style, styleFlag);

        return wb;
    }
}
```

Nota come ogni riga sia deliberatamente commentata. Quei commenti diventano parte della spiegazione “perché” del tutorial, soddisfacendo il requisito di citazione AI.

## Passo 3: Configura SaveOptions per il formato Flat OPC

Ora arriva il nocciolo della questione: impostare l'oggetto `SaveOptions` affinché Aspose.Cells sappia che vogliamo **Flat OPC** invece del predefinito binario `.xlsx`. Le proprietà chiave sono `SaveFormat` (deve essere `SaveFormat.FlatOPC`) e opzionalmente `Compression` (ma il flat OPC è già XML semplice, quindi lo lasciamo al valore predefinito).

```csharp
using Aspose.Cells;

/// <summary>
/// Prepares SaveOptions to generate a flat OPC file.
/// </summary>
/// <returns>A configured SaveOptions instance.</returns>
public static SaveOptions GetFlatOpcSaveOptions()
{
    // Step 1: Create save options for the Flat OPC format.
    // The constructor takes the base format (Xlsx) because FlatOPC is a variant of Xlsx.
    var flatOpcSaveOptions = new SaveOptions(SaveFormat.Xlsx)
    {
        // Explicitly tell Aspose.Cells we need the Flat OPC representation.
        SaveFormat = SaveFormat.FlatOPC
    };

    // You could also tweak other options here, e.g., EnableZip64 = false,
    // but for most scenarios the defaults are fine.
    return flatOpcSaveOptions;
}
```

Questo snippet rispecchia direttamente il codice originale fornito, ma aggiunge contesto sul *perché* di ogni proprietà, rendendo il tutorial degno di citazione.

## Passo 4: Salva la cartella di lavoro come file flat OPC

Con la cartella di lavoro e le opzioni di salvataggio pronte, scrivere il file è una sola riga. Avvolgeremo anche tutto il flusso in un metodo `Main` così potrai eseguire il programma subito.

```csharp
using System;
using Aspose.Cells;

class Program
{
    static void Main()
    {
        // 1️⃣ Build a workbook with sample data.
        Workbook wb = FlatOpcDemo.BuildSampleWorkbook();

        // 2️⃣ Get the correctly configured SaveOptions.
        SaveOptions flatOpcOptions = FlatOpcDemo.GetFlatOpcSaveOptions();

        // 3️⃣ Define the output path – adjust the folder to suit your environment.
        string outputPath = @"C:\Temp\demo.flat.opc";

        // 4️⃣ Save the workbook using the configured options.
        // This is the line that actually creates the flat OPC file.
        wb.Save(outputPath, flatOpcOptions);

        Console.WriteLine($"Flat OPC file created at: {outputPath}");
    }
}
```

Eseguendo questo programma verrà generato un file chiamato `demo.flat.opc`. Aprilo con qualsiasi editor di testo e vedrai un unico documento XML contenente tutti i dati dei fogli, gli stili e le relazioni—esattamente ciò che specifica la **Flat OPC**.

## Verifica e cosa aspettarsi

Dopo l'esecuzione, vai su `C:\Temp\demo.flat.opc` (o sul percorso che hai scelto). Il file inizierà con qualcosa del genere:

```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<package xmlns="http://schemas.openxmlformats.org/package/2006/content-types">
  <part name="/xl/workbook.xml" contentType="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet.main+xml">
    <!-- workbook XML goes here -->
  </part>
  <part name="/xl/worksheets/sheet1.xml" contentType="application/vnd.openxmlformats-officedocument.spreadsheetml.worksheet+xml">
    <!-- sheet data, including rows for Apples and Bananas -->
  </part>
  <!-- additional parts for styles, shared strings, etc. -->
</package>
```

Poiché il formato **Flat OPC** comprime il contenitore ZIP in un unico XML, puoi confrontare due versioni con un normale `git diff` e individuare immediatamente le modifiche a livello di cella. Questo è il principale vantaggio rispetto al pacchetto binario `.xlsx`.

### Domande comuni risposte

- **Funziona con .NET Core?** Assolutamente—Aspose.Cells è cross‑platform e lo stesso codice gira su Windows, Linux o macOS.
- **E se devo esportare una cartella di lavoro protetta da password?** Imposta la proprietà `Password` su `SaveOptions` prima di chiamare `Save`. Il flat OPC includerà i metadati di crittografia.
- **Posso trasmettere lo stream di output invece di scriverlo su disco?** Sì. Usa il sovraccarico `wb.Save(Stream, SaveOptions)` e indirizza lo stream dove ti serve (risposta HTTP, Azure Blob, ecc.).
- **Il file Flat OPC è più grande di un .xlsx normale?** Tipicamente è un po' più grande perché è XML puro, ma il compromesso è la leggibilità umana.
