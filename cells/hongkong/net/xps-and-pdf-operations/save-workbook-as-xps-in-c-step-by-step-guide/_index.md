# 在 C# 中將工作簿另存為 XPS – 完整程式指南

有沒有試過 **將工作簿另存為 XPS**，結果因為文件說明不清而卡住？你並不是唯一遇到這個問題的人。無論你是需要將財務報表匯出成可列印的 XPS 版本，或只是想實驗向量格式，將 Excel 工作簿轉成 XPS 文件其實相當簡單——只要掌握正確的 API 呼叫。

在本指南中，我們會一步步說明整個流程，從建立全新工作簿到處理 Unicode 變體選擇符（例如「A️」的範例）。同時，我們也會觸及一個常見問題：**如何使用流行的 .NET 函式庫將 Excel 匯出為 XPS**。完成後，你將擁有可執行的程式碼片段、每一步的說明，以及幾個避免邊緣案例的專業技巧。

## 你將學到什麼

- 從頭建立 `Aspose.Cells` 工作簿。  
- 插入包含變體選擇符（隱藏的「emoji 風格」字元）的文字。  
- 設定 XPS 儲存選項（預設值通常已足夠）。  
- 將工作簿持久化為 XPS 檔案並驗證結果。  
- 可選：如果使用其他函式庫或需要自訂頁面設定，提供替代的 **export Excel to XPS** 方法。

### 前置條件

- .NET 6.0 或更新版本（此程式碼亦可在 .NET Framework 4.6+ 上執行）。  
- 有效的 **Aspose.Cells for .NET** 授權（可先使用免費試用版）。  
- 你熟悉的開發環境——Visual Studio、Rider，或甚至 VS Code 都可以。  

如果上述條件都已備妥，我們就開始吧。

## 步驟 1：建立新工作簿（初始化文件）

首先，我們需要一個乾淨的工作簿物件，作為 XPS 畫布的基礎。

```csharp
// Step 1: Instantiate a fresh workbook
Workbook workbook = new Workbook();
```

`Workbook` 類別是 Aspose.Cells 所有功能的入口。把它想像成一本空白筆記本，之後會在裡面加入工作表、儲存格與樣式。這裡沒有隱藏的魔法——只是普通的 C# 物件，準備好存放資料。

## 步驟 2：存取第一個工作表

全新的工作簿會自動帶有一個預設工作表。取得它，以便開始填寫儲存格。

```csharp
// Step 2: Pull the first (and only) worksheet out of the workbook
Worksheet worksheet = workbook.Worksheets[0];
```

為什麼是索引 `[0]`？因為 Aspose.Cells 以零基索引的集合儲存工作表。如果之後新增工作表，只要調整索引或遍歷集合即可。

## 步驟 3：插入含變體選擇符的文字

這裡是 **export Excel to XPS** 範例稍微有趣的地方。我們會放入一個字元，後面接上變體選擇符（`\uFE0F`）。這段不可見的程式碼會告訴 Unicode 渲染器在可能的情況下將前面的字元當作 emoji 風格的字形呈現。

```csharp
// Step 3: Write a string that includes a variation selector (e.g., "A️")
worksheet.Cells[0, 0].PutValue("A\uFE0F");
```

- `Cells[0, 0]` 指向儲存格 **A1**（第 0 列，第 0 欄）。  
- `PutValue` 會自動推斷資料類型，因此我們可以直接傳入字串。  
- `\uFE0F` 是 Unicode *variation selector‑16*；大多數現代檢視器會將「A️」呈現為樣式化的「A」。

**專業提示：** 若之後發現 XPS 輸出只顯示普通的「A」而非樣式化版本，請確認你的 XPS 檢視器支援 Unicode 變體選擇符。舊版檢視器可能不支援。

## 步驟 4：準備 XPS 儲存選項（通常使用預設值）

Aspose.Cells 內建 `XpsSaveOptions` 類別，可讓你微調頁面大小、邊距等設定。對於簡單的轉換，預設值已相當足夠，但我們仍會實例化此物件，以示範使用模式。

```csharp
// Step 4: Create XPS save options – default settings are fine for most cases
XpsSaveOptions xpsOptions = new XpsSaveOptions();
```

如果需要自訂頁面方向或嵌入字型，只要在儲存之前對 `xpsOptions` 設定相應屬性。例如：

```csharp
xpsOptions.PageSetup.Orientation = PageOrientation.Landscape;
xpsOptions.EmbedStandardFonts = true;
```

上述程式碼屬於可選項目，為了保持範例簡潔，在核心示例中已省略。

## 步驟 5：將工作簿另存為 XPS 文件

現在到了關鍵時刻——將工作簿持久化為 XPS 檔案。選擇一個你有寫入權限的資料夾；範例使用的路徑僅為佔位符，請自行替換。

```csharp
// Step 5: Persist the workbook as an XPS file
string outputPath = @"C:\Temp\variation.xps";
workbook.Save(outputPath, xpsOptions);
```

執行此行程式後，你會在 `C:\Temp` 中看到 `variation.xps`。使用任何 XPS 檢視器（例如 Windows XPS Viewer）開啟，應該會看到「A️」字元依系統字型設定正確呈現。

### 預期結果

- **檔案類型：** XPS（XML Paper Specification）——向量式、頁面導向的格式。  
- **內容：** 單一頁面，左上角儲存格顯示文字「A️」。  
- **驗證方式：** 開啟檔案；若檢視器支援變體選擇符，字元應以樣式化的「A」顯示。

## 替代做法：使用 OpenXML 與 System.Drawing 匯出 Excel 為 XPS

如果你不想依賴 Aspose.Cells，也可以透過 Open XML SDK 搭配 `System.Drawing.Printing` 命名空間來 **export Excel to XPS**。此工作流程較為手動：

1. 使用 OpenXML 讀取 `.xlsx`，取得儲存格值。  
2. 使用 `Graphics`（或第三方渲染器）將每個工作表渲染成位圖。  
3. 透過 `XpsDocumentWriter` 建立 XPS 文件，並將位圖繪製到每一頁。

以下是一個骨架程式碼，說明概念——*這不是即插即用的替代方案*，但若無法取得 Aspose 授權，仍能提供實作方向。

```csharp
using DocumentFormat.OpenXml.Packaging;
using System.Drawing;
using System.Printing;
using System.Windows.Xps;
using System.Windows.Xps.Packaging;

// Load the Excel file
using (SpreadsheetDocument doc = SpreadsheetDocument.Open(@"C:\Temp\source.xlsx", false))
{
    // Extract data (omitted for brevity)
}

// Render to bitmap (pseudo‑code)
Bitmap bitmap = RenderWorksheetToBitmap(); // You need a renderer here

// Write XPS
using (XpsDocument xpsDoc = new XpsDocument(@"C:\Temp\output.xps", FileAccess.Write))
{
    XpsDocumentWriter writer = XpsDocument.CreateXpsDocumentWriter(xpsDoc);
    Visual visual = new DrawingVisual();
    using (DrawingContext dc = ((DrawingVisual)visual).RenderOpen())
    {
        dc.DrawImage(bitmap, new Rect(0, 0, bitmap.Width, bitmap.Height));
    }
    writer.Write(visual);
}
```

**為什麼仍建議使用 Aspose.Cells？**  
- 一行儲存呼叫 (`workbook.Save`) 對比數十行的渲染程式碼。  
- 完整保留公式、圖表與 Unicode 字元的忠實度。  
- 內建支援頁面設定、邊距與字型嵌入。

如果你只需要快速匯出且已擁有 Aspose，建議仍使用上方的 **save workbook as XPS** 方法。

## 常見陷阱與避免方式

| 症狀 | 可能原因 | 解決方法 |
|---------|--------------|-----|
| XPS 檔案為空或只有空白頁 | 儲存前未寫入任何儲存格 | 確認在 `Save` 之前已呼叫 `PutValue`（或其他寫入方法）。 |
| 「A️」顯示為普通「A」 | 檢視器不支援變體選擇符 | 使用 Windows 10 以上的 XPS Viewer 或現代的 PDF‑to‑XPS 轉換工具測試。 |
| 儲存時拋出 `UnauthorizedAccessException` | 輸出資料夾唯讀或路徑錯誤 | 確認資料夾存在且程式有寫入權限。 |
| XPS 中字型顯示不一致 | 字型未嵌入 | 在儲存前設定 `xpsOptions.EmbedStandardFonts = true;`。 |

## 完整可執行範例（直接複製貼上）

```csharp
using Aspose.Cells;
using System;

class Program
{
    static void Main()
    {
        // 1️⃣ Create a new workbook
        Workbook workbook = new Workbook();

        // 2️⃣ Grab the first worksheet
        Worksheet worksheet = workbook.Worksheets[0];

        // 3️⃣ Insert text with a variation selector (e.g., "A️")
        worksheet.Cells[0, 0].PutValue("A\uFE0F");

        // 4️⃣ Prepare default XPS save options
        XpsSaveOptions xpsOptions = new XpsSaveOptions();

        // 5️⃣ Define output path and save as XPS
        string outputPath = @"C:\Temp\variation.xps";
        workbook.Save(outputPath, xpsOptions);

        Console.WriteLine($"Workbook successfully saved as XPS at: {outputPath}");
    }
}
```

執行程式後，開啟 `C:\Temp\variation.xps`，即可看到字元正確呈現。主控台訊息會確認操作成功。
