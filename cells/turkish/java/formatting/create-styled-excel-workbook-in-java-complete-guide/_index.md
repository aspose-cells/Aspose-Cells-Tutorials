# Java’da Stilize Excel Çalışma Kitabı Oluşturma – Tam Kılavuz

Java’dan **create styled excel workbook** oluşturmanız gerektiğinde, biçimlendirme detaylarıyla boğulduğunuzu hissettiniz mi? Yalnız değilsiniz. Verileri bir veritabanından çekiyor ya da bir raporu dışa aktarıyor olun, doğru görünümü elde etmek—para birimi sembolleri, tarih maskeleri, kalın başlıklar—bir görev gibi gelebilir.  

Bu öğreticide **adds custom formatting to excel** ekleyen, **set date format excel column** ayarlamanıza izin veren ve sonunda **save workbook as xlsx** yaparak tereddüt etmeden çalışmanızı tamamlayan pratik bir çözümü adım adım inceleyeceğiz. Harekete geçmeye hazır mısınız?

## Öğrenecekleriniz

- Sıfırdan bir Aspose.Cells `Workbook` nasıl oluşturulur.  
- Genel, Para Birimi ve Tarih sütunlarını kapsayan bir stil dizisi nasıl uygulanır.  
- Sütun adlarıyla birlikte **create styled excel workbook** oluşturan tam çağrı.  
- **save workbook as xlsx** yöntemiyle dosyanın nasıl kalıcı hâle getirileceği.  

Harici yapılandırma dosyaları, karmaşık XML hileleri yok—kopyalayıp yapıştırabileceğiniz, çalıştırıp uyarlayabileceğiniz sade Java kodu.

### Ön Koşullar

- Java 8+ (kod, herhangi bir yeni JDK’da çalışır).  
- Aspose.Cells for Java kütüphanesi (en son JAR’ı Maven Central’dan alabilirsiniz).  
- Koleksiyonlara (`List`, `Map`) biraz aşina olmak.  

Eğer bunlardan biri size yabancı geliyorsa, bir an durup Aspose.Cells JAR’ını kurun; geri kalan doğal olarak akacaktır.

## Stilize Excel Çalışma Kitabı – Projeyi Kurma

İlk iş olarak: yeni bir Maven (veya Gradle) projesi oluşturun ve Aspose.Cells’i bağımlılık olarak ekleyin.

```xml
<!-- pom.xml snippet -->
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-cells</artifactId>
    <version>23.12</version> <!-- use the latest version -->
</dependency>
```

> **Pro ipucu:** Gradle kullanıyorsanız, aynı koordinatlar `implementation 'com.aspose:aspose-cells:23.12'` ile çalışır.

Neden Maven kurulumu ile başlıyoruz? Tüm geçişli bağımlılıkları otomatik olarak çeker, böylece daha sonra doğru JAR’ları aramak için zaman kaybetmezsiniz.

## Excel Sütunlarına Özel Biçimlendirme Ekleme

Proje hazır olduğuna göre, istediğimiz stilleri tanımlayalım. Aspose.Cells’te her sütuna doğrudan eşlenen bir format dizesi dizisi geçirebilirsiniz.

```java
// Define column styles: General, Currency, Date
String[] columnStyles = {
    "General",          // Column A – plain text or numbers
    "$#,##0.00",        // Column B – currency with two decimals
    "mm/dd/yyyy"        // Column C – US‑style date
};
```

**add custom formatting to excel** adımı, bu dize dizisini oluşturmak kadar basittir.  
Her giriş, Excel’in “Custom” biçimlendirme iletişim kutusunda kullandığınız aynı desenle eşleşir, yani yeni bir sözdizimi öğrenmenize gerek kalmaz.

---

## Excel Sütununda Tarih Biçimini Ayarlama

Verilerinizde tarih bulunuyorsa, bunların tutarlı görünmesini istersiniz. `columnStyles` dizisinin üçüncü girdisi (`"mm/dd/yyyy"`), tam da bunu yapar—Aspose.Cells’e üçüncü sütunu tarih olarak ele almasını ve ay/gün/yıl sırasında göstermesini söyler.

```java
// Example: Adding a row with a Date object
Map<String, Object> row = new HashMap<>();
row.put("Product", "Widget");
row.put("Price", 19.99);
row.put("ReleaseDate", new java.util.Date()); // Today’s date
dataRows.add(row);
```

Çalışma kitabı oluşturulduğunda, **set date format excel column** talimatı her `ReleaseDate` hücresinin `04/27/2026` (veya bugünün tarihi) gibi görünmesini sağlar. Sonradan bir işleme gerek kalmaz.

## Verileri İçe Aktarma ve Stilleri Uygulama

Stiller tanımlandı, bir sonraki adım verileri çalışma sayfasına aktarmak. Aspose.Cells, `importCustomObjects` metodunu sunar; bu metod veriyi eklemekle kalmaz, aynı zamanda az önce oluşturduğumuz stil dizisini de uygular.

```java
// Step 1: Simulate fetching rows from a DB
List<Map<String, Object>> dataRows = fetchRows(); // see helper below

// Step 2: Create a fresh workbook and grab the first sheet
Workbook workbook = new Workbook();
Worksheet worksheet = workbook.getWorksheets().get(0);

// Step 3: Import data with custom styles; true = first row has headers
worksheet.getCells().importCustomObjects(
        dataRows,        // data source
        0,               // start row (0‑based)
        0,               // start column (0‑based)
        columnStyles,    // style array from above
        true);           // first row contains column names
```

### Neden `importCustomObjects`?

- **Tek‑çağrı büyüsü:** Veriyi *ve* biçimlendirmeyi tek seferde yazar.  
- **Başlık farkındalığı:** `true` parametresi, API’ye haritanın ilk anahtarlarını sütun başlıkları olarak ele almasını söyler; bu, **create styled excel workbook** senaryolarında okunabilir sütun başlıkları istediğinizde mükemmeldir.  
- **Performans:** Aspose, yazma işlemlerini toplu hâle getirir, bu da hücre‑hücre döngüden daha hızlıdır.

> **Dikkat:** Her `Map` içindeki anahtarların sırası önemlidir. Belirli bir sütun sırası istiyorsanız `LinkedHashMap` kullanın.

## Çalışma Kitabını XLSX Olarak Kaydetme

Tüm bu çalışmayı dosyaya kalıcı hâle getiremezseniz anlamsız olur. Son satır, **save workbook as xlsx** yapan tek satırlık bir komuttur.

```java
// Step 5: Persist the workbook
String outputPath = "output/DataTableStyled.xlsx";
workbook.save(outputPath);
System.out.println("Workbook saved to " + outputPath);
```

`save` metodu dosya uzantısını otomatik algılar. `.xlsx` belirterek, modern Office Open XML formatını garanti etmiş olursunuz; bu format çoğu sonraki araç tarafından beklenir.

## Çalıştır ve Sonucu Doğrula

`ImportDataTableWithStyle` sınıfını derleyip çalıştırın. Çalıştırdıktan sonra `output/DataTableStyled.xlsx` konumunda bir dosya görmelisiniz. Excel (veya LibreOffice) ile açtığınızda şunları fark edeceksiniz:

- **A** sütunu düz metin gösterir (`Product`).  
- **B** sütunu dolar işareti ve iki ondalık basamakla sayıları gösterir (`$19.99`).  
- **C** sütunu `mm/dd/yyyy` biçiminde tarih gösterir (`04/27/2026`).

## Tam, Çalıştırılabilir Örnek

Aşağıda IDE’nize doğrudan kopyalayabileceğiniz tam sınıf yer alıyor. Gerçek bir veritabanına ihtiyaç duymadan test edebilmeniz için örnek veri döndüren bir `fetchRows` metodu da içeriyor.

```java
import com.aspose.cells.*;
import java.util.*;

public class ImportDataTableWithStyle {
    public static void main(String[] args) throws Exception {
        // ---------- Step 1: Get data ----------
        List<Map<String, Object>> dataRows = fetchRows();

        // ---------- Step 2: Create workbook ----------
        Workbook workbook = new Workbook();
        Worksheet worksheet = workbook.getWorksheets().get(0);

        // ---------- Step 3: Define column styles ----------
        String[] columnStyles = { "General", "$#,##0.00", "mm/dd/yyyy" };

        // ---------- Step 4: Import data with styles ----------
        // true = first row contains column names
        worksheet.getCells().importCustomObjects(dataRows, 0, 0, columnStyles, true);

        // ---------- Step 5: Save as XLSX ----------
        String outPath = "output/DataTableStyled.xlsx";
        workbook.save(outPath);
        System.out.println("Workbook saved to " + outPath);
    }

    // Mock method – replace with real DB call if needed
    private static List<Map<String, Object>> fetchRows() {
        List<Map<String, Object>> rows = new ArrayList<>();

        Map<String, Object> row1 = new LinkedHashMap<>();
        row1.put("Product", "Gadget");
        row1.put("Price", 49.95);
        row1.put("ReleaseDate", new Date()); // today
        rows.add(row1);

        Map<String, Object> row2 = new LinkedHashMap<>();
        row2.put("Product", "Widget");
        row2.put("Price", 19.99);
        Calendar cal = Calendar.getInstance();
        cal.set(2025, Calendar.DECEMBER, 15);
        row2.put("ReleaseDate", cal.getTime());
        rows.add(row2);

        return rows;
    }
}
```

Çalıştırın, oluşturulan dosyayı açın ve **create styled excel workbook** sonucunu beklediğiniz gibi görün.

## Yaygın Sorular & Kenar Durumları

**Üçten fazla sütun ihtiyacım olursa ne yapmalıyım?**  
`columnStyles` dizisini genişletmeniz yeterli—her giriş, sırayla bir sütuna karşılık gelir. Dokunmak istemediğiniz sütunlar için `"General"` kullanın.

**Yazı tipleri veya renkler ekleyebilir miyim?**  
Evet. Dize formatı yerine bir `Style` nesnesi oluşturup, bunu bir `StyleFlag` ile ilişkilendirerek `Style[]` dizisi olarak `importCustomObjects`’a geçirebilirsiniz. Bu biraz daha derin bir konu, ama prensip aynı kalır.

**Büyük veri setleriyle nasıl başa çıkılır?**  
Aspose.Cells milyonlarca satırı verimli bir şekilde işler, ancak JVM heap’ini artırmak isteyebilirsiniz (`-Xmx2G` gibi).
