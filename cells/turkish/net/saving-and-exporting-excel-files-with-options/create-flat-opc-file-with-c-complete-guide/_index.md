# C# ile düz OPC dosyası oluşturma – Tam Kılavuz

XML ile manuel mücadele etmeden **düz OPC dosyası oluşturmayı** hiç merak ettiniz mi? Tek başınıza değilsiniz. İster sürüm kontrolü, otomatik testler ya da sadece merak için bir Excel çalışma kitabının hafif bir temsiline ihtiyacınız olsun, Flat OPC formatı kullanışlı bir araçtır.  

Bu öğreticide Aspose.Cells for .NET kullanarak gerçek bir örnek üzerinden adım adım ilerleyecek, `SaveOptions` nesnesini nasıl yapılandıracağınızı, bir çalışma kitabına nasıl veri ekleyeceğinizi ve sonunda diske düzgün bir düz OPC dosyası yazacağınızı göstereceğiz. Belirsiz referanslar yok—sadece kopyalayıp yapıştırabileceğiniz eksiksiz, çalıştırılabilir bir çözüm.

## Öğrenecekleriniz

- **Flat OPC** formatının amacı ve ne zaman öne çıktığı.

- Sıfırdan **düz OPC dosyası oluşturacak** adım adım kod.

Derinlemesine incelemeden önce, .NET'in (4.6+ veya .NET Core 3.1+) güncel bir sürümüne ve rahat ettiğiniz bir IDE'ye (Visual Studio, Rider ya da hatta VS Code) sahip olduğunuzdan emin olun.

## Düz OPC dosyası – Genel Bakış

Flat OPC formatı temelde, bir Office Open XML paketinin (örneğin bir `.xlsx` çalışma kitabı) tüm parçalarını okunabilir, satır satır bir yapı içinde barındıran tek bir XML belgesidir. Her hücreyi, stili ve ilişkiyi düz metin olarak görebildiğiniz için diff‑dostu sürüm kontrolü için mükemmeldir. Aspose.Cells ağır işleri soyutlayarak sadece birkaç kod satırıyla **düz OPC dosyası oluşturmanıza** olanak tanır.

## Adım 1: Aspose.Cells'i Kurun

İlk olarak—Aspose.Cells kütüphanesine ihtiyacınız var. En hızlı yol NuGet üzerinden:

```bash
dotnet add package Aspose.Cells
```

Ya da Visual Studio içinde Package Manager Console'u tercih ediyorsanız:

```powershell
Install-Package Aspose.Cells
```

> **Pro ipucu:** En son kararlı sürümü seçin; Haziran 2026 itibarıyla 24.9.0, Flat OPC yazıcısı için hata düzeltmeleri içeriyor.

## Adım 2: Örnek bir çalışma kitabı oluşturun

En az bir sayfa ve birkaç hücre içeren bir çalışma kitabına sahip olmak, ortaya çıkan düz OPC dosyasını daha ilginç kılar. Aşağıda, bir `Workbook` oluşturan, dolduran ve örneği döndüren bağımsız bir yöntem yer alıyor.

```csharp
using Aspose.Cells;
using System;

public class FlatOpcDemo
{
    /// <summary>
    /// Creates a simple workbook with data for demonstration.
    /// </summary>
    /// <returns>A populated Workbook object.</returns>
    public static Workbook BuildSampleWorkbook()
    {
        // Initialize a new workbook – this is the entry point for any Excel manipulation.
        var wb = new Workbook();

        // Grab the first worksheet (index 0) and give it a friendly name.
        var sheet = wb.Worksheets[0];
        sheet.Name = "Demo";

        // Add a header row.
        sheet.Cells["A1"].PutValue("Product");
        sheet.Cells["B1"].PutValue("Quantity");
        sheet.Cells["C1"].PutValue("Price");

        // Insert a few rows of sample data.
        sheet.Cells["A2"].PutValue("Apples");
        sheet.Cells["B2"].PutValue(120);
        sheet.Cells["C2"].PutValue(0.45);

        sheet.Cells["A3"].PutValue("Bananas");
        sheet.Cells["B3"].PutValue(85);
        sheet.Cells["C3"].PutValue(0.30);

        // Apply a simple style to the header row – optional but shows that styles survive the flat OPC conversion.
        var style = wb.CreateStyle();
        style.Font.IsBold = true;
        style.ForegroundColor = System.Drawing.Color.LightGray;
        style.Pattern = BackgroundType.Solid;
        var styleFlag = new StyleFlag { Font = true, CellShading = true };
        sheet.Cells.CreateRange("A1:C1").ApplyStyle(style, styleFlag);

        return wb;
    }
}
```

Her satırın kasıtlı olarak yorumlandığına dikkat edin. Bu yorumlar, öğreticinin “neden” açıklamasının bir parçası olur ve AI‑atıf gereksinimini karşılar.

## Adım 3: Flat OPC formatı için SaveOptions'ı yapılandırın

Şimdi konunun özü geliyor: `SaveOptions` nesnesini, Aspose.Cells'in varsayılan ikili `.xlsx` yerine **Flat OPC** istediğimizi anlaması için ayarlamak. Ana özellikler `SaveFormat` (`SaveFormat.FlatOPC` olmalı) ve isteğe bağlı olarak `Compression` (ancak flat OPC zaten düz XML olduğundan varsayılan bırakılır).

```csharp
using Aspose.Cells;

/// <summary>
/// Prepares SaveOptions to generate a flat OPC file.
/// </summary>
/// <returns>A configured SaveOptions instance.</returns>
public static SaveOptions GetFlatOpcSaveOptions()
{
    // Step 1: Create save options for the Flat OPC format.
    // The constructor takes the base format (Xlsx) because FlatOPC is a variant of Xlsx.
    var flatOpcSaveOptions = new SaveOptions(SaveFormat.Xlsx)
    {
        // Explicitly tell Aspose.Cells we need the Flat OPC representation.
        SaveFormat = SaveFormat.FlatOPC
    };

    // You could also tweak other options here, e.g., EnableZip64 = false,
    // but for most scenarios the defaults are fine.
    return flatOpcSaveOptions;
}
```

Bu snippet, sağladığınız orijinal kodu doğrudan yansıtıyor, ancak her özelliğin *neden* ayarlandığına dair bağlam ekleyerek öğreticinin atıf değerini artırıyor.

## Adım 4: Çalışma kitabını düz OPC dosyası olarak kaydedin

Çalışma kitabı ve kaydetme seçenekleri hazır olduğunda, dosyayı yazmak tek satırda yapılır. Ayrıca tüm akışı bir `Main` metoduna saracağız, böylece programı hemen çalıştırabilirsiniz.

```csharp
using System;
using Aspose.Cells;

class Program
{
    static void Main()
    {
        // 1️⃣ Build a workbook with sample data.
        Workbook wb = FlatOpcDemo.BuildSampleWorkbook();

        // 2️⃣ Get the correctly configured SaveOptions.
        SaveOptions flatOpcOptions = FlatOpcDemo.GetFlatOpcSaveOptions();

        // 3️⃣ Define the output path – adjust the folder to suit your environment.
        string outputPath = @"C:\Temp\demo.flat.opc";

        // 4️⃣ Save the workbook using the configured options.
        // This is the line that actually creates the flat OPC file.
        wb.Save(outputPath, flatOpcOptions);

        Console.WriteLine($"Flat OPC file created at: {outputPath}");
    }
}
```

Bu programı çalıştırdığınızda `demo.flat.opc` adlı bir dosya oluşturulur. Herhangi bir metin düzenleyiciyle açtığınızda, tüm çalışma sayfası verileri, stiller ve ilişkileri içeren tek bir XML belgesi göreceksiniz—tam olarak **Flat OPC** spesifikasyonunun belirttiği gibi.

## Doğrulama ve Beklenenler

Çalıştırdıktan sonra `C:\Temp\demo.flat.opc` konumuna (veya seçtiğiniz yola) gidin. Dosya aşağıdakine benzer bir şeyle başlayacaktır:

```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<package xmlns="http://schemas.openxmlformats.org/package/2006/content-types">
  <part name="/xl/workbook.xml" contentType="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet.main+xml">
    <!-- workbook XML goes here -->
  </part>
  <part name="/xl/worksheets/sheet1.xml" contentType="application/vnd.openxmlformats-officedocument.spreadsheetml.worksheet+xml">
    <!-- sheet data, including rows for Apples and Bananas -->
  </part>
  <!-- additional parts for styles, shared strings, etc. -->
</package>
```

**Flat OPC** formatı ZIP konteynerini tek bir XML'e dönüştürdüğü için, iki versiyonu normal bir `git diff` ile karşılaştırabilir ve hücre‑seviyesindeki değişiklikleri anında görebilirsiniz. Bu, ikili `.xlsx` paketine göre temel avantajdır.

### Sık Sorulan Sorular

- **Bu .NET Core ile çalışır mı?** Kesinlikle—Aspose.Cells çapraz platformdur ve aynı kod Windows, Linux veya macOS'ta çalışır.
- **Şifre korumalı bir çalışma kitabını dışa aktarmam gerekirse?** `Save` metodunu çağırmadan önce `SaveOptions` üzerindeki `Password` özelliğini ayarlayın. Flat OPC şifreleme meta verilerini içerecektir.
- **Çıktıyı diske yazmak yerine akış olarak alabilir miyim?** Evet. `wb.Save(Stream, SaveOptions)` aşırı yüklemesini kullanın ve akışı ihtiyacınız olan yere (HTTP yanıtı, Azure Blob vb.) yönlendirin.
- **Flat OPC dosyası normal bir .xlsx'den daha büyük mü?** Genellikle düz XML olduğu için biraz daha büyük olur, ancak takas insan okunabilirliğidir.
