# Конвертация docx в xps на C# – Полное руководство

Когда‑то вам нужно было **конвертировать docx в xps**, но вы не знали, какой вызов API использовать? Вы не одиноки — многие разработчики сталкиваются с этой проблемой, пытаясь автоматизировать генерацию отчётов или архивировать файлы Word в формате фиксированного макета. Хорошая новость? С несколькими строками C# и правильными параметрами вы можете экспортировать Word в XPS, загрузить docx в коде и сохранить документ как XPS без внешних инструментов.

В этом руководстве мы пройдём весь процесс, от чтения файла `.docx` с диска до получения высококачественного XPS‑файла, сохраняющего шрифты, макет и даже селекторы вариаций шрифтов. К концу вы получите готовый к запуску пример, который можно добавить в любой .NET‑проект.

## Что понадобится

Прежде чем начать, убедитесь, что у вас есть:

* **Aspose.Words for .NET** (или любая библиотека, предоставляющая `Document`, `XpsSaveOptions` и т.п.). Имя пакета NuGet — `Aspose.Words`.
* **.NET 6.0** или новее — код также работает на .NET Framework 4.6+, но мы будем использовать .NET 6 для краткости.
* **Пример DOCX**‑файла, который вы хотите конвертировать. Поместите его в папку, например `C:\Docs\input.docx`.
* IDE (Visual Studio, Rider или VS Code) — что‑угодно, где можно собрать C#.

Дополнительные зависимости не требуются; библиотека сама справится со всей тяжёлой работой.

> **Pro tip:** Если вы работаете на CI‑сервере, добавьте пакет NuGet в ваш `csproj`, чтобы сборка восстанавливала его автоматически.

## Шаг 1 — Загрузка DOCX в коде

Первое, что нужно сделать, — сообщить библиотеке, где находится исходный документ. Это шаг **load docx in code**, и он так же прост, как создание объекта `Document`.

```csharp
using Aspose.Words;

// Step 1: Load the source document
string inputPath = @"C:\Docs\input.docx";
Document doc = new Document(inputPath);
```

*Почему это важно:* Загрузка DOCX даёт вам представление файла Word в памяти, включая стили, изображения и пользовательские XML‑части. Теперь вы можете программно манипулировать документом — добавлять колонтитулы, заменять текст или, как мы сделаем дальше, **export word to xps**.

## Шаг 2 — Настройка параметров сохранения XPS (включение Font Variation Selectors)

Если просто вызвать `doc.Save("output.xps")`, библиотека использует настройки по умолчанию. Для большинства сценариев этого достаточно, но если ваш документ использует OpenType‑селекторы вариаций шрифтов (например, переменные шрифты для адаптивного дизайна), нужно включить эту возможность. Здесь и находится конфигурация **save document as xps**.

```csharp
// Step 2: Create XPS save options and enable font variation selectors
XpsSaveOptions xpsOptions = new XpsSaveOptions
{
    // Ensures variable fonts are retained in the XPS output
    FontVariationSelectors = true
};
```

Включение `FontVariationSelectors` гарантирует, что итоговый XPS‑файл будет выглядеть идентично оригинальному макету Word, даже на устройствах, поддерживающих переменные шрифты.

## Шаг 3 — Сохранение документа как XPS

Теперь, когда документ загружен и параметры заданы, пришло время **save word as xps**. Этот шаг записывает XPS‑файл на диск.

```csharp
// Step 3: Save the document as XPS with the configured options
string outputPath = @"C:\Docs\var-font.xps";
doc.Save(outputPath, xpsOptions);
```

Если всё прошло успешно, рядом с исходным файлом появится `var-font.xps`. Откройте его в Windows XPS Viewer, чтобы убедиться, что макет, шрифты и селекторы вариаций сохранены.

## Полный рабочий пример

Объединив три шага, получаем компактную, автономную программу, которую можно запускать из командной строки.

```csharp
using System;
using Aspose.Words;

namespace DocxToXpsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Paths – adjust to your environment
            string inputPath = @"C:\Docs\input.docx";
            string outputPath = @"C:\Docs\var-font.xps";

            // Load the DOCX file (load docx in code)
            Document doc = new Document(inputPath);

            // Configure XPS options (export word to xps with font variation selectors)
            XpsSaveOptions options = new XpsSaveOptions
            {
                FontVariationSelectors = true
            };

            // Save as XPS (save word as xps / save document as xps)
            doc.Save(outputPath, options);

            Console.WriteLine($"Successfully converted '{inputPath}' to XPS at '{outputPath}'.");
        }
    }
}
```

Запуск программы выводит сообщение‑подтверждение, и у вас появляется корректный XPS‑файл, готовый к распространению, архивированию или печати.

## Проверка результата

После конвертации может возникнуть вопрос: *Действительно ли шрифты остались теми же?* Самый простой способ проверить:

1. Откройте сгенерированный XPS‑файл в **Windows XPS Viewer**.
2. Сравните страницу, использующую переменный шрифт (например, заголовок с изменением толщины), с оригинальным документом Word.
3. Если визуальное отображение совпадает, конвертация прошла успешно.

Если заметны расхождения, дважды проверьте, что исходный DOCX действительно содержит данные о вариациях шрифтов и что на целевой машине установлены необходимые шрифты.

## Крайние случаи и распространённые подводные камни

| Ситуация | На что обратить внимание | Решение / обходной путь |
|-----------|-------------------|-------------------|
| **Большой DOCX ( > 100 МБ )** | Давление на память при загрузке | Использовать `LoadOptions` с `LoadFormat.Docx` и поток `FileStream`, чтобы избежать полной загрузки файла в память. |
| **Отсутствующие шрифты** | XPS переходит к шрифту по умолчанию, меняя макет | Установить недостающие шрифты на сервере конвертации или встроить их, задав `XpsSaveOptions.EmbedFullFonts = true`. |
| **DOCX с паролем** | `Document` бросает исключение | Передать пароль через `LoadOptions.Password`. |
| **Нужна только часть документа** | Конвертация всего файла тратит время | Использовать `Document.Clone()` для извлечения конкретного `Section` и сохранять только эту секцию. |
| **Запуск на Linux/macOS** | Нет Windows XPS Viewer | Использовать сторонний рендерер XPS (например, `PdfSharp` для конвертации XPS → PDF) или просматривать с помощью `libgxps`. |

Учёт этих сценариев делает ваш конвейер **convert docx to xps** надёжным для производственных нагрузок.

## Когда использовать XPS вместо PDF

Вы можете задаться вопросом: «Зачем вообще XPS, если PDF так популярен?» Вот несколько причин:

* **Фиксированная точность макета** — XPS сохраняет точный макет и рендеринг шрифтов, что важно для юридических документов.
* **Интеграция с печатью Windows** — XPS нативно поддерживается стеком печати Windows.
* **Будущее‑ориентированность** — Некоторые корпоративные решения для архивирования требуют XPS для соответствия требованиям.

Если нужен формат, открываемый везде, вы можете сначала **export word to xps**, а затем преобразовать XPS в PDF с помощью `Aspose.Pdf` или открытых утилит.

## Следующие шаги

Теперь, когда вы знаете, как **convert docx to xps**, можно расширить процесс:

* **Пакетная конвертация** — пробегать по папке с DOCX‑файлами и создавать ZIP‑архив XPS‑документов.
* **Добавление водяных знаков** — использовать `DocumentBuilder` для вставки водяного знака перед сохранением.
* **Внедрение метаданных** — заполнять свойства XPS‑документа (author, title) через `XpsSaveOptions` для лучшего управления документами.

Все эти возможности опираются на те же базовые шаги, которые мы рассмотрели, поэтому переход будет плавным.

### Быстрый обзор

* Загрузите DOCX в коде (конструктор `Document`).
