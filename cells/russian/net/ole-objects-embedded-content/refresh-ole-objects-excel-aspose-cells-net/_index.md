### Обновление объектов OLE (H2)

#### Обзор:
Обновление объектов OLE гарантирует, что ваши встроенные или связанные данные отражают последние обновления. Эта функция особенно полезна для поддержания актуальных отчетов и панелей мониторинга непосредственно в файлах Excel.

#### Пошаговая реализация:

##### 1. Загрузите существующую рабочую книгу
```csharp
// Укажите исходный каталог
string SourceDir = @"YOUR_SOURCE_DIRECTORY";
Workbook wb = new Workbook(SourceDir + "sample.xlsx");
```
*Почему?*Этот шаг инициализирует вашу рабочую книгу и подготавливает ее к изменению путем загрузки существующего файла.

##### 2. Доступ к определенному рабочему листу
```csharp
// Доступ к первому рабочему листу
Worksheet sheet = wb.Worksheets[0];
```
*Почему?*: Выбор соответствующего рабочего листа имеет важное значение для точного определения местонахождения объектов OLE.

##### 3. Установите свойство AutoLoad для объектов OLE
```csharp
// Обновите первый объект OLE, установив его свойство AutoLoad в значение true.
sheet.OleObjects[0].AutoLoad = true;
```
*Почему?*: Эта конфигурация предписывает Excel автоматически обновлять данные, гарантируя, что у вас всегда будет самая актуальная информация.

##### 4. Сохраните обновленную рабочую книгу.
```csharp
// Укажите выходной каталог и сохраните книгу.
string outputDir = @"YOUR_OUTPUT_DIRECTORY";
wb.Save(outputDir + "RefreshOLEObjects_out.xlsx", SaveFormat.Xlsx);
```
*Почему?*: Сохранение рабочей книги закрепляет ваши изменения, делая их доступными для использования в будущем.
