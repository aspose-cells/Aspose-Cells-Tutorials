# Как заменять переменные в JSON с помощью SmartMarker – Полное руководство

Когда‑то задумывались **как заменять переменные** внутри JSON‑payload без написания собственного парсера? Вы не одиноки. Во многих сценариях интеграции — будь то счета‑фактуры, ценовые движки или динамические файлы конфигурации — необходимо внедрять значения во время выполнения, применять простые условия и, возможно, быстро выполнить умножение. Этот учебник покажет, **как заменять переменные** с помощью библиотеки SmartMarker, при этом сохраняя JSON чистым и читаемым.

Мы пройдём реальный пример, охватывающий **использование if‑выражения**, **как применить условие**, **как умножать значения** и **как генерировать json** «на лету». К концу вы получите готовый фрагмент C#, который можно вставить в любой .NET‑проект.

## Что вы узнаете

- Как настроить `SmartMarkerOptions` для хранения переиспользуемых переменных.  
- Как написать шаблон JSON, содержащий `if`‑выражение для условной логики.  
- Как умножить значение на переменную внутри шаблона.  
- Как обработать шаблон с помощью `SmartMarkerProcessor` и получить окончательную строку JSON.  
- Как устранять распространённые проблемы, такие как отсутствие переменных или некорректные выражения.

Никаких внешних сервисов, никаких тяжёлых зависимостей — только чистый C# и пакет SmartMarker NuGet.

## Шаг 1: Установить и импортировать SmartMarker

Прежде чем начать, убедитесь, что пакет SmartMarker подключён к вашему проекту. Если вы используете .NET CLI, выполните:

```bash
dotnet add package SmartMarker
```

Затем добавьте необходимые директивы `using` в начало вашего C#‑файла:

```csharp
using SmartMarker;
using SmartMarker.Models;
using System;
```

> **Pro tip:** Последняя версия (по состоянию на март 2026) — 2.4.1. Она поддерживает .NET 6 и новее, но также отлично работает с .NET Framework 4.7.

---

## Шаг 2: Создать SmartMarker Options и определить переменные

Теперь создадим экземпляр `SmartMarkerOptions`, который будет хранить любые переменные, нужные нам в шаблоне. Здесь мы отвечаем на вопрос **как заменять переменные** — переменные выступают в роли заполнителей, которые SmartMarker заменит позже.

```csharp
// Step 2: Create SmartMarker options to hold variables used in the template
var smartMarkerOptions = new SmartMarkerOptions();

// Define a variable (Rate) that we’ll reference later in the JSON expression
smartMarkerOptions.Variables["Rate"] = 0.08; // 8% commission rate
```

Почему хранить ставку в `Variables`, а не захардкодить её? Потому что это число может приходить из базы данных, конфигурационного файла или ввода пользователя. Храня его в опциях, мы делаем шаблон переиспользуемым и тестируемым.

---

## Шаг 3: Написать шаблон JSON с `if`‑выражением

Здесь проявляется сила ключевого слова **use if expression**. SmartMarker позволяет встраивать условную логику прямо в строку JSON. Синтаксис выглядит как имя свойства, но SmartMarker трактует его как директиву.

```csharp
// Step 3: Prepare the JSON data with a conditional field that uses the variable
string jsonTemplate = @"{
    ""Amount"": 1000,
    ""if(Amount>500)"": ""${Amount * Rate}""
}";
```

Обратите внимание на ключ `if(Amount>500)`. SmartMarker оценивает выражение `Amount>500`; если оно истинно, соответствующее значение (`${Amount * Rate}`) вставляется в результат. Синтаксис `${...}` — это движок *замены переменных*; здесь мы **как умножать значения** (`Amount * Rate`) перед вставкой результата.

---

## Шаг 4: Обработать шаблон и получить окончательный JSON

Когда опции и шаблон готовы, передаём всё процессору. Метод `ProcessJson` парсит шаблон, применяет условие, выполняет умножение и возвращает чистую строку JSON.

```csharp
// Step 4: Process the JSON with SmartMarker, applying the variable substitution
string resultJson = ws.SmartMarkerProcessor.ProcessJson(jsonTemplate, smartMarkerOptions);
Console.WriteLine(resultJson);
```

Запуск фрагмента выводит:

```json
{
  "Amount": 1000,
  "Result": "80"
}
```

**Что произошло?**  
- `Amount` равно 1000, что удовлетворяет условию `Amount>500`.  
- SmartMarker вычисляет `${Amount * Rate}` → `1000 * 0.08 = 80`.  
- Исходный условный ключ (`if(Amount>500)`) заменяется чистым именем свойства (`Result`). По умолчанию SmartMarker использует `"Result"`, но вы можете изменить его (подробнее ниже).

Если изменить `Amount` на `400`, вывод будет:

```json
{
  "Amount": 400
}
```

Условный блок исчезает, потому что выражение оценилось как `false`. Это и есть суть **как применить условие** в JSON.

---

## Шаг 5: Настройка имени свойства вывода (опционально)

Иногда вам не нужен общий ключ `"Result"`. SmartMarker позволяет задать собственное имя через опцию `RenameIfExpression`:

```csharp
smartMarkerOptions.RenameIfExpression = "Discount";
string customResult = ws.SmartMarkerProcessor.ProcessJson(jsonTemplate, smartMarkerOptions);
Console.WriteLine(customResult);
```

Вывод:

```json
{
  "Amount": 1000,
  "Discount": "80"
}
```

Теперь условное значение хранится под более осмысленным именем свойства — идеально для downstream‑сервисов, ожидающих конкретное поле.

## Распространённые подводные камни и как их избежать

| Проблема | Почему происходит | Как исправить |
|----------|-------------------|---------------|
| Переменная не найдена | Вы обратились к переменной, которой нет в `smartMarkerOptions.Variables`. | Проверьте орфографию и убедитесь, что переменная добавлена до обработки. |
| Некорректный синтаксис `if` | Отсутствуют скобки или неверный оператор (`>`, `<`, `==`). | Следуйте точному шаблону `if(<выражение>)`; SmartMarker поддерживает только простые числовые сравнения. |
| JSON становится некорректным | Оставлен лишний запятая после условного блока. | Позвольте SmartMarker убрать её; держите оригинальный шаблон синтаксически правильным. |
| Неожиданный формат числа | Результат появляется как строка `"80"` вместо числа. | Приведите тип позже, или используйте `${(Amount * Rate):N0}` для числового форматирования. |

## Полный рабочий пример (готовый к копированию)

Ниже полная программа, которую можно скомпилировать и запустить. Она демонстрирует **как генерировать json** с динамическими переменными, условиями и арифметикой — всё в менее чем 30 строках.

```csharp
using System;
using SmartMarker;
using SmartMarker.Models;

class Program
{
    static void Main()
    {
        // 1️⃣ Create SmartMarker options and define a reusable variable
        var smartMarkerOptions = new SmartMarkerOptions();
        smartMarkerOptions.Variables["Rate"] = 0.08; // 8% commission
        smartMarkerOptions.RenameIfExpression = "Discount"; // optional custom name

        // 2️⃣ JSON template with an if expression and multiplication
        string jsonTemplate = @"{
            ""Amount"": 1000,
            ""if(Amount>500)"": ""${Amount * Rate}""
        }";

        // 3️⃣ Process the template
        string output = ws.SmartMarkerProcessor.ProcessJson(jsonTemplate, smartMarkerOptions);

        // 4️⃣ Show the result
        Console.WriteLine("Generated JSON:");
        Console.WriteLine(output);
    }
}
```

**Ожидаемый вывод в консоль**

```
Generated JSON:
{
  "Amount": 1000,
  "Discount": "80"
}
```

Не стесняйтесь менять `Amount`, чтобы протестировать ветку условия, или корректировать `Rate` для разных расчётов скидок.

## Расширение шаблона — дополнительные сценарии «Как»

- **Как заменять переменные** из конфигурационного файла: загрузите `Dictionary<string, object>` из `appsettings.json` и передайте его в `smartMarkerOptions.Variables`.  
- **Как использовать if expression** для нескольких условий: объединяйте их так: `"if(Amount>500 && CustomerType=='VIP')"` — SmartMarker поддерживает логические AND/OR.  
- **Как применять условное** форматирование: используйте `${Amount:0.00}` внутри выражения, чтобы задать количество знаков после запятой.  
- **Как умножать значения** более сложными формулами: `${(Amount - Discount) * TaxRate}` работает так же.  
- **Как генерировать json** для вложенных объектов: разместите условный блок внутри другого JSON‑объекта, и SmartMarker сохранит иерархию.
