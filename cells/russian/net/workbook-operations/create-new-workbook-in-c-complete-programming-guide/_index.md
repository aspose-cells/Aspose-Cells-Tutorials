# Создание новой книги в C# – Полное руководство по программированию

Ever needed to **create new workbook** in C# but weren’t sure where to start? You’re not the only one. Whether you’re automating a reporting pipeline or just playing with Excel formulas in code, the ability to spin up a workbook, drop in formulas like `EXPAND` or `COT`, and then **save workbook to file** is a core skill for any .NET developer.

В этом руководстве мы пройдем реальный пример, который делает именно это: создадим новую книгу, используем функцию `EXPAND`, чтобы превратить статический массив в динамический столбец, вычислим котангенс с помощью функции `COT` и, наконец, **сохраним книгу в файл** в формате `.xlsx`. К концу вы получите готовый к запуску фрагмент кода, поймёте *почему* каждый вызов важен и увидите несколько полезных вариантов для особых случаев.

> **Pro tip:** Весь код ниже работает с последней версией Aspose.Cells для .NET (по состоянию на март 2026). Если вы используете более старую версию, API в основном одинаковый, но проверьте импорт пространств имён.

## Что понадобится

- .NET 6.0 или новее (пример ориентирован на .NET 6, но .NET 5 тоже работает)  
- Aspose.Cells для .NET, установленный через NuGet (`Install-Package Aspose.Cells`)  
- Небольшие знания C# (у вас всё получится)  

Вот и всё — никаких дополнительных DLL, без COM‑interop и, конечно, без установленного Excel на машине. Готовы? Погружаемся.

## Шаг 1: Создать новую книгу

Первое, что нужно сделать, — создать экземпляр класса `Workbook`. Представьте его как открытый пустой файл Excel в памяти. Этот объект содержит коллекцию листов, стилей и всё остальное, что понадобится позже.

```csharp
using Aspose.Cells;

class ExcelDemo
{
    static void Main()
    {
        // Step 1: Create a new workbook and grab the first worksheet
        Workbook workbook = new Workbook();               // <-- creates an empty .xlsx structure
        Worksheet ws = workbook.Worksheets[0];            // default sheet is named "Sheet1"
```

Зачем сразу брать первый лист? Большинство быстрых примеров работают с единственным листом, и доступ `Worksheets[0]` — самый быстрый способ получить ссылку без цикла. Если позже понадобится несколько листов, их можно добавить с помощью `workbook.Worksheets.Add()`.

## Шаг 2: Как использовать EXPAND для создания динамических диапазонов

`EXPAND` — более новая функция Excel, которая принимает массив и заполняет его до указанного размера. В нашем коде мы расширим литеральный массив `{1,2,3}` в **столбец из 5 строк**, начиная с ячейки `A1`. Синтаксис внутри строки точно такой же, как в Excel, поэтому вы можете скопировать‑вставить его напрямую в ячейку позже, если захотите.

```csharp
        // Step 2: Apply EXPAND to turn {1,2,3} into a 5‑row vertical range
        ws.Cells["A1"].Formula = "=EXPAND({1,2,3},5,1)"; // rows=5, cols=1
```

### Что происходит под капотом?

- `{1,2,3}` — горизонтальный массив‑литерал.  
- Второй аргумент (`5`) указывает Excel расширить массив до **5 строк**.  
- Третий аргумент (`1`) заставляет выводить **один столбец**.  

Если опустить третий аргумент, Excel попытается сохранить исходную форму, что может дать блок 5×3 вместо одного столбца. Это распространённая ловушка при первом эксперименте с `EXPAND`.

#### Вариации, которые могут понадобиться

| Желаемая форма | Пример формулы |
|----------------|----------------|
| Блок 3 строки, 2 столбца | `=EXPAND({1,2,3},3,2)` |
| Заполнение только вниз (тот же столбец) | `=EXPAND({10,20},10,1)` |
| Расширить до большего количества столбцов | `=EXPAND({5},5,4)` |

Не стесняйтесь менять литералы или размеры, чтобы соответствовать вашей логике генерации данных.

## Шаг 3: Как вычислить котангенс с помощью функции COT

Функция `COT` возвращает котангенс угла, выраженного в радианах. В нашем примере мы вычисляем котангенс 45° (π/4 радиан). Результат, `1`, попадает в ячейку `B1`.

```csharp
        // Step 3: Use COT to calculate cotangent of 45 degrees (π/4 radians)
        ws.Cells["B1"].Formula = "=COT(PI()/4)"; // PI() returns π, divided by 4 = 45°
```

### Почему использовать COT вместо ручного вычисления?

Excel уже умеет выполнять тригонометрическое преобразование, поэтому вы избегаете ошибок округления с плавающей точкой, которые могут появиться при попытке вычислить `1 / TAN(angle)`. Кроме того, формула остаётся читаемой для любого, кто позже будет просматривать таблицу.

#### Пограничный случай: углы за пределами 0‑360°

Если задать угол больше `2*PI()` (или отрицательный), Excel автоматически выполнит его обёртку, но результат может удивить. Чтобы быть уверенным, лучше сначала нормализовать угол:

```csharp
        // Normalize angle to 0‑2π range before applying COT
        ws.Cells["C1"].Formula = "=COT(MOD(PI()*3, 2*PI()))";
```

Этот фрагмент демонстрирует, как сочетать `MOD` с `COT` для надёжных вычислений.

## Шаг 4: Как сохранить книгу в файл (Excel)

Теперь, когда формулы находятся на месте, последний шаг — **сохранить книгу в файл**. Вы можете выбрать любой путь — просто убедитесь, что каталог существует и у вас есть права записи.

```csharp
        // Step 4 (optional): Save the workbook so you can inspect the results
        string outputPath = Path.Combine(
            Environment.GetFolderPath(Environment.SpecialFolder.Desktop),
            "output.xlsx");

        workbook.Save(outputPath);
        Console.WriteLine($"Workbook saved to {outputPath}");
    }
}
```

### Что именно сохраняется?

Когда вы откроете `output.xlsx` в Excel, вы увидите:

| A | B |
|---|---|
| 1 | 1 |
| 2 |   |
| 3 |   |
|   |   |
|   |   |

- Столбец **A** содержит расширенный массив `{1,2,3}`, за которым следуют две пустые ячейки (поскольку мы запросили 5 строк).  
- Ячейка **B1** показывает `1`, котангенс 45°.  

Если обновить книгу (нажать `F9` или включить автоматический расчёт), Excel вычислит формулы и отобразит результаты. Aspose.Cells также предоставляет метод `CalculateFormula`, если нужны значения без открытия Excel:

```csharp
        workbook.CalculateFormula();
        double cotResult = ws.Cells["B1"].DoubleValue; // should be 1.0
```

## Часто задаваемые вопросы и подводные камни

| Вопрос | Ответ |
|--------|-------|
| **Нужно ли включать вычисление вручную?** | Нет. По умолчанию Aspose.Cells сохраняет формулы как есть; Excel вычислит их при открытии. Используйте `workbook.CalculateFormula()` для предварительного расчёта. |
| **Можно ли записать формулы в несколько ячеек одновременно?** | Конечно. Используйте `ws.Cells["D1:D5"].Formula = "=RAND()"`, чтобы заполнить диапазон случайными числами. |
| **Что делать, если целевая папка не существует?** | Создайте её сначала: `Directory.CreateDirectory(Path.GetDirectoryName(outputPath));` |
| **Поддерживается ли `EXPAND` в более старых версиях Excel?** | `EXPAND` появился в Excel 365/2019. Если нужна совместимость со старыми файлами, рассмотрите использование комбинаций `INDEX`/`SEQUENCE`. |
| **Как скрыть просмотр формул?** | Установите `ws.Cells["A1"].FormulaHidden = true;` и защитите лист, если не хотите, чтобы пользователи видели исходную формулу. |
