### Добавление свойств ContentType
Aspose.Cells упрощает добавление пользовательских свойств, которые можно использовать для различных целей, например, для определения метаданных или отслеживания дополнительной информации о ваших книгах Excel.

#### Пошаговый обзор
1. **Создайте новую рабочую книгу:** Инициализируйте новый экземпляр `Workbook` сорт.
2. **Добавьте свойства ContentType:** Используйте `ContentTypeProperties.Add()` метод включения пользовательских свойств.
3. **Настроить свойство Nillable:** Установите, можно ли обнулить каждое свойство или нет.

#### Реализация кода
```csharp
using Aspose.Cells.WebExtensions;
using System;

namespace Aspose.Cells.Examples.CSharp._Workbook
{
    public class WorkingWithContentTypeProperties
    {
        public static void Run()
        {
            // Инициализируйте новую рабочую книгу в формате XLSX
            Workbook workbook = new Workbook(FileFormatType.Xlsx);
            
            // Добавьте строку ContentType Property "MK31"
            int index1 = workbook.ContentTypeProperties.Add("MK31", "Simple Data");
            workbook.ContentTypeProperties[index1].IsNillable = false;
            
            // Добавьте свойство DateTime ContentType «MK32»
            int index2 = workbook.ContentTypeProperties.Add("MK32", DateTime.Now.ToString("yyyy-MM-dd'T'hh:mm:ss"), "DateTime");
            workbook.ContentTypeProperties[index2].IsNillable = true;

            // Сохраните рабочую книгу
            string outputDir = RunExamples.Get_OutputDirectory();
            workbook.Save(outputDir + "WorkingWithContentTypeProperties_out.xlsx");

            Console.WriteLine("ContentType Properties added successfully.");
        }
    }
}
```

### Объяснение параметров и методов
- **Добавить метод:** The `Add` Метод принимает уникальный идентификатор, значение и необязательный тип содержимого.
  - **Параметры:**
    - Идентификатор (строка): уникальное имя свойства.
    - Значение (объект): данные, связанные с этим свойством.
    - Тип содержимого (необязательно, строка): указывает тип данных, например «DateTime».
- **IsNillable:** Логическое значение, указывающее, можно ли оставить свойство пустым.

### Советы по устранению неполадок
- Во избежание конфликтов обеспечьте уникальные идентификаторы для каждого свойства ContentType.
- Убедитесь, что при добавлении свойств используются правильные типы данных.
