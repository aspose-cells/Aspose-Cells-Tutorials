# Как вставлять строки в GridJs – быстро добавлять несколько строк

Когда‑то задавались вопросом **как вставлять строки** в data‑grid GridJs без написания бесконечного цикла? Вы не одиноки. Во многих корпоративных приложениях наступает момент, когда нужно освободить место для массового импорта, шаблона или просто заполнителя для будущих данных. Хорошая новость: GridJs предоставляет один метод, который делает всю тяжёлую работу за вас.

В этом руководстве мы пройдём через полностью готовый к запуску пример, показывающий, как **добавить 100 строк**, **создать пустые строки** и **проверить общее количество строк** после операции. К концу вы получите надёжный шаблон, который можно внедрить в любой C#‑проект, использующий GridJs.

## Требования

Прежде чем приступить, убедитесь, что у вас есть:

- .NET 6.0 или новее (API работает одинаково и в .NET Framework 4.8, но более новый SDK предоставляет более удобные инструменты).
- Ссылка на пакет NuGet `GridJs` или скомпилированный DLL, содержащий класс `GridJs`.
- Базовое знакомство с синтаксисом C# — ничего экзотического, только стандартные `using`‑директивы и основы объектно‑ориентированного программирования.

Если что‑то из перечисленного вызывает вопросы, сделайте паузу и исправьте ситуацию. Последующие шаги предполагают, что объект сетки уже создан и готов принимать строки.

![how to insert rows illustration](gridjs-insert-rows.png)

## Шаг 1: Создание экземпляра Grid

Первым делом нужен объект `GridJs`. В реальном приложении он, скорее всего, будет получен из сервисного слоя или внедрён через dependency injection, но для наглядности создадим его локально.

```csharp
using System;
using GridJsLibrary;   // <-- replace with the actual namespace of GridJs

namespace GridJsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Step 1: Create or obtain the grid you want to modify
            GridJs gridJs = new GridJs();   // replace with your actual grid initialization
```

> **Почему это важно:** Создание сетки даёт чистый лист, гарантируя, что логика вставки строк не конфликтует с оставшимся состоянием от предыдущих запусков.

## Шаг 2: Вставка 100 строк в определённый индекс

Теперь переходим к основной части **как вставлять строки**. Метод `InsertRows` принимает два аргумента: нулевой индекс начала и количество строк, которые нужно добавить. Вставим 100 строк, начиная с пятой строки (индекс 5).

```csharp
            // Step 2: Insert 100 rows starting at row index 5 (zero‑based)
            // This pushes existing rows down and creates space for new data.
            gridJs.InsertRows(5, 100);
```

> **Совет:** Если нужно добавить строки в конец сетки, используйте `gridJs.RowCount` в качестве стартового индекса. Таким образом вы фактически «добавляете» строки, а не вставляете их.

### Что происходит «под капотом»?

- **Выделение памяти:** `InsertRows` внутри выделяет блок пустых объектов строк, так что вам не придётся вручную создавать каждый объект.
- **Смещение индексов:** Все строки, находившиеся на индексе 5 и далее, сдвигаются вниз на 100 позиций, сохраняя свои данные.
- **Производительность:** Поскольку операция выполняется одним вызовом, она обычно быстрее, чем 100‑кратный вызов `InsertRow`.

## Шаг 3: Проверка вставки (подсчёт общего количества строк)

После добавления строк полезно **проверить общее количество строк**, чтобы убедиться, что операция прошла успешно. Свойство `RowCount` возвращает текущее число строк в сетке.

```csharp
            // Step 3: (Optional) Verify the insertion or continue processing
            int newRowCount = gridJs.RowCount; // example property to check total rows
            Console.WriteLine($"Grid now contains {newRowCount} rows.");
```

Если у вас было, скажем, 20 строк, в консоли должно отобразиться `120`. Эта простая проверка может сэкономить часы отладки позже.

## Шаг 4: Заполнение только что созданных пустых строк (по желанию)

Часто требуется заполнить эти только что созданные строки данными‑заполнителями или объектами по умолчанию. Поскольку `InsertRows` предоставляет блок пустых строк, можно пройтись по диапазону и присвоить значения.

```csharp
            // Optional: Fill the newly created rows with default values
            for (int i = 5; i < 5 + 100; i++)
            {
                var row = gridJs.GetRow(i); // assume GetRow returns a mutable row object
                row["Name"] = $"Placeholder {i - 4}";
                row["CreatedOn"] = DateTime.UtcNow;
            }

            // Verify a sample row
            var sample = gridJs.GetRow(5);
            Console.WriteLine($"First inserted row name: {sample["Name"]}");
        }
    }
}
```

> **Зачем это может понадобиться:** Создание пустых строк удобно, когда нужен шаблон для ввода пользователем, заполнитель для пакетной загрузки или просто резервирование места для будущих вычислений.

## Распространённые варианты и граничные случаи

### Добавление менее чем 100 строк

Если нужно **добавить несколько строк** — например, 10 или 25, тот же вызов `InsertRows` работает; просто замените `100` нужным числом.

```csharp
gridJs.InsertRows(startIndex, 25); // adds 25 rows
```

### Вставка в начало сетки

Хотите добавить строки в начало? Используйте `0` в качестве стартового индекса:

```csharp
gridJs.InsertRows(0, 5); // adds 5 rows at the very beginning
```

### Обработка индексов вне диапазона

Передача индекса, превышающего `RowCount`, вызывает `ArgumentOutOfRangeException`. Защититесь от этого:

```csharp
int safeIndex = Math.Min(requestedIndex, gridJs.RowCount);
gridJs.InsertRows(safeIndex, 100);
```

### Работа с только‑для‑чтения сетками

Некоторые конфигурации GridJs предоставляют только‑для‑чтения представление. В таком случае нужно переключиться на записываемый экземпляр или временно отключить флаг только‑для‑чтения перед вызовом `InsertRows`.

## Советы по производительности

- **Пакетные операции:** Если вставляете строки многократно в цикле, объединяйте их в один вызов `InsertRows` насколько это возможно. Это уменьшает количество внутренних перераспределений списка.
- **Избегайте обновления UI:** В UI‑связанных сетках приостановите рендеринг (`gridJs.BeginUpdate()`) перед вставкой строк и возобновите (`gridJs.EndUpdate()`) после, чтобы избежать мерцания.
- **Профилирование памяти:** Большие вставки (например, >10 000 строк) могут резко увеличить потребление памяти. Рассмотрите постраничную загрузку или потоковую передачу данных вместо одной массивной вставки.

## Полный рабочий пример

Объединяя всё вместе, получаем готовую к копированию и запуску программу:

```csharp
using System;
using GridJsLibrary;   // replace with the actual namespace

namespace GridJsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Create the grid instance
            GridJs gridJs = new GridJs();

            // Insert 100 rows starting at index 5
            gridJs.InsertRows(5, 100);

            // Verify insertion
            int newRowCount = gridJs.RowCount;
            Console.WriteLine($"Grid now contains {newRowCount} rows.");

            // Optional: Fill new rows with placeholder data
            for (int i = 5; i < 5 + 100; i++)
            {
                var row = gridJs.GetRow(i);
                row["Name"] = $"Placeholder {i - 4}";
                row["CreatedOn"] = DateTime.UtcNow;
            }

            // Show a sample row
            var sample = gridJs.GetRow(5);
            Console.WriteLine($"First inserted row name: {sample["Name"]}");
        }
    }
}
```

Запустите её, и в консоли отобразится подтверждение количества строк и имя первой заполненной строки‑заполнителя. Это полный ответ на вопрос **как вставлять строки** в GridJs, включая проверку и необязательное заполнение данными.
