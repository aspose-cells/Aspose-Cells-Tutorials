## Загрузить книгу с паролем
### Обзор
Эта функция позволяет вам открыть зашифрованный файл Excel с помощью указанного пароля. Это необходимо при работе с защищенными рабочими книгами, содержащими конфиденциальные данные.
### Этапы реализации:
#### 1. Укажите исходный каталог
Определите, где хранятся ваши файлы Excel. Этот путь к каталогу будет использоваться для поиска и загрузки рабочей книги.
```csharp
string SourceDir = @"YOUR_SOURCE_DIRECTORY";
```
#### 2. Создайте LoadOptions и установите пароль
Инициализировать `LoadOptions` и назначьте пароль, необходимый для открытия зашифрованного файла.
```csharp
LoadOptions loadOptions = new LoadOptions();
loadOptions.Password = "1234"; // Используйте здесь свой настоящий пароль
```
#### 3. Откройте зашифрованный файл Excel.
Используйте `Workbook` класс с указанными параметрами загрузки для доступа к файлу.
```csharp
Workbook workbook = new Workbook(SourceDir + "/sampleCheckPasswordToModify.xlsx", loadOptions);
```
**Советы по устранению неполадок:**
- Убедитесь, что пароль правильный и совпадает с паролем, используемым для шифрования.
- Убедитесь, что путь к файлу указан правильно и доступен из контекста вашего приложения.
## Подтвердите пароль для изменения рабочей книги
### Обзор
После загрузки рабочей книги вам может потребоваться проверить, разрешает ли данный пароль внесение изменений. Эта функция гарантирует, что только авторизованные пользователи могут изменять защищенные рабочие книги.
### Этапы реализации:
#### 1. Откройте файл Excel с помощью LoadOptions.
Предположим, что параметры нагрузки уже определены на предыдущем шаге:
```csharp
Workbook workbook = new Workbook(SourceDir + "/sampleCheckPasswordToModify.xlsx", loadOptions);
```
#### 2. Проверка паролей модификации
Использовать `ValidatePassword` для проверки возможности внесения изменений в определенные пароли.
```csharp
bool isCorrectPassword1 = workbook.Settings.WriteProtection.ValidatePassword("567");
bool isCorrectPassword2 = workbook.Settings.WriteProtection.ValidatePassword("5678");
```
**Основные соображения:**
- Только действительные пароли модификации вернут значение true.
- Убедитесь, что ваше приложение корректно обрабатывает ложные проверки, чтобы избежать попыток несанкционированного доступа.
