### Шифрование файлов Excel с помощью Aspose.Cells для .NET
**Обзор:**
Зашифруйте файлы Excel, чтобы защитить конфиденциальную информацию от несанкционированного доступа. В этом разделе показано, как применять различные типы шифрования с помощью Aspose.Cells.

#### Шаг 1: Настройте свой проект и загрузите рабочую книгу
```csharp
// Убедитесь, что вы правильно указали пути к каталогам в своей среде.
string SourceDir = @"YOUR_SOURCE_DIRECTORY";
string OutputDir = @"YOUR_OUTPUT_DIRECTORY";

Workbook workbook = new Workbook(SourceDir + "/Book1.xls");
```

#### Шаг 2: Укажите параметры шифрования
Выберите тип шифрования XOR или Strong Cryptographic Provider:
```csharp
// Используйте шифрование XOR с длиной ключа 40.
workbook.SetEncryptionOptions(EncryptionType.XOR, 40);

// В качестве альтернативы используйте надежное шифрование RC4 с длиной ключа 128 бит.
workbook.SetEncryptionOptions(EncryptionType.StrongCryptographicProvider, 128);
```

#### Шаг 3: Установите пароль файла
```csharp
// Защитите свой файл Excel, установив пароль.
workbook.Settings.Password = "1234";
```

#### Шаг 4: Сохраните зашифрованную книгу
```csharp
// Сохраните зашифрованную книгу в выходном каталоге.
workbook.Save(OutputDir + "/encryptedBook1.out.xls");
```

### Защита паролем для модификации с помощью Aspose.Cells
**Обзор:**
Предотвратите несанкционированное внесение изменений, установив пароль, необходимый для редактирования.

#### Шаг 1: Загрузите существующую рабочую книгу
```csharp
Workbook workbook = new Workbook(SourceDir + "/Book1.xls");
```

#### Шаг 2: Установите пароль защиты от записи
```csharp
// Определите пароль, необходимый для изменения файла Excel.
workbook.Settings.WriteProtection.Password = "1234";
```

#### Шаг 3: Сохраните защищенную книгу.
```csharp
// Сохраните книгу с включенной защитой от изменений.

```

**В1: Могу ли я использовать Aspose.Cells для .NET без лицензии?**

**В2: В чем разница между шифрованием XOR и шифрованием Strong Cryptographic Provider?**
A2: XOR менее безопасен при использовании ключей меньшей длины, в то время как Strong Cryptographic Provider обеспечивает повышенную безопасность с использованием шифрования RC4.

**В3: Как обрабатывать исключения при шифровании файлов с помощью Aspose.Cells?**
A3: Используйте блоки try-catch в своем коде, чтобы изящно управлять любыми потенциальными ошибками во время файловых операций.

**В4: Может ли Aspose.Cells защищать только определенные листы в файле Excel?**
A4: Хотя Aspose.Cells применяет параметры безопасности на уровне рабочей книги, вы можете программно управлять разрешениями доступа для отдельных листов, используя дополнительные функции .NET.

**В5: Какова максимальная длина пароля, разрешенная Aspose.Cells для шифрования?**
A5: Aspose.Cells поддерживает надежные пароли длиной до 255 символов.
