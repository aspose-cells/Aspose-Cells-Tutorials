## Руководство по внедрению

### Добавление зарегистрированных ссылок в проект VBA
В этом разделе вы узнаете, как добавлять зарегистрированные ссылки, например: `stdole` и `Office` Библиотеки, использующие Aspose.Cells для .NET.

#### Обзор
Добавление зарегистрированных ссылок имеет решающее значение, когда ваши макросы зависят от внешних DLL-библиотек, поскольку обеспечивает доступность всех компонентов во время выполнения.

#### Этапы внедрения
1. **Инициализировать рабочую книгу**
   Создайте новый экземпляр `Workbook` класс, представляющий файл Excel:
   ```csharp
   Workbook workbook = new Workbook();
   ```
2. **Доступ к проекту VBA**
   Получите проект VBA, связанный с вашей рабочей книгой:
   ```csharp
   VbaProject vbaProj = workbook.VbaProject;
   ```
3. **Добавить зарегистрированные ссылки**
   Добавить зарегистрированную ссылку на `stdole` и `Office` библиотеки, использующие соответствующие им идентификаторы GUID.
   - **Для библиотеки stdole:**
     ```csharp
     vbaProj.References.AddRegisteredReference(
         "stdole",
         "*\\G{00020430-0000-0000-C000-000000000046}#2.0#0#C:\\Windows\\system32\\stdole2.tlb#OLE Automation"
     );
     ```
   - **Для офисной библиотеки:**
     ```csharp
     vbaProj.References.AddRegisteredReference(
         "Office",
         "*\\G{2DF8D04C-5BFA-101B-BDE5-00AA0044DE52}#2.0#0#C:\\Program Files\\Common Files\\Microsoft Shared\\OFFICE14\\MSO.DLL#Microsoft Office 14.0 Object Library"
     );
     ```
   Каждый `AddRegisteredReference` Для вызова требуется имя библиотеки, ее GUID, номер версии и путь к файлу.
4. **Сохранить рабочую книгу**
   Сохраните вашу рабочую книгу с обновленным проектом VBA:
   ```csharp
   string outputPath = "YOUR_OUTPUT_DIRECTORY" + "Output_out.xlsm";
   workbook.Save(outputPath);
   ```
#### Советы по устранению неполадок
- Проверьте правильность и доступность путей к файлам DLL или TLB.
- Убедитесь, что GUID соответствует версии вашей библиотеки.
