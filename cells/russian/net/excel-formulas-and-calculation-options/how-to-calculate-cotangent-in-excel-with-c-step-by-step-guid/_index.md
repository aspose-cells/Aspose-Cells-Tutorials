# Как вычислить котангенс в Excel с помощью C# – Полный учебник

Вы когда‑нибудь задумывались **как вычислить котангенс** непосредственно в листе Excel из C#‑приложения? Возможно, вы создаёте финансовую модель, научный калькулятор или просто автоматизируете отчёт, и вам нужен котангенс угла без переноса данных в отдельный инструмент. Хорошая новость? С несколькими строками кода вы можете **создать книгу Excel**, поместить формулу `COT` в ячейку и наблюдать, как Excel выполняет вычисления.

В этом учебнике мы пройдём весь процесс: от инициализации книги, через использование функции `EXPAND` для преобразования данных, до **установки формулы** для котангенса и, наконец, **сохранения Excel**, чтобы открыть его в пользовательском интерфейсе. К концу вы получите готовый к запуску фрагмент C#, который можно скопировать и вставить в любой .NET‑проект.

- **.NET 6+** (или .NET Framework 4.6+). Код работает на любой современной платформе.

Это всё. Никакой дополнительной конфигурации, без COM‑interop, без установленного Excel на сервере. Библиотека полностью обрабатывает формат файла в памяти.

## Шаг 1 – Создание книги Excel из C#

Первое, что нужно сделать, – **create excel workbook** программно. Представьте книгу как контейнер, в котором находятся все листы, стили и формулы.

```csharp
using Aspose.Cells;

public class CotangentDemo
{
    public static void Main()
    {
        // Initialize a new workbook – this is our blank Excel file
        Workbook workbook = new Workbook();

        // Grab the first (default) worksheet
        Worksheet worksheet = workbook.Worksheets[0];
```

> **Почему это важно:**  
> Создание книги в коде даёт полный контроль над расположением листов до того, как в них появятся данные. Это также избавляет от накладных расходов на открытие существующего файла только для добавления формулы.

## Шаг 2 – Использование функции EXPAND для построения матрицы (Как использовать Expand)

Функция Excel `EXPAND` удобна, когда нужно превратить одномерный массив в диапазон нескольких строк и столбцов. В нашем примере мы сформируем **матрицу 3 × 2** из простого списка `{1,2,3}`. Это демонстрирует **how to use expand** и показывает, что формулы могут возвращать массивы, а не только одиночные значения.

```csharp
        // Place the EXPAND formula in cell A1
        // =EXPAND({1,2,3},3,2) creates a 3‑row, 2‑column matrix
        worksheet.Cells["A1"].Formula = "=EXPAND({1,2,3},3,2)";
```

При открытии сохранённого файла ячейки A1:B3 будут содержать:

| A | B |
|---|---|
| 1 | 2 |
| 2 | 3 |
| 3 | 0 |

(Вторая колонка заполняется нулями, потому что исходный массив содержит только три элемента.)

> **Полезный совет:** Если нужна другая форма, просто измените второй и третий аргументы `EXPAND`. Функция автоматически дополняет недостающие ячейки нулями.

## Шаг 3 – Установка формулы COT (Как вычислить котангенс)

Теперь к главному: **how to calculate cotangent**. В Excel есть функция `COT`, которая принимает угол в радианах. В качестве простого примера используем `PI()/4` (45°); результат должен быть ровно `1`.

```csharp
        // Put the cotangent formula in cell B1
        // =COT(PI()/4) evaluates to 1 because cot(45°) = 1
        worksheet.Cells["B1"].Formula = "=COT(PI()/4)";
```

Вы можете заменить `PI()/4` любой ссылкой на другую ячейку, содержащую значение в радианах, или даже преобразованием из градусов в радианы, например `RADIANS(A2)`.

> **Почему использовать формулу, а не C#‑математику?**  
> Оставляя вычисление внутри Excel, вы получаете автоматическое обновление результата при изменении исходного угла. Кроме того, тяжёлые расчёты делегируются вычислительному движку Excel, который высоко оптимизирован.

## Шаг 4 – Сохранение книги (Как сохранить Excel)

Последний элемент головоломки – сохранить файл, чтобы его можно было открыть в Excel или передать дальше. Здесь и проявляется **how to save excel** на практике.

```csharp
        // Define the output path – adjust as needed
        string outputPath = @"C:\Temp\CotangentDemo.xlsx";

        // Save the workbook in XLSX format
        workbook.Save(outputPath);

        // Optional: let the user know we’re done
        System.Console.WriteLine($"Workbook saved to {outputPath}");
    }
}
```

> **Особый случай:** Если каталог не существует, `Save` бросит исключение. Оберните вызов в `try/catch` или заранее создайте папку.

Это полностью готовая к запуску программа. Скомпилируйте и запустите, затем откройте `CotangentDemo.xlsx`. Вы увидите расширенную матрицу в `A1:B3` и значение котангенса `1` в `B1`.

## Полный рабочий пример – все шаги вместе

Ниже приведён полный код, где все части соединены вместе. Скопируйте‑вставьте его в новый консольный проект и нажмите **F5**.

```csharp
using System;
using Aspose.Cells;

namespace ExcelCotangentDemo
{
    class Program
    {
        static void Main()
        {
            // Step 1 – create a new workbook and get its first worksheet
            Workbook workbook = new Workbook();
            Worksheet worksheet = workbook.Worksheets[0];

            // Step 2 – use EXPAND to generate a 3×2 matrix from a 1‑D array
            worksheet.Cells["A1"].Formula = "=EXPAND({1,2,3},3,2)";

            // Step 3 – set a COT formula that calculates cotangent of 45°
            worksheet.Cells["B1"].Formula = "=COT(PI()/4)";

            // Step 4 – save the workbook to view the results
            string outputPath = @"C:\Temp\CotangentDemo.xlsx";
            workbook.Save(outputPath);

            Console.WriteLine($"Workbook successfully saved at: {outputPath}");
        }
    }
}
```

### Ожидаемый результат при открытии файла

| A | B |
|---|---|
| 1 | 1 |
| 2 | 0 |
| 3 | 0 |

- **A1‑B3**: Матрица, созданная функцией `EXPAND`.  
- **B1**: Результат `COT(PI()/4)` – ровно **1**.

## Часто задаваемые вопросы (FAQ)

### 1. Можно ли вычислять котангенс для углов, хранящихся в других ячейках?
Конечно. Замените литерал `PI()/4` на ссылку, например `=COT(RADIANS(C2))`, где `C2` содержит угол в градусах.

### 2. Что делать, если нужен результат в градусах, а не в радианах?
Используйте `DEGREES(ATAN(1/yourValue))` для преобразования арктангенса обратно в градусы, либо просто оберните преобразование угла в `RADIANS`, как показано выше.

### 3. Автоматически ли Aspose.Cells вычисляет формулы?
Да. При **сохранении** книги библиотека по умолчанию рассчитывает все формулы. Если нужны значения в коде до сохранения, вызовите `workbook.CalculateFormula()`.

### 4. Чем это отличается от EPPlus или ClosedXML?
API схож: создаёте `Workbook`, получаете доступ к `Worksheets`, задаёте `Formula`. Основные различия – лицензирование и некоторые продвинутые возможности. Основные концепции (создание, установка формул, сохранение) остаются теми же.

### 5. Как записать результат обратно в C#?
После вызова `workbook.CalculateFormula()` можно прочитать свойство `Value` ячейки:

```csharp
double cotValue = worksheet.Cells["B1"].DoubleValue; // should be 1.0
```

## Советы и подводные камни, с которыми вы можете столкнуться

- **Нули в конце при EXPAND:** Если исходный массив короче запрашиваемого размера, Excel заполняет недостающие ячейки нулями. Это ожидаемое поведение, но учитывайте его, если полагаетесь на ненулевые значения.  
- **Локаль формул:** В некоторых установках Excel в качестве разделителя аргументов используется точка с запятой (`;`). Библиотека всегда ожидает запятые, так что региональные настройки вас не затронут.  
- **Разрешения файлов:** При работе под IIS или сервисным аккаунтом убедитесь, что процесс имеет право записи в целевую папку.  
- **Совместимость версий:** Функция `EXPAND` появилась в Excel 365/2021. Если нужна обратная совместимость, придётся имитировать её поведение с помощью вспомогательных столбцов.
