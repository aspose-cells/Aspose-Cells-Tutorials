## Руководство по внедрению

### Функция 1: Создание и заполнение DataTable

**Обзор:** В этом разделе демонстрируется создание `DataTable` под названием «OppLineItems» и заполнив его образцами данных.

#### Шаг 1: Создание DataTable

```csharp
// Определить исходный каталог
string SourceDir = @"YOUR_SOURCE_DIRECTORY";

// Создать новый объект DataTable
DataTable table = new DataTable("OppLineItems");

// Добавьте столбцы в DataTable
table.Columns.Add("PRODUCT_FAMILY");
table.Columns.Add("OPPORTUNITY_LINEITEM_PRODUCTNAME");
```

**Почему это важно:** Определение структуры данных позволяет Aspose.Cells правильно отображать их во время обработки интеллектуальных маркеров.

#### Шаг 2: Заполнение данными

```csharp
// Добавьте строки, представляющие позиции продуктовой линейки.
table.Rows.Add(new object[] { "MMM", "P1" });
table.Rows.Add(new object[] { "MMM", "P2" });
table.Rows.Add(new object[] { "DDD", "P1" });
table.Rows.Add(new object[] { "DDD", "P2" });
table.Rows.Add(new object[] { "AAA", "P1" });
```

**Объяснение:** Каждая строка здесь соответствует позиции линейки продуктов, что упрощает сопоставление данных.

### Функция 2: Загрузка и обработка рабочей книги с помощью интеллектуальных маркеров

**Обзор:** Загрузите файл Excel в Aspose.Cells, настройте интеллектуальные маркеры и обработайте книгу с помощью `WorkbookDesigner`.

#### Шаг 1: Загрузите свою рабочую книгу

```csharp
string outputDir = @"YOUR_OUTPUT_DIRECTORY";
Workbook workbook = new Workbook(SourceDir + "sampleGetSmartMarkerNotifications.xlsx");
```

**Почему это важно:** Загрузка рабочей книги инициализирует ваш шаблон дизайна для интеграции данных.

#### Шаг 2: Настройка WorkbookDesigner

```csharp
// Инициализация объекта WorkbookDesigner
WorkbookDesigner designer = new WorkbookDesigner(workbook);

// Назначить DataTable в качестве источника данных
designer.SetDataSource(table);
```

**Объяснение:** The `WorkbookDesigner` устраняет разрыв между вашими данными и шаблоном Excel, обеспечивая динамическую интеграцию контента.

#### Шаг 3: Обработка интеллектуальных маркеров

```csharp
// Реализовать логику обработки обратного вызова
designer.CallBack = new SmartMarkerCallBack(workbook);

// Обрабатывать интеллектуальные маркеры без регистрации
designer.Process(false);
```

**Почему это важно:** Настройка функции обратного вызова обеспечивает индивидуальную обработку, повышая гибкость и контроль над заполнением данных.

### Функция 3: Обработка обратного вызова интеллектуального маркера

**Обзор:** Реализуйте механизм пользовательской логики для динамической обработки событий интеллектуальных маркеров.

#### Шаг 1: Определите класс обратного вызова

```csharp
class SmartMarkerCallBack : ISmartMarkerCallBack
{
    Workbook workbook;

    public SmartMarkerCallBack(Workbook workbook)
    {
        this.workbook = workbook;
    }

    public void Process(int sheetIndex, int rowIndex, int colIndex, String tableName, String columnName)
    {
        Console.WriteLine($"Processing Cell: {workbook.Worksheets[sheetIndex].Name}!{CellsHelper.CellIndexToName(rowIndex, colIndex)}");
        Console.WriteLine($"Processing Marker: {tableName}.{columnName}");
    }
}
```

**Объяснение:** Этот обратный вызов обеспечивает подключение к циклу обработки маркеров, позволяя выполнять пользовательскую логику на каждом этапе.
