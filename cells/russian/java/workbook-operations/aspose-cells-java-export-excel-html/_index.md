## Руководство по внедрению

Теперь давайте перейдем к реализации `FilePathProvider` класс для управления экспортом HTML.

### Реализация FilePathProvider

Цель этой функции — предоставить удобный способ экспортировать каждый рабочий лист в виде отдельного HTML-файла, сохраняя при этом их взаимосвязи. 

#### Шаг 1: Создание класса FilePathProvider

```java
package AsposeCellsExamples.TechnicalArticles;

import AsposeCellsExamples.Utils;

public class FilePathProvider {
    // Возвращает полный путь к файлу по имени рабочего листа при экспорте в HTML.
    public String getFullName(String sheetName) {
        String dataDir = Utils.getSharedDataDir(FilePathProvider.class) + "TechnicalArticles/";
        
        if ("Sheet2".equals(sheetName)) {
            return dataDir + "Sheet2.html";
        } else if ("Sheet3".equals(sheetName)) {
            return dataDir + "Sheet3.html";
        }
        
        return "";
    }
}
```

#### Объяснение
- **Utils.getSharedDataDir:** Извлекает путь к каталогу для хранения экспортированных HTML-файлов.
- **Имя листа:** Определяет, какой рабочий лист экспортировать, корректно сохраняя ссылки.

### Советы по устранению неполадок

- Убедитесь, что все названия листов указаны правильно. `getFullName` метод.
- Убедитесь, что `dataDir` путь существует и доступен для записи.
