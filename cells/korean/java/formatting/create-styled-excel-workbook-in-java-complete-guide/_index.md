# Java에서 스타일이 적용된 Excel 워크북 만들기 – 완전 가이드

Java에서 **create styled excel workbook**을(를) 만들어야 했지만 서식 세부 사항에 얽혀 힘들었던 적이 있나요? 당신만 그런 것이 아닙니다. 데이터베이스에서 데이터를 가져오든 보고서를 내보내든, 통화 기호, 날짜 형식, 굵은 헤더와 같은 올바른 모양을 맞추는 것이 번거롭게 느껴질 수 있습니다.  

이번 튜토리얼에서는 **adds custom formatting to excel**을(를) 수행하고, **set date format excel column**을(를) 설정하며, 마지막으로 **save workbook as xlsx**를 땀 흘리지 않고 수행하는 실용적인 솔루션을 단계별로 살펴보겠습니다. 바로 확인해 보시겠어요?

## 배울 내용

- 처음부터 Aspose.Cells `Workbook`을(를) 생성하는 방법.  
- General, Currency, Date 열을 모두 포함하는 스타일 배열을 적용하는 방법.  
- 열 이름이 포함된 **creates styled excel workbook**을(를) 정확히 호출하는 방법.  
- **save workbook as xlsx** 메서드를 사용하여 파일을 저장하는 방법.  

외부 설정 파일도 없고, 복잡한 XML 트릭도 없습니다—그냥 복사‑붙여넣기하고 실행하고 적용할 수 있는 순수 Java 코드만 있습니다.

### 사전 요구 사항

- Java 8+ (코드는 최신 JDK에서 모두 동작합니다).  
- Aspose.Cells for Java 라이브러리 (Maven Central에서 최신 JAR를 받을 수 있습니다).  
- 컬렉션(`List`, `Map`)에 대한 약간의 이해.  

위 항목 중 익숙하지 않은 것이 있다면 잠시 멈춰 Aspose.Cells JAR를 설치하세요; 나머지는 자연스럽게 진행될 것입니다.

## 스타일이 적용된 Excel 워크북 만들기 – 프로젝트 설정

먼저, 새로운 Maven(또는 Gradle) 프로젝트를 만들고 Aspose.Cells를 의존성으로 추가합니다.

```xml
<!-- pom.xml snippet -->
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-cells</artifactId>
    <version>23.12</version> <!-- use the latest version -->
</dependency>
```

> **Pro tip:** Gradle를 사용하는 경우, 동일한 좌표를 `implementation 'com.aspose:aspose-cells:23.12'`와 함께 사용할 수 있습니다.

왜 Maven 설정부터 시작하나요? Maven은 모든 전이적 의존성을 자동으로 가져오므로 나중에 올바른 JAR를 찾는 데 시간을 낭비하지 않아도 됩니다.

## Excel 열에 사용자 지정 서식 추가

프로젝트가 준비되었으니, 원하는 스타일을 정의해 보겠습니다. Aspose.Cells에서는 각 열에 직접 매핑되는 형식 문자열 배열을 전달할 수 있습니다.

```java
// Define column styles: General, Currency, Date
String[] columnStyles = {
    "General",          // Column A – plain text or numbers
    "$#,##0.00",        // Column B – currency with two decimals
    "mm/dd/yyyy"        // Column C – US‑style date
};
```

**add custom formatting to excel** 단계는 이 문자열 배열을 만드는 것만큼 간단합니다.  
각 항목은 Excel의 “Custom” 서식 대화 상자에서 사용하는 패턴과 동일하므로 새로운 구문을 배울 필요가 없습니다.

## Excel 열에 날짜 형식 설정

데이터에 날짜가 포함되어 있다면 일관되게 표시되도록 해야 합니다. `columnStyles`의 세 번째 항목(`"mm/dd/yyyy"`)이 바로 그 역할을 합니다—Aspose.Cells에 세 번째 열을 날짜로 처리하고 월/일/연도 순서로 표시하도록 지시합니다.

```java
// Example: Adding a row with a Date object
Map<String, Object> row = new HashMap<>();
row.put("Product", "Widget");
row.put("Price", 19.99);
row.put("ReleaseDate", new java.util.Date()); // Today’s date
dataRows.add(row);
```

워크북이 생성될 때, **set date format excel column** 지시문은 모든 `ReleaseDate` 셀을 `04/27/2026`(또는 오늘 날짜)와 같이 표시하도록 보장합니다. 별도의 후처리가 필요 없습니다.

## 데이터 가져오기 및 스타일 적용

스타일을 정의했으니, 다음 단계는 데이터를 워크시트에 입력하는 것입니다. Aspose.Cells는 `importCustomObjects`를 제공하는데, 이 메서드는 데이터를 삽입할 뿐만 아니라 방금 만든 스타일 배열도 적용합니다.

```java
// Step 1: Simulate fetching rows from a DB
List<Map<String, Object>> dataRows = fetchRows(); // see helper below

// Step 2: Create a fresh workbook and grab the first sheet
Workbook workbook = new Workbook();
Worksheet worksheet = workbook.getWorksheets().get(0);

// Step 3: Import data with custom styles; true = first row has headers
worksheet.getCells().importCustomObjects(
        dataRows,        // data source
        0,               // start row (0‑based)
        0,               // start column (0‑based)
        columnStyles,    // style array from above
        true);           // first row contains column names
```

### `importCustomObjects`를 사용하는 이유

- **Single‑call magic:** 데이터를 *그리고* 서식을 한 번에 작성합니다.  
- **Header awareness:** `true`를 전달하면 API가 첫 번째 Map의 키를 열 헤더로 간주하도록 지시합니다. 이는 읽기 쉬운 열 제목이 필요한 **create styled excel workbook** 시나리오에 적합합니다.  
- **Performance:** 내부적으로 Aspose가 쓰기 작업을 배치 처리하여 셀별로 반복하는 것보다 더 빠릅니다.

> **Watch out:** 각 `Map`의 키 순서는 중요합니다. 결정적인 열 순서가 필요하면 `LinkedHashMap`을 사용하세요.

## 워크북을 XLSX 파일로 저장

파일을 저장하지 못한다면 지금까지의 모든 작업이 무의미합니다. 마지막 줄은 **save workbook as xlsx**를 수행하는 한 줄 코드입니다.

```java
// Step 5: Persist the workbook
String outputPath = "output/DataTableStyled.xlsx";
workbook.save(outputPath);
System.out.println("Workbook saved to " + outputPath);
```

`save` 메서드는 파일 확장자를 자동으로 감지합니다. `.xlsx`를 지정하면 최신 Office Open XML 형식을 보장하게 되며, 이는 대부분의 하위 도구가 기대하는 형식입니다.

## 전체 실행 가능한 예제

아래는 IDE에 바로 복사해 넣을 수 있는 전체 클래스입니다. 샘플 데이터를 반환하는 모의 `fetchRows` 메서드가 포함되어 있어 실제 데이터베이스가 없어도 테스트할 수 있습니다.

```java
import com.aspose.cells.*;
import java.util.*;

public class ImportDataTableWithStyle {
    public static void main(String[] args) throws Exception {
        // ---------- Step 1: Get data ----------
        List<Map<String, Object>> dataRows = fetchRows();

        // ---------- Step 2: Create workbook ----------
        Workbook workbook = new Workbook();
        Worksheet worksheet = workbook.getWorksheets().get(0);

        // ---------- Step 3: Define column styles ----------
        String[] columnStyles = { "General", "$#,##0.00", "mm/dd/yyyy" };

        // ---------- Step 4: Import data with styles ----------
        // true = first row contains column names
        worksheet.getCells().importCustomObjects(dataRows, 0, 0, columnStyles, true);

        // ---------- Step 5: Save as XLSX ----------
        String outPath = "output/DataTableStyled.xlsx";
        workbook.save(outPath);
        System.out.println("Workbook saved to " + outPath);
    }

    // Mock method – replace with real DB call if needed
    private static List<Map<String, Object>> fetchRows() {
        List<Map<String, Object>> rows = new ArrayList<>();

        Map<String, Object> row1 = new LinkedHashMap<>();
        row1.put("Product", "Gadget");
        row1.put("Price", 49.95);
        row1.put("ReleaseDate", new Date()); // today
        rows.add(row1);

        Map<String, Object> row2 = new LinkedHashMap<>();
        row2.put("Product", "Widget");
        row2.put("Price", 19.99);
        Calendar cal = Calendar.getInstance();
        cal.set(2025, Calendar.DECEMBER, 15);
        row2.put("ReleaseDate", cal.getTime());
        rows.add(row2);

        return rows;
    }
}
```

실행하고 생성된 파일을 열면 기대했던 **create styled excel workbook** 결과를 확인할 수 있습니다.

## 일반적인 질문 및 엣지 케이스

**세 개 이상의 열이 필요하면 어떻게 하나요?**  
`columnStyles` 배열을 확장하면 됩니다—각 항목이 순서대로 열에 매핑됩니다. 서식을 적용하지 않을 열은 `"General"`을 사용하세요.

**글꼴이나 색상을 적용할 수 있나요?**  
네. 문자열 형식 대신 `Style` 객체를 만들고 이를 `StyleFlag`에 할당한 뒤 `Style[]` 배열을 `importCustomObjects`에 전달하면 됩니다. 좀 더 깊은 내용이지만 원리는 동일합니다.
