# Πώς να αντιγράψετε περιοχή με συγκεντρωτικούς πίνακες σε C# – Πλήρης Οδηγός

Έχετε αναρωτηθεί ποτέ **πώς να αντιγράψετε μια περιοχή** που περιέχει έναν συγκεντρωτικό πίνακα χωρίς να σπάσει η σύνδεση με τα δεδομένα προέλευσης; Δεν είστε ο μόνος. Σε πολλά πραγματικά έργα έχω αντιμετωπίσει αυτό το ακριβές πρόβλημα — τα αρχεία Excel έρχονται με πολύπλοκους συγκεντρωτικούς πίνακες, και η απαίτηση είναι να τους μετακινήσετε ή να αντιγράψετε τα δεδομένα αλλού.  

Τα καλά νέα; Η λύση είναι αρκετά απλή μόλις ξέρετε **πώς να φορτώσετε ένα βιβλίο εργασίας**, να κάνετε ένα αντίγραφο, και στη συνέχεια **πώς να αποθηκεύσετε το βιβλίο εργασίας** ξανά. Σε αυτό το tutorial θα περάσουμε από όλη τη διαδικασία, συμπεριλαμβανομένου του **αντιγραφής συγκεντρωτικών πινάκων**, και ακόμη μια γρήγορη συμβουλή για **μετακίνηση συγκεντρωτικού πίνακα** αν τον χρειάζεστε αλλού στο ίδιο φύλλο.

Στο τέλος αυτού του οδηγού θα έχετε ένα πλήρως λειτουργικό απόσπασμα C# που:

1. Φορτώνει ένα υπάρχον αρχείο Excel.  
2. Αντιγράφει μια περιοχή (συμπεριλαμβανομένου του συγκεντρωτικού πίνακα) σε νέα θέση.  
3. Αποθηκεύει το τροποποιημένο βιβλίο εργασίας σε νέο αρχείο.

Χωρίς εξωτερικά scripts, χωρίς χειροκίνητη παρέμβαση — μόνο καθαρός, επαναχρησιμοποιήσιμος κώδικας.

---

## Προαπαιτούμενα

- **.NET 6+** (οποιαδήποτε πρόσφατη έκδοση λειτουργεί).  
- **Aspose.Cells for .NET** – η βιβλιοθήκη που παρέχει `Workbook`, `WorksheetCopyOptions`, κ.λπ. Μπορείτε να την εγκαταστήσετε μέσω NuGet:

```bash
dotnet add package Aspose.Cells
```

- Ένα αρχείο εισόδου (`input.xlsx`) που ήδη περιέχει έναν συγκεντρωτικό πίνακα στην περιοχή `A1:G20`.  
- Βασική εξοικείωση με C# και Visual Studio (ή το αγαπημένο σας IDE).

> **Pro tip:** Αν χρησιμοποιείτε διαφορετική βιβλιοθήκη Excel (π.χ., EPPlus), οι έννοιες είναι οι ίδιες — απλώς αντικαταστήστε τις κλήσεις API.

## Βήμα 1 – Πώς να φορτώσετε το βιβλίο εργασίας (Πρωταρχική Ρύθμιση)

Πριν μπορέσουμε να αντιγράψουμε οτιδήποτε, πρέπει να φορτώσουμε το αρχείο Excel στη μνήμη.

```csharp
using Aspose.Cells;

// Step 1: Load the source workbook
var sourceWorkbook = new Workbook(@"YOUR_DIRECTORY\input.xlsx");

// Grab the first worksheet – this is where our pivot lives
var sourceWorksheet = sourceWorkbook.Worksheets[0];
```

**Γιατί είναι σημαντικό:**  
Η φόρτωση του βιβλίου εργασίας σας παρέχει ένα μοντέλο αντικειμένων που μπορείτε να χειριστείτε. Χωρίς το `how to load workbook` σωστά, οποιαδήποτε επόμενη λειτουργία αντιγραφής θα προκαλέσει εξαίρεση *FileNotFound* ή *InvalidOperation*.

> **Προσοχή:** Αν το αρχείο είναι μεγάλο, σκεφτείτε να χρησιμοποιήσετε `LoadOptions` με `MemorySetting` για να ελέγξετε τη χρήση μνήμης.

## Βήμα 2 – Πώς να αντιγράψετε περιοχή (συμπεριλαμβανομένου του συγκεντρωτικού πίνακα)

Τώρα έρχεται το αστέρι της παράστασης: η αντιγραφή μιας περιοχής που περιέχει έναν συγκεντρωτικό πίνακα. Η μέθοδος `CopyRange`, σε συνδυασμό με `WorksheetCopyOptions`, κάνει τη βαριά δουλειά.

```csharp
// Step 2: Copy a range that includes a pivot table to a new location
sourceWorksheet.CopyRange(
    "A1:G20",                                   // Source range
    new WorksheetCopyOptions { CopyPivotTables = true }, // Ensure pivot tables travel with the data
    sourceWorksheet,                           // Destination worksheet (same sheet in this case)
    "A25");                                     // Upper‑left corner of the destination
```

**Γιατί ορίζουμε `CopyPivotTables = true`:**  
Από προεπιλογή, η αντιγραφή μιας περιοχής μετακινεί μόνο τα ακατέργαστα κελιά. Η κρυφή μνήμη του συγκεντρωτικού παραμένει πίσω, και ο αντιγραμμένος συγκεντρωτικός πίνακας γίνεται στατικός πίνακας. Ορίζοντας `CopyPivotTables` διατηρεί τη ζωντανή σύνδεση, ώστε ο διπλότυπος συγκεντρωτικός πίνακας να ενημερώνεται όταν τα δεδομένα προέλευσης αλλάζουν.

**Περίπτωση άκρης:** Αν η περιοχή προορισμού επικαλύπτεται με την πηγή, το Aspose.Cells θα πετάξει `ArgumentException`. Πάντα επιλέξτε έναν μη επικαλυπτόμενο στόχο ή δημιουργήστε πρώτα ένα νέο φύλλο εργασίας.

## Βήμα 3 – Πώς να αποθηκεύσετε το βιβλίο εργασίας (Διατήρηση των αλλαγών)

Μετά την αντιγραφή, θα θέλετε να γράψετε τις αλλαγές πίσω στο δίσκο. Εδώ έρχεται το **πώς να αποθηκεύσετε το βιβλίο εργασίας**.

```csharp
// Step 3: Save the modified workbook to a new file
sourceWorkbook.Save(@"YOUR_DIRECTORY\output.xlsx");
```

**Τι συμβαίνει στο παρασκήνιο:**  
`Save` σειριοποιεί το βιβλίο εργασίας στη μνήμη, συμπεριλαμβανομένου του νεοαντιγραμμένου συγκεντρωτικού πίνακα, σε ένα τυπικό πακέτο `.xlsx`. Αν χρειάζεστε διαφορετική μορφή (CSV, PDF, κλπ.), απλώς αλλάξτε την επέκταση αρχείου ή χρησιμοποιήστε την υπερφόρτωση που δέχεται `SaveFormat`.

> **Συμβουλή:** Χρησιμοποιήστε `Workbook.Save(string, SaveOptions)` αν χρειάζεται να προστατέψετε το αρχείο με κωδικό πρόσβασης ή να ορίσετε άλλες επιλογές εξαγωγής.

## Πλήρες Παράδειγμα Λειτουργίας

Συνδυάζοντας όλα, εδώ είναι το πλήρες, έτοιμο προς εκτέλεση πρόγραμμα:

```csharp
using System;
using Aspose.Cells;

class Program
{
    static void Main()
    {
        // 1️⃣ How to load workbook
        var sourceWorkbook = new Workbook(@"YOUR_DIRECTORY\input.xlsx");
        var sourceWorksheet = sourceWorkbook.Worksheets[0];

        // 2️⃣ How to copy range (including pivot tables)
        sourceWorksheet.CopyRange(
            "A1:G20",
            new WorksheetCopyOptions { CopyPivotTables = true },
            sourceWorksheet,
            "A25");

        // 3️⃣ How to save workbook
        sourceWorkbook.Save(@"YOUR_DIRECTORY\output.xlsx");

        Console.WriteLine("✅ Range copied and workbook saved successfully!");
    }
}
```

**Αναμενόμενο αποτέλεσμα:**  
Ανοίξτε το `output.xlsx`. Θα δείτε τον αρχικό συγκεντρωτικό πίνακα ακόμα στην περιοχή `A1:G20`, και ένα πανομοιότυπο, πλήρως λειτουργικό αντίγραφο που αρχίζει στο `A25`. Και οι δύο συγκεντρωτικοί δείχνουν στα ίδια δεδομένα προέλευσης, έτσι η ενημέρωση του ενός ενημερώνει και το άλλο.

## Συχνές Ερωτήσεις & Παραλλαγές

### Μπορώ να **μετακινήσω συγκεντρωτικό πίνακα** αντί να τον αντιγράψω;

Απόλυτα. Μετά την αντιγραφή, απλώς καθαρίστε την αρχική περιοχή (ή χρησιμοποιήστε `sourceWorksheet.Cells.ClearRange(0, 0, 19, 6)`) και, αν χρειάζεται, μετονομάστε την περιοχή προορισμού. Αυτό ουσιαστικά “μετακινεί” τον συγκεντρωτικό πίνακα.

### Τι γίνεται αν ο συγκεντρωτικός πίνακας χρησιμοποιεί εξωτερική πηγή δεδομένων;

`CopyPivotTables = true` αντιγράφει μόνο τον ορισμό του συγκεντρωτικού, όχι τη σύνδεση με την εξωτερική πηγή. Βεβαιωθείτε ότι το βιβλίο εργασίας προορισμού έχει πρόσβαση στην ίδια πηγή δεδομένων, ή δημιουργήστε ξανά τη σύνδεση μετά την αντιγραφή.

### Πώς να αντιγράψω σε **διαφορετικό φύλλο εργασίας**;

Απλώς περάστε το αντικείμενο του φύλλου προορισμού αντί για `sourceWorksheet`:

```csharp
var destWorksheet = sourceWorkbook.Worksheets.Add("CopiedPivot");
sourceWorksheet.CopyRange("A1:G20", new WorksheetCopyOptions { CopyPivotTables = true }, destWorksheet, "A1");
```

### Υπάρχει τρόπος να αντιγράψετε **πολλαπλές περιοχές** ταυτόχρονα;

Μπορείτε να καλέσετε το `CopyRange` επανειλημμένα ή να χρησιμοποιήσετε `CopyRows`/`CopyColumns` για μεγαλύτερα τμήματα. Η επανάληψη πάνω σε μια λίστα διευθύνσεων είναι μια καθαρή προσέγγιση.

## Συνηθισμένα Πιθανά Σφάλματα & Pro Συμβουλές

- **Μέγεθος cache συγκεντρωτικού:** Μεγάλες κρυφές μνήμες συγκεντρωτικών μπορούν να αυξήσουν το μέγεθος του βιβλίου εργασίας. Αν χρειάζεστε μόνο τα εμφανιζόμενα δεδομένα, σκεφτείτε `CopyPivotTables = false` και στη συνέχεια χρησιμοποιήστε `PivotTable.RefreshData()` στον προορισμό.  
- **Διαδρομές αρχείων:** Χρησιμοποιήστε `Path.Combine` για να αποφύγετε σκληρά κωδικοποιημένους διαχωριστές, ειδικά σε .NET πολλαπλών πλατφορμών.  
- **Απόδοση:** Για τεράστια βιβλία εργασίας, τυλίξτε την αντιγραφή σε `using (var stream = new MemoryStream())` και αποθηκεύστε πρώτα στο ρεύμα, μετά γράψτε στο δίσκο. Αυτό μειώνει το φόρτο I/O.
