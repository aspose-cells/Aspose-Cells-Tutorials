# Πώς να Αντικαταστήσετε Μεταβλητές σε JSON με το SmartMarker – Πλήρης Οδηγός

Έχετε αναρωτηθεί ποτέ **πώς να αντικαταστήσετε μεταβλητές** μέσα σε ένα JSON payload χωρίς να γράψετε έναν προσαρμοσμένο parser; Δεν είστε μόνοι. Σε πολλές περιπτώσεις ενσωμάτωσης—σκεφτείτε τιμολόγια, μηχανές τιμολόγησης ή δυναμικά αρχεία ρυθμίσεων—πρέπει να ενσωματώσετε τιμές χρόνου εκτέλεσης, να εφαρμόσετε απλούς όρους και ίσως ακόμη και να κάνετε μια γρήγορη πολλαπλασιαστική πράξη. Αυτό το tutorial σας δείχνει ακριβώς **πώς να αντικαταστήσετε μεταβλητές** χρησιμοποιώντας τη βιβλιοθήκη SmartMarker, διατηρώντας το JSON καθαρό και ευανάγνωστο.

Θα περάσουμε από ένα πραγματικό παράδειγμα που καλύπτει **χρήση if expression**, **πώς να εφαρμόσετε conditional**, **πώς να πολλαπλασιάσετε τιμές**, και **πώς να δημιουργήσετε json** επί τόπου. Στο τέλος, θα έχετε ένα έτοιμο προς εκτέλεση απόσπασμα C# που μπορείτε να ενσωματώσετε σε οποιοδήποτε έργο .NET.

## Τι Θα Μάθετε

- Διαμορφώστε το `SmartMarkerOptions` για αποθήκευση επαναχρησιμοποιήσιμων μεταβλητών.  
- Γράψτε ένα πρότυπο JSON που περιέχει μια `if` έκφραση για λογική υπό όρους.  
- Πολλαπλασιάστε μια τιμή με μια μεταβλητή μέσα στο πρότυπο.  
- Επεξεργαστείτε το πρότυπο με το `SmartMarkerProcessor` και λάβετε το τελικό JSON string.  
- Αντιμετωπίστε κοινά προβλήματα όπως ελλιπείς μεταβλητές ή κακοδιατυπωμένες εκφράσεις.  

Χωρίς εξωτερικές υπηρεσίες, χωρίς βαριές εξαρτήσεις—μόνο απλό C# και το πακέτο NuGet SmartMarker.

## Βήμα 1: Εγκατάσταση και Εισαγωγή του SmartMarker

Πριν ξεκινήσετε, βεβαιωθείτε ότι το πακέτο SmartMarker αναφέρεται στο έργο σας. Εάν χρησιμοποιείτε το .NET CLI, εκτελέστε:

```bash
dotnet add package SmartMarker
```

Στη συνέχεια, προσθέστε τις απαραίτητες οδηγίες `using` στην κορυφή του αρχείου C#:

```csharp
using SmartMarker;
using SmartMarker.Models;
using System;
```

> **Συμβουλή:** Η πιο πρόσφατη έκδοση (από τον Μάρτιο 2026) είναι η 2.4.1. Υποστηρίζει .NET 6 και νεότερα, αλλά λειτουργεί εξίσου καλά με το .NET Framework 4.7.

---

## Βήμα 2: Δημιουργία SmartMarker Options και Ορισμός Μεταβλητών

Τώρα θα δημιουργήσουμε μια παρουσία του `SmartMarkerOptions` που θα κρατάει όλες τις μεταβλητές που θέλουμε να επαναχρησιμοποιήσουμε στο πρότυπο. Εδώ απαντάμε στο ερώτημα **πώς να αντικαταστήσετε μεταβλητές**—οι μεταβλητές λειτουργούν ως placeholders που το SmartMarker θα αντικαταστήσει αργότερα.

```csharp
// Step 2: Create SmartMarker options to hold variables used in the template
var smartMarkerOptions = new SmartMarkerOptions();

// Define a variable (Rate) that we’ll reference later in the JSON expression
smartMarkerOptions.Variables["Rate"] = 0.08; // 8% commission rate
```

Γιατί να αποθηκεύσουμε το ποσοστό στα `Variables` αντί να το κωδικοποιήσουμε σκληρά; Επειδή μπορεί να αντλήσετε αυτόν τον αριθμό από μια βάση δεδομένων, ένα αρχείο ρυθμίσεων ή μια είσοδο χρήστη. Η διατήρηση του στις επιλογές κάνει το πρότυπο επαναχρησιμοποιήσιμο και δοκιμαστέο.

---

## Βήμα 3: Γράψτε το Πρότυπο JSON με μια `if` Έκφραση

Εδώ είναι που το κλειδί **use if expression** λάμπει. Το SmartMarker σας επιτρέπει να ενσωματώσετε λογική υπό όρους απευθείας μέσα στο JSON string. Η σύνταξη μοιάζει λίγο με όνομα ιδιότητας, αλλά το SmartMarker το αντιμετωπίζει ως οδηγία.

```csharp
// Step 3: Prepare the JSON data with a conditional field that uses the variable
string jsonTemplate = @"{
    ""Amount"": 1000,
    ""if(Amount>500)"": ""${Amount * Rate}""
}";
```

Παρατηρήστε το κλειδί `if(Amount>500)`. Το SmartMarker αξιολογεί την έκφραση `Amount>500`; αν είναι αληθής, η αντίστοιχη τιμή (`${Amount * Rate}`) εισάγεται στο αποτέλεσμα. Η σύνταξη `${...}` είναι η *μηχανή αντικατάστασης μεταβλητών*—εδώ **πώς να πολλαπλασιάσετε τιμές** (`Amount * Rate`) πριν ενσωματωθεί το αποτέλεσμα.

---

## Βήμα 4: Επεξεργασία του Προτύπου και Λήψη του Τελικού JSON

Με τις επιλογές και το πρότυπο έτοιμα, παραδίδουμε τα πάντα στον επεξεργαστή. Η μέθοδος `ProcessJson` αναλύει το πρότυπο, εφαρμόζει τον όρο, εκτελεί τον πολλαπλασιασμό και επιστρέφει ένα καθαρό JSON string.

```csharp
// Step 4: Process the JSON with SmartMarker, applying the variable substitution
string resultJson = ws.SmartMarkerProcessor.ProcessJson(jsonTemplate, smartMarkerOptions);
Console.WriteLine(resultJson);
```

Η εκτέλεση του αποσπάσματος εκτυπώνει:

```json
{
  "Amount": 1000,
  "Result": "80"
}
```

**Τι συνέβη;**  
- `Amount` είναι 1000, το οποίο ικανοποιεί το `Amount>500`.  
- Το SmartMarker αξιολογεί `${Amount * Rate}` → `1000 * 0.08 = 80`.  
- Το αρχικό κλειδί υπό όρους (`if(Amount>500)`) αντικαθίσταται από ένα καθαρό όνομα ιδιότητας (`Result`). Από προεπιλογή το SmartMarker χρησιμοποιεί το `"Result"` αλλά μπορείτε να το προσαρμόσετε (περισσότερα αργότερα).

Αν αλλάξετε το `Amount` σε `400`, η έξοδος γίνεται:

```json
{
  "Amount": 400
}
```

Το μπλοκ υπό όρους εξαφανίζεται επειδή η έκφραση αξιολογήθηκε ως `false`. Αυτή είναι η ουσία του **πώς να εφαρμόσετε conditional** λογικής σε JSON.

---

## Βήμα 5: Προσαρμογή του Ονόματος Ιδιότητας Εξόδου (Προαιρετικό)

Μερικές φορές δεν θέλετε το γενικό κλειδί `"Result"`. Το SmartMarker σας επιτρέπει να καθορίσετε ένα προσαρμοσμένο όνομα χρησιμοποιώντας την επιλογή `RenameIfExpression`:

```csharp
smartMarkerOptions.RenameIfExpression = "Discount";
string customResult = ws.SmartMarkerProcessor.ProcessJson(jsonTemplate, smartMarkerOptions);
Console.WriteLine(customResult);
```

Έξοδος:

```json
{
  "Amount": 1000,
  "Discount": "80"
}
```

Τώρα η υπό όρους τιμή αποθηκεύεται κάτω από ένα πιο σημασιολογικό όνομα ιδιότητας—τέλειο για downstream υπηρεσίες που αναμένουν ένα συγκεκριμένο πεδίο.

## Συχνά Προβλήματα και Πώς να τα Αποφύγετε

| Πρόβλημα | Γιατί Συμβαίνει | Διόρθωση |
|----------|----------------|----------|
| Η μεταβλητή δεν βρέθηκε | Αναφέρατε μια μεταβλητή που δεν υπάρχει στα `smartMarkerOptions.Variables`. | Ελέγξτε ξανά την ορθογραφία και βεβαιωθείτε ότι η μεταβλητή έχει προστεθεί πριν την επεξεργασία. |
| Μη έγκυρη σύνταξη `if` | Λείπουν παρενθέσεις ή υπάρχει λανθασμένος τελεστής (`>`, `<`, `==`). | Ακολουθήστε ακριβώς το μοτίβο `if(<expression>)`; το SmartMarker υποστηρίζει μόνο απλές αριθμητικές συγκρίσεις. |
| Το JSON γίνεται κατεστραμμένο | Απομένει τυχαία μια τελική κόμμα μετά το μπλοκ υπό όρους. | Αφήστε το SmartMarker να διαχειριστεί την αφαίρεση· διατηρήστε το αρχικό πρότυπο συντακτικά σωστό. |
| Απρόσμενη μορφή αριθμού | Το αποτέλεσμα εμφανίζεται ως συμβολοσειρά `"80"` αντί για αριθμό. | Κάντε cast ή parse αργότερα, ή χρησιμοποιήστε `${(Amount * Rate):N0}` για αριθμητική μορφοποίηση. |

## Πλήρες Παράδειγμα Εργασίας (Έτοιμο για Αντιγραφή‑Επικόλληση)

Παρακάτω είναι το πλήρες πρόγραμμα που μπορείτε να μεταγλωττίσετε και να εκτελέσετε. Δείχνει **πώς να δημιουργήσετε json** με δυναμικές μεταβλητές, όρους και αριθμητικές πράξεις—όλα σε λιγότερο από 30 γραμμές.

```csharp
using System;
using SmartMarker;
using SmartMarker.Models;

class Program
{
    static void Main()
    {
        // 1️⃣ Create SmartMarker options and define a reusable variable
        var smartMarkerOptions = new SmartMarkerOptions();
        smartMarkerOptions.Variables["Rate"] = 0.08; // 8% commission
        smartMarkerOptions.RenameIfExpression = "Discount"; // optional custom name

        // 2️⃣ JSON template with an if expression and multiplication
        string jsonTemplate = @"{
            ""Amount"": 1000,
            ""if(Amount>500)"": ""${Amount * Rate}""
        }";

        // 3️⃣ Process the template
        string output = ws.SmartMarkerProcessor.ProcessJson(jsonTemplate, smartMarkerOptions);

        // 4️⃣ Show the result
        Console.WriteLine("Generated JSON:");
        Console.WriteLine(output);
    }
}
```

**Αναμενόμενη έξοδος κονσόλας**

```
Generated JSON:
{
  "Amount": 1000,
  "Discount": "80"
}
```

Μη διστάσετε να αλλάξετε το `Amount` για να δοκιμάσετε το κλαδί υπό όρους, ή να προσαρμόσετε το `Rate` για να δείτε διαφορετικούς υπολογισμούς έκπτωσης.

## Επέκταση του Σχεδίου – Περισσότερα Σενάρια “Πώς να”

- **How to substitute variables** από αρχείο ρυθμίσεων: Φορτώστε ένα `Dictionary<string, object>` από το `appsettings.json` και περάστε το στο `smartMarkerOptions.Variables`.  
- **How to use if expression** για πολλαπλές συνθήκες: Συνδέστε τα όπως `"if(Amount>500 && CustomerType=='VIP')"`—το SmartMarker υποστηρίζει λογικό AND/OR.  
- **How to apply conditional** μορφοποίηση: Χρησιμοποιήστε `${Amount:0.00}` μέσα στην έκφραση για να ελέγξετε τα δεκαδικά ψηφία.  
- **How to multiply values** με πιο σύνθετη μαθηματική πράξη: `${(Amount - Discount) * TaxRate}` λειτουργεί με τον ίδιο τρόπο.  
- **How to generate json** για ένθετα αντικείμενα: Τοποθετήστε το μπλοκ υπό όρους μέσα σε άλλο αντικείμενο JSON, και το SmartMarker θα διατηρήσει την ιεραρχία.
