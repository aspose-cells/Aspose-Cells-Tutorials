# Επανάληψη δεδομένων στο Excel – Συμπλήρωση προτύπου με SmartMarker

Έχετε ποτέ χρειαστεί να **επαναλάβετε δεδομένα στο Excel** αλλά δεν ήξερες πώς να αποφύγεις την χειροκίνητη αντιγραφή‑επικόλληση; Δεν είστε μόνοι. Σε πολλές περιπτώσεις αναφοράς έχετε μια λίστα αντικειμένων που πρέπει να επεκταθεί σε γραμμές αυτόματα, και η χειροκίνητη εκτέλεση είναι συνταγή για λάθη.

Το θέμα είναι—χρησιμοποιώντας το SmartMarkerProcessor από τη βιβλιοθήκη **GemBox.Spreadsheet** σας επιτρέπει να **συμπληρώσετε ένα πρότυπο Excel** με μια μόνο γραμμή C# και να κάνετε τις γραμμές να επαναλαμβάνονται για κάθε στοιχείο στη συλλογή σας. Σε αυτόν τον οδηγό θα περάσουμε βήμα προς βήμα, θα σας δείξουμε τον πλήρη κώδικα και θα εξηγήσουμε γιατί κάθε μέρος είναι σημαντικό, ώστε να μπορείτε με σιγουριά να επαναλαμβάνετε γραμμές στο Excel χωρίς καμία δυσκολία.

## Τι θα μάθετε

* Πώς να ορίσετε τη δομή δεδομένων που καθοδηγεί τη λειτουργία επανάληψης.  
* Πώς να συνδέσετε ένα `SmartMarkerProcessor` σε ένα βιβλίο εργασίας που περιέχει ένα κρυφό φύλλο προτύπου.  
* Πώς ο δείκτης `${Repeat:Item}` επεκτείνεται σε πολλαπλές γραμμές αυτόματα.  
* Συμβουλές για τη διαχείριση ειδικών περιπτώσεων όπως κενές συλλογές ή προσαρμοσμένη μορφοποίηση.  

Στο τέλος αυτού του σεμιναρίου θα μπορείτε να **συμπληρώσετε το Excel από δεδομένα** με τρόπο που κλιμακώνεται, είναι εύκολο στη συντήρηση και λειτουργεί με οποιοδήποτε έργο .NET.

---

## Προαπαιτούμενα

* .NET 6.0 ή νεότερο (ο κώδικας χρησιμοποιεί σύγχρονα χαρακτηριστικά C#).  
* Το πακέτο NuGet **GemBox.Spreadsheet** (η δωρεάν έκδοση λειτουργεί για έως 150 γραμμές).  
* Ένα βασικό αρχείο προτύπου Excel (`Template.xlsx`) με ένα κρυφό φύλλο με όνομα `HiddenTemplate`.  
* Γνώση των αντικειμένων C# και LINQ είναι χρήσιμη αλλά όχι απαραίτητη.

---

## Βήμα 1 – Ορισμός της δομής δεδομένων για επανάληψη

Πρώτα, χρειάζεστε μια πηγή δεδομένων που η μηχανή SmartMarker μπορεί να διατρέξει. Στις περισσότερες πραγματικές εφαρμογές αυτό προέρχεται από μια βάση δεδομένων, ένα API ή ένα αρχείο CSV. Για λόγους σαφήνειας θα χρησιμοποιήσουμε έναν ανώνυμο τύπο με μια μόνο ιδιότητα που ονομάζεται `Item` και περιέχει έναν πίνακα συμβολοσειρών.

```csharp
// Step 1: Define the data that will be repeated in the template
var repeatData = new { Item = new[] { "A", "B", "C" } };
```

> **Γιατί είναι σημαντικό:** Ο δείκτης `${Repeat:Item}` μέσα στο πρότυπο Excel ψάχνει για μια ιδιότητα με όνομα `Item`. Αν μετονομάσετε την ιδιότητα, ενημερώστε τον δείκτη αναλόγως. Αυτή η στενή σύνδεση εξασφαλίζει ότι το πρότυπο παραμένει συγχρονισμένο με τον κώδικά σας, κάνοντας πιο εύκολο το **συμπλήρωμα προτύπου Excel** χωρίς να μαντεύετε τα ονόματα των στηλών.

### Συνηθισμένες παραλλαγές

* **Complex objects:** Αντί για έναν απλό πίνακα συμβολοσειρών μπορείτε να παρέχετε μια λίστα αντικειμένων (`new[] { new { Name = "A", Qty = 10 } }`). Ο δείκτης θα επαναλάβει τις γραμμές και μπορείτε να αναφέρετε `${Item.Name}` και `${Item.Qty}` στο φύλλο.  
* **Empty collections:** Αν το `Item` είναι κενό, το SmartMarker απλώς αφαιρεί το μπλοκ επανάληψης, αφήνοντας το πρότυπο άθικτο—ιδανικό για προαιρετικές ενότητες.

## Βήμα 2 – Δημιουργία του SmartMarkerProcessor για το κρυφό φύλλο προτύπου

Στη συνέχεια, φορτώστε το βιβλίο εργασίας σας και δημιουργήστε ένα `SmartMarkerProcessor`. Στοχεύστε το στο βιβλίο εργασίας που περιέχει το κρυφό φύλλο προτύπου· το SmartMarker θα αντιγράψει αυτό το φύλλο σε ένα ορατό και θα επεκτείνει τους δείκτες επανάληψης.

```csharp
using GemBox.Spreadsheet;

// Load the workbook that holds the hidden template sheet.
var wb = ExcelFile.Load("Template.xlsx");

// Step 2: Create a SmartMarkerProcessor for the workbook that holds the hidden template sheet
var processor = new SmartMarkerProcessor(wb);
```

> **Συμβουλή επαγγελματία:** Αν έχετε πολλαπλά πρότυπα στο ίδιο αρχείο, μπορείτε να καθορίσετε το όνομα του φύλλου προέλευσης όταν καλείτε το `processor.Process`. Αυτό βοηθά όταν χρειάζεται να **επαναλάβετε γραμμές στο Excel** για διαφορετικές ενότητες μιας αναφοράς.

### Διαχείριση ειδικών περιπτώσεων

* **Missing template sheet:** Τυλίξτε τη φόρτωση σε try/catch και καταγράψτε ένα σαφές σφάλμα—αυτό αποτρέπει σιωπηλές αποτυχίες όταν η διαδρομή του αρχείου είναι λανθασμένη.  
* **Large data sets:** Για χιλιάδες γραμμές, σκεφτείτε τη ροή εξόδου σε αρχείο (`processor.Save`) αντί να κρατάτε όλα στη μνήμη.

## Βήμα 3 – Εφαρμογή των δεδομένων και επέκταση του δείκτη `${Repeat:Item}`

Τώρα έρχεται η μαγική γραμμή που πραγματικά επαναλαμβάνει τις γραμμές. Περνάτε το αντικείμενο που δημιουργήσατε στο Βήμα 1 στο `processor.Process`. Το SmartMarker θα εντοπίσει κάθε δείκτη `${Repeat:Item}`, θα αντιγράψει τη γραμμή για κάθε στοιχείο και θα αντικαταστήσει τα placeholders με τις πραγματικές τιμές.

```csharp
// Step 3: Apply the data to the template, expanding the ${Repeat:Item} marker
processor.Process(repeatData);

// Save the resulting workbook.
wb.Save("Result.xlsx");
```

### Τι θα πρέπει να δείτε

Όταν ανοίξετε το `Result.xlsx`, το κρυφό φύλλο προτύπου έχει αντιγραφεί σε ένα νέο ορατό φύλλο (προεπιλογή ονομάζεται `Sheet1`). Η γραμμή που περιείχε `${Repeat:Item}` εμφανίζεται τώρα τρεις φορές, με τα κελιά να δείχνουν **A**, **B**, και **C** αντίστοιχα.

| Item |
|------|
| A    |
| B    |
| C    |

Αν προσθέσατε περισσότερες στήλες όπως `${Item.Price}`, αυτές θα συμπληρώνονταν αυτόματα από την πηγή δεδομένων.

## Προχωρημένες συμβουλές & συχνές παγίδες

### 1. Μορφοποίηση των επαναλαμβανόμενων γραμμών

Το SmartMarker αντιγράφει ολόκληρη τη γραμμή—συμπεριλαμβανομένων των στυλ κελιών, περιγραμμάτων και υπό συνθήκη μορφοποίησης. Αν χρειάζεστε διαφορετικό στυλ για την πρώτη ή την τελευταία γραμμή, προσθέστε επιπλέον δείκτες όπως `${If:Item.IsFirst}` και χρησιμοποιήστε υπό συνθήκη τύπους μέσα στο Excel.

### 2. Διαχείριση μεγάλων συνόλων δεδομένων

Όταν εργάζεστε με > 10 000 γραμμές, απενεργοποιήστε τον αυτόματο υπολογισμό του Excel πριν από την επεξεργασία:

```csharp
wb.WorkbookOptions = new WorkbookOptions { RecalculateAllFormulas = false };
```

Ενεργοποιήστε ξανά μετά την αποθήκευση για να διατηρήσετε την απόδοση γρήγορη.

### 3. Συμπλήρωση Excel από δεδομένα σε πραγματική βάση δεδομένων

```csharp
var orders = dbContext.Orders
    .Where(o => o.Date >= start && o.Date <= end)
    .Select(o => new { o.OrderId, o.CustomerName, o.Total })
    .ToArray();

processor.Process(new { Order = orders });
```

Στη συνέχεια χρησιμοποιήστε `${Repeat:Order}` στο πρότυπο για να απαριθμήσετε κάθε παραγγελία. Αυτό το μοτίβο δείχνει πόσο εύκολο είναι να **συμπληρώσετε το Excel από δεδομένα** απευθείας από το Entity Framework.

### 4. Χρήση πολλαπλών μπλοκ επανάληψης

Μπορείτε να έχετε πολλούς δείκτες `${Repeat:...}` στο ίδιο φύλλο ή σε διαφορετικά φύλλα. Το SmartMarker τα επεξεργάζεται διαδοχικά, έτσι η σειρά έχει σημασία μόνο αν ένα μπλοκ εξαρτάται από το αποτέλεσμα του άλλου.

## Πλήρες εκτελέσιμο παράδειγμα

Παρακάτω υπάρχει μια αυτόνομη εφαρμογή κονσόλας που μπορείτε να επικολλήσετε στο Visual Studio και να εκτελέσετε αμέσως. Δείχνει όλα τα τρία βήματα καθώς και την αποθήκευση του αρχείου.

```csharp
using System;
using GemBox.Spreadsheet;

class Program
{
    static void Main()
    {
        // License free version (up to 150 rows). For production use, set your license key.
        SpreadsheetInfo.SetLicense("FREE-LIMITED-KEY");

        // 1️⃣ Define the data to repeat.
        var repeatData = new { Item = new[] { "A", "B", "C" } };

        // 2️⃣ Load the template workbook (ensure Template.xlsx exists next to the exe).
        var wb = ExcelFile.Load("Template.xlsx");

        // Create processor bound to the workbook.
        var processor = new SmartMarkerProcessor(wb);

        // 3️⃣ Process the data – this expands the ${Repeat:Item} marker.
        processor.Process(repeatData);

        // Save the populated workbook.
        wb.Save("Result.xlsx");

        Console.WriteLine("Excel file generated successfully – check Result.xlsx");
    }
}
```

**Αναμενόμενο αποτέλεσμα:** Το `Result.xlsx` περιέχει ένα φύλλο όπου η γραμμή με `${Repeat:Item}` εμφανίζεται τρεις φορές, δείχνοντας A, B και C. Δεν απαιτούνται χειροκίνητες προσαρμογές.
