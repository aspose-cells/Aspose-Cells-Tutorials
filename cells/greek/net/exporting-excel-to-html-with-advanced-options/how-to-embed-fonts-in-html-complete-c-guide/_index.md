# Πώς να Ενσωματώσετε Γραμματοσειρές σε HTML – Πλήρης Οδηγός C# 

Έχετε αναρωτηθεί ποτέ **πώς να ενσωματώσετε γραμματοσειρές σε HTML** κατά την εξαγωγή ενός βιβλίου εργασίας Excel; Δεν είστε μόνοι. Πολλοί προγραμματιστές αντιμετωπίζουν πρόβλημα όταν το παραγόμενο HTML φαίνεται σωστό στον υπολογιστή τους αλλά χάνει την τυπογραφία του σε άλλη συσκευή. Τα καλά νέα; Με το Aspose.Cells για .NET μπορείτε να ενσωματώσετε τα ακριβή αρχεία γραμματοσειρών απευθείας στο HTML αποτέλεσμα — χωρίς να λείπουν χαρακτήρες.

Σε αυτό το tutorial θα περάσουμε βήμα‑βήμα ένα ολοκληρωμένο παράδειγμα που όχι μόνο δείχνει **πώς να ενσωματώσετε γραμματοσειρές σε HTML**, αλλά επίσης παρουσιάζει **αναγκαστικό υπολογισμό τύπων**, **μετατροπή Excel σε HTML**, και ακόμη **πώς να ορίσετε περιοχή εκτύπωσης** πριν εξαγάγετε ένα γράφημα σε επεξεργάσιμο PPTX. Στο τέλος θα έχετε ένα ενιαίο, εκτελέσιμο πρόγραμμα C# που μπορείτε να ενσωματώσετε σε οποιοδήποτε έργο .NET.

---

## Τι Θα Κατασκευάσετε

- Δημιουργήστε ένα νέο βιβλίο εργασίας, γράψτε μερικούς τύπους πίνακα και **αναγκάστε τον υπολογισμό τύπων** ώστε τα αποτελέσματα να ενσωματωθούν στο αρχείο.  
- Αποθηκεύστε το βιβλίο εργασίας ως HTML ενώ **ενσωματώνετε γραμματοσειρές** και τους επιλογείς παραλλαγής τους.  
- Φορτώστε ένα δεύτερο βιβλίο εργασίας που περιέχει γράφημα, ορίστε μια **περιοχή εκτύπωσης**, και εξαγάγετε αυτό το φύλλο σε επεξεργάσιμο PowerPoint.  
- Όλα αυτά χρησιμοποιώντας μόνο λίγες γραμμές καθαρού, καλά σχολιασμένου κώδικα C#.

Χωρίς εξωτερικά εργαλεία, χωρίς χειροκίνητη αντιγραφή‑επικόλληση αρχείων γραμματοσειρών — το Aspose.Cells κάνει τη σκληρή δουλειά για εσάς.

## Βήμα 1 – Δημιουργία Φύλλου Εργασίας και Εγγραφή Πίνακα Τύπων  

Αρχικά δημιουργούμε μια νέα παρουσία `Workbook` και τοποθετούμε δύο τύπους πίνακα στα κελιά **A1** και **A3**. Αυτοί οι τύποι (`WRAPCOLS` και `WRAPROWS`) παράγουν έναν μικρό πίνακα 2‑στήλες/2‑γραμμές που θα δούμε αργότερα αποδομένο στην έξοδο HTML.

```csharp
using Aspose.Cells;

namespace FontEmbeddingDemo
{
    class Program
    {
        static void Main()
        {
            // Step 1: Create a new workbook and get the first worksheet
            Workbook workbook = new Workbook();
            Worksheet worksheet = workbook.Worksheets[0];

            // Write WRAPCOLS formula – returns a 2‑column array
            worksheet.Cells[0, 0].Formula = "=WRAPCOLS({1,2,3,4},2)";

            // Write WRAPROWS formula – returns a 2‑row array
            worksheet.Cells[2, 0].Formula = "=WRAPROWS({1;2;3;4},2)";
```

> **Γιατί είναι σημαντικό:** Εισάγοντας τύπους λαμβάνετε δυναμικό περιεχόμενο που θα αξιολογηθεί όταν αναγκάσουμε τον υπολογισμό αργότερα. Επίσης δείχνει ότι η εξαγωγή HTML μπορεί να διαχειριστεί σωστά τα αποτελέσματα πίνακα.

---

## Βήμα 2 – Αναγκαστικός Υπολογισμός Τύπων  

Το Aspose.Cells αξιολογεί τους τύπους με καθυστέρηση. Για να διασφαλίσουμε ότι το HTML μας περιέχει τις υπολογισμένες τιμές (αντί για ακατέργαστους τύπους), καλούμε τη μέθοδο `CalculateFormula()`.

```csharp
            // Step 2: Force calculation so the formulas are evaluated
            worksheet.CalculateFormula();
```

> **Συμβουλή:** Αν παραλείψετε αυτό το βήμα, το HTML θα εμφανίσει το κείμενο του τύπου (`=WRAPCOLS...`) αντί για τους αριθμούς, κάτι που αναιρεί το σκοπό μιας επαγγελματικής εξαγωγής.

---

## Βήμα 3 – Διαμόρφωση Επιλογών Αποθήκευσης HTML για Ενσωμάτωση Γραμματοσειρών  

Τώρα έρχεται το αστέρι της παράστασης: η ενσωμάτωση γραμματοσειρών. Ορίζοντας το `EmbedFonts` σε `true` λέτε στο Aspose να συμπεριλάβει τα δεδομένα της γραμματοσειράς ως ροές κωδικοποιημένες σε Base64 μέσα στο παραγόμενο αρχείο HTML. Η ενεργοποίηση του `EmbedFontVariationSelectors` εξασφαλίζει ότι οποιοιδήποτε επιλογείς παραλλαγής OpenType (χρησιμοποιούνται για προχωρημένη τυπογραφία) διατηρούνται επίσης.

```csharp
            // Step 3: Prepare HTML save options that embed fonts and their variation selectors
            HtmlSaveOptions htmlSaveOptions = new HtmlSaveOptions
            {
                EmbedFonts = true,
                EmbedFontVariationSelectors = true
            };
```

> **Πώς λειτουργεί:** Όταν γράφεται το HTML, το Aspose εισάγει ένα μπλοκ `<style>` με κανόνες `@font-face` που αναφέρονται στα ενσωματωμένα data URIs. Τα προγράμματα περιήγησης θα εμφανίσουν την ακριβώς ίδια γραμματοσειρά ανεξάρτητα από τις εγκατεστημένες γραμματοσειρές του χρήστη.

## Βήμα 4 – Αποθήκευση του Φύλλου Εργασίας ως HTML  

Αρχικά αποθηκεύουμε το βιβλίο εργασίας σε αρχείο `.xlsx` (για την περίπτωση που χρειαστείτε την πηγή) και στη συνέχεια το εξάγουμε σε HTML χρησιμοποιώντας τις επιλογές που ορίσαμε.

```csharp
            // Step 4: Save the workbook as HTML using the configured options
            string outputDir = @"C:\Demo\Output\"; // adjust to your environment
            workbook.Save(Path.Combine(outputDir, "fontDemo.xlsx"));
            workbook.Save(Path.Combine(outputDir, "fontDemo.html"), htmlSaveOptions);
```

> **Αποτέλεσμα:** Ανοίξτε το `fontDemo.html` σε οποιοδήποτε σύγχρονο πρόγραμμα περιήγησης και θα δείτε τις τιμές του πίνακα να εμφανίζονται με την ενσωματωμένη γραμματοσειρά, ακόμη και αν η γραμματοσειρά δεν είναι εγκατεστημένη στον υπολογιστή σας.

## Βήμα 5 – Φόρτωση Βιβλίου Εργασίας με Γράφημα και Ορισμός Περιοχής Εκτύπωσης  

Στη συνέχεια δείχνουμε **πώς να ορίσετε περιοχή εκτύπωσης** πριν εξαγάγετε ένα φύλλο που περιέχει γράφημα. Η περιοχή εκτύπωσης περιορίζει τι θα αποδοθεί, κάτι χρήσιμο όταν θέλετε μόνο ένα συγκεκριμένο εύρος στο τελικό PPTX.

```csharp
            // Step 5: Load a workbook that contains a chart and configure PPTX export options
            Workbook chartWorkbook = new Workbook(Path.Combine(outputDir, "chartEditable.xlsx"));

            // Define the print area (e.g., A1:G20) – this is the SECONDARY keyword in action
            chartWorkbook.Worksheets[0].PageSetup.PrintArea = "A1:G20";
```

> **Γιατί να ορίσετε περιοχή εκτύπωσης;** Χωρίς αυτήν, το Aspose θα εξάγει ολόκληρο το φύλλο, ενδεχομένως να συμπεριλάβει κενές γραμμές/στήλες και να αυξήσει το μέγεθος του αρχείου PPTX.

## Βήμα 6 – Εξαγωγή του Φύλλου Εργασίας σε Επεξεργάσιμο PPTX  

Τέλος εξάγουμε το φύλλο εργασίας σε επεξεργάσιμο αρχείο PowerPoint. Ορίζοντας το `ExportChartAsEditable = true`, το γράφημα αποθηκεύεται ως εγγενή σχήματα PowerPoint, επιτρέποντας στους τελικούς χρήστες να το τροποποιήσουν απευθείας στο PowerPoint.

```csharp
            // Step 6: Configure PPTX export options
            ImageOrPrintOptions pptSaveOptions = new ImageOrPrintOptions
            {
                SaveFormat = SaveFormat.Pptx,
                ExportChartAsEditable = true
            };

            // Step 7: Save as editable PPTX
            chartWorkbook.Save(Path.Combine(outputDir, "editableChart.pptx"), pptSaveOptions);
        }
    }
}
```

> **Τι λαμβάνετε:** Το `editableChart.pptx` περιέχει το γράφημα από το `chartEditable.xlsx` ως επεξεργάσιμα αντικείμενα PowerPoint, περιορισμένα στο εύρος `A1:G20`.

## Συχνές Ερωτήσεις & Ακραίες Περιπτώσεις  

### Τι γίνεται αν η γραμματοσειρά μου δεν είναι εγκατεστημένη στον διακομιστή;  

Το Aspose.Cells θα ενσωματώσει μόνο τις γραμματοσειρές που είναι *διαθέσιμες* στο χρόνο εκτέλεσης. Εάν λείπει ένα συγκεκριμένο αρχείο γραμματοσειράς, το HTML θα επιστρέψει στην προεπιλεγμένη γραμματοσειρά του προγράμματος περιήγησης. Για να εξασφαλίσετε την ενσωμάτωση, αντιγράψτε τα απαιτούμενα αρχεία `.ttf`/`.otf` στο φάκελο της εφαρμογής σας και αναφερθείτε σε αυτά μέσω του `FontInfo` (προχωρημένο σενάριο).

### Μπορώ να ενσωματώσω μόνο ένα υποσύνολο χαρακτήρων για να μειώσω το μέγεθος του αρχείου;  

Ναι. Χρησιμοποιήστε `HtmlSaveOptions.FontEmbeddingMode = FontEmbeddingset`. Αυτό λέει στο Aspose να συμπεριλάβει μόνο τα γλύφους που χρησιμοποιούνται πραγματικά στο βιβλίο εργασίας, μειώνοντας δραστικά το μέγεθος του HTML.

### Λειτουργεί ο **αναγκαστικός υπολογισμός τύπων** και για ευμετάβλητες συναρτήσεις όπως `NOW()`;  

Απόλυτα. Η `CalculateFormula()` αξιολογεί όλους τους τύπους, συμπεριλαμβανομένων των ευμετάβλητων, τη στιγμή που την καλείτε. Εάν χρειάζεστε τον υπολογισμό να αντανακλά μια συγκεκριμένη ημερομηνία/ώρα, ορίστε πρώτα τις `CalculationOptions` του βιβλίου εργασίας.

### Τι γίνεται με μεγάλα βιβλία εργασίας – θα αυξήσει η ενσωμάτωση γραμματοσειρών το μέγεθος του HTML;  

Η ενσωμάτωση γραμματοσειρών προσθέτει περίπου 100‑200 KB ανά γραμματοσειρά (ανάλογα με το μέγεθος). Για τεράστιες αναφορές, σκεφτείτε να συνδέσετε σε γραμματοσειρές που φιλοξενούνται στο web αντί για ενσωμάτωση, ή χρησιμοποιήστε τη λειτουργία υποσυνόλου που αναφέρθηκε νωρίτερα.

## Συμβουλές & Καλές Πρακτικές  

- **Batch saves:** Εάν δημιουργείτε δεκάδες αρχεία HTML, επαναχρησιμοποιήστε ένα μόνο αντικείμενο `HtmlSaveOptions` για να αποφύγετε περιττές κατανομές μνήμης.  
- **Cache print areas:** Κατά την εξαγωγή πολλών φύλλων, αποθηκεύστε την επιθυμητή περιοχή εκτύπωσης σε αρχείο ρυθμίσεων ώστε ο κώδικάς σας να παραμένει DRY.  
- **Validate output:** Μετά την αποθήκευση του HTML, εκτελέστε έναν γρήγορο έλεγχο με headless πρόγραμμα περιήγησης (π.χ., Puppeteer) για να διασφαλίσετε ότι οι γραμματοσειρές αποδίδονται σωστά πριν τη διανομή στους χρήστες.  
- **Version lock:** Ο παραπάνω κώδικας στοχεύει στο Aspose.Cells 23.12+. Νεότερες εκδόσεις μπορεί να προσθέσουν επιπλέον επιλογές όπως `FontEmbeddingMode`. Πάντα ελέγχετε τις σημειώσεις έκδοσης.
