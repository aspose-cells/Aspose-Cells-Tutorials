# Πώς να Εισάγετε Γραμμές στο GridJs – Προσθέστε Πολλαπλές Γραμμές Γρήγορα

Έχετε αναρωτηθεί ποτέ **πώς να εισάγετε γραμμές** σε ένα data‑grid του GridJs χωρίς να γράψετε έναν βρόχο που διαρκεί για πάντα; Δεν είστε ο μόνος. Σε πολλές επιχειρησιακές εφαρμογές θα φτάσετε σε σημείο όπου χρειάζεται να δημιουργήσετε χώρο για μαζική εισαγωγή, ένα πρότυπο ή απλώς ένα placeholder για μελλοντικά δεδομένα. Τα καλά νέα; Το GridJs σας παρέχει μια ενιαία μέθοδο που κάνει τη σκληρή δουλειά για εσάς.

Σε αυτό το tutorial θα περάσουμε από ένα πλήρες, εκτελέσιμο παράδειγμα που δείχνει πώς να **προσθέσετε 100 γραμμές**, **δημιουργήσετε κενές γραμμές**, και **ελέγξετε τον συνολικό αριθμό γραμμών** μετά τη λειτουργία. Στο τέλος θα έχετε ένα σταθερό μοτίβο που μπορείτε να ενσωματώσετε σε οποιοδήποτε έργο C# που χρησιμοποιεί GridJs.

## Προαπαιτούμενα

Πριν βουτήξουμε, βεβαιωθείτε ότι έχετε:

- .NET 6.0 ή νεότερο (το API λειτουργεί το ίδιο και στο .NET Framework 4.8, αλλά το νεότερο SDK προσφέρει καλύτερα εργαλεία).
- Μια αναφορά στο πακέτο NuGet `GridJs` ή το μεταγλωττισμένο DLL που περιέχει την κλάση `GridJs`.
- Βασική εξοικείωση με τη σύνταξη C# — τίποτα εξωτικό, μόνο τυπικές δηλώσεις `using` και βασικές αρχές αντικειμενοστραφούς προγραμματισμού.

Αν κάποιο από τα παραπάνω σας προκαλεί ανησυχία, κάντε ένα διάλειμμα και τακτοποιήστε το. Τα επόμενα βήματα υποθέτουν ότι το αντικείμενο grid έχει ήδη δημιουργηθεί και είναι έτοιμο να δεχτεί γραμμές.

## Βήμα 1: Ρύθμιση του Αντικειμένου Grid

Πρώτα απ' όλα, χρειάζεστε ένα αντικείμενο `GridJs`. Σε μια πραγματική εφαρμογή αυτό πιθανότατα προέρχεται από ένα service layer ή εισάγεται μέσω dependency injection, αλλά για σαφήνεια θα το δημιουργήσουμε τοπικά.

```csharp
using System;
using GridJsLibrary;   // <-- replace with the actual namespace of GridJs

namespace GridJsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Step 1: Create or obtain the grid you want to modify
            GridJs gridJs = new GridJs();   // replace with your actual grid initialization
```

> **Γιατί είναι σημαντικό:** Η δημιουργία του grid σας δίνει ένα καθαρό ξεκίνημα, εξασφαλίζοντας ότι η λογική εισαγωγής γραμμών δεν θα συγκρουστεί με κατάλοιπα κατάστασης από προηγούμενες εκτελέσεις.

## Βήμα 2: Εισαγωγή 100 Γραμμών σε Συγκεκριμένο Δείκτη

Τώρα έρχεται η ουσία του **πώς να εισάγετε γραμμές**. Η μέθοδος `InsertRows` δέχεται δύο παραμέτρους: τον μηδενικό δείκτη εκκίνησης και τον αριθμό των γραμμών που θέλετε να προσθέσετε. Ας εισάγουμε 100 γραμμές ξεκινώντας από τη γραμμή 5.

```csharp
            // Step 2: Insert 100 rows starting at row index 5 (zero‑based)
            // This pushes existing rows down and creates space for new data.
            gridJs.InsertRows(5, 100);
```

> **Pro tip:** Αν χρειάζεται να προσθέσετε γραμμές στο τέλος του grid, μπορείτε να χρησιμοποιήσετε το `gridJs.RowCount` ως δείκτη εκκίνησης. Με αυτόν τον τρόπο «προσαρτάτε» ουσιαστικά αντί να εισάγετε.

### Τι Συμβαίνει Πίσω από τη Σκηνή;

- **Memory Allocation:** Η `InsertRows` κατανέμει εσωτερικά ένα μπλοκ κενών αντικειμένων γραμμής, ώστε να μην χρειάζεται να δημιουργήσετε χειροκίνητα το καθένα.
- **Index Shifting:** Όλες οι γραμμές που βρίσκονταν στον δείκτη 5 ή μετά μετακινούνται προς τα κάτω κατά 100 θέσεις, διατηρώντας τα αρχικά τους δεδομένα.
- **Performance:** Επειδή η λειτουργία εκτελείται με μία κλήση, συνήθως είναι ταχύτερη από το να κάνετε βρόχο `InsertRow` 100 φορές.

## Βήμα 3: Επαλήθευση της Εισαγωγής (Έλεγχος Συνολικού Αριθμού Γραμμών)

Αφού προσθέσετε τις γραμμές, είναι καλή πρακτική να **ελέγξετε τον συνολικό αριθμό γραμμών** για να επιβεβαιώσετε ότι η λειτουργία πέτυχε. Η ιδιότητα `RowCount` σας δίνει τον τρέχοντα αριθμό γραμμών στο grid.

```csharp
            // Step 3: (Optional) Verify the insertion or continue processing
            int newRowCount = gridJs.RowCount; // example property to check total rows
            Console.WriteLine($"Grid now contains {newRowCount} rows.");
```

Αν ξεκινήσατε, για παράδειγμα, με 20 γραμμές, θα πρέπει να δείτε `120` να εκτυπώνεται στην κονσόλα. Αυτό το απλό βήμα επαλήθευσης μπορεί να σας εξοικονομήσει ώρες εντοπισμού σφαλμάτων αργότερα.

## Βήμα 4: Συμπλήρωση των Νέων Κενών Γραμμών (Προαιρετικό)

Συχνά θέλετε να γεμίσετε αυτές τις φρέσκιες κενές γραμμές με placeholder δεδομένα ή προεπιλεγμένα αντικείμενα. Επειδή η `InsertRows` παρέχει ένα μπλοκ κενών γραμμών, μπορείτε να κάνετε βρόχο πάνω στο εύρος και να αναθέσετε τιμές.

```csharp
            // Optional: Fill the newly created rows with default values
            for (int i = 5; i < 5 + 100; i++)
            {
                var row = gridJs.GetRow(i); // assume GetRow returns a mutable row object
                row["Name"] = $"Placeholder {i - 4}";
                row["CreatedOn"] = DateTime.UtcNow;
            }

            // Verify a sample row
            var sample = gridJs.GetRow(5);
            Console.WriteLine($"First inserted row name: {sample["Name"]}");
        }
    }
}
```

> **Γιατί μπορεί να το κάνετε:** Η δημιουργία κενών γραμμών είναι χρήσιμη όταν χρειάζεστε ένα πρότυπο για εισαγωγή από τον χρήστη, ένα placeholder για μαζική μεταφόρτωση, ή απλώς θέλετε να κρατήσετε χώρο για μελλοντικούς υπολογισμούς.

## Συνηθισμένες Παραλλαγές & Ακραίες Περιπτώσεις

### Προσθήκη Λιγότερων από 100 Γραμμές

Αν χρειάζεται μόνο να **προσθέσετε πολλαπλές γραμμές** — π.χ. 10 ή 25 — η ίδια κλήση `InsertRows` λειτουργεί· απλώς αντικαταστήστε το `100` με τον επιθυμητό αριθμό.

```csharp
gridJs.InsertRows(startIndex, 25); // adds 25 rows
```

### Εισαγωγή στην Αρχή του Grid

Θέλετε να προσθέσετε γραμμές στην αρχή; Χρησιμοποιήστε `0` ως δείκτη εκκίνησης:

```csharp
gridJs.InsertRows(0, 5); // adds 5 rows at the very beginning
```

### Διαχείριση Δεικτών Εκτός Εύρους

Η παράδοση ενός δείκτη μεγαλύτερου από το `RowCount` προκαλεί `ArgumentOutOfRangeException`. Προστατέψτε τον κώδικά σας:

```csharp
int safeIndex = Math.Min(requestedIndex, gridJs.RowCount);
gridJs.InsertRows(safeIndex, 100);
```

### Αντιμετώπιση Grids μόνο για Ανάγνωση

Κάποιες ρυθμίσεις του GridJs εκθέτουν μια προβολή μόνο για ανάγνωση. Σε αυτήν την περίπτωση, θα πρέπει να μεταβείτε σε μια εγγράψιμη παρουσία ή να απενεργοποιήσετε προσωρινά τη σημαία read‑only πριν καλέσετε την `InsertRows`.

## Συμβουλές Απόδοσης

- **Batch Operations:** Αν εισάγετε γραμμές επανειλημμένα μέσα σε βρόχο, ομαδοποιήστε τις σε μία κλήση `InsertRows` όποτε είναι δυνατόν. Αυτό μειώνει τις εσωτερικές επανακατανομές λιστών.
- **Avoid UI Refreshes:** Σε grids που συνδέονται με UI, αναστείλετε το rendering (`gridJs.BeginUpdate()`) πριν την εισαγωγή γραμμών και επαναλάβετε (`gridJs.EndUpdate()`) μετά, ώστε να αποφύγετε τρεμόπαιγμα.
- **Memory Profiling:** Μεγάλες εισαγωγές (π.χ. >10 000 γραμμές) μπορούν να αυξήσουν τη χρήση μνήμης. Σκεφτείτε σελιδοποίηση ή ροή δεδομένων αντί για μία τεράστια εισαγωγή.

## Συνοπτική Παρουσίαση Πλήρους Παραδείγματος

Συνδυάζοντας όλα τα παραπάνω, εδώ είναι το πλήρες, έτοιμο για αντιγραφή‑επικόλληση πρόγραμμα:

```csharp
using System;
using GridJsLibrary;   // replace with the actual namespace

namespace GridJsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Create the grid instance
            GridJs gridJs = new GridJs();

            // Insert 100 rows starting at index 5
            gridJs.InsertRows(5, 100);

            // Verify insertion
            int newRowCount = gridJs.RowCount;
            Console.WriteLine($"Grid now contains {newRowCount} rows.");

            // Optional: Fill new rows with placeholder data
            for (int i = 5; i < 5 + 100; i++)
            {
                var row = gridJs.GetRow(i);
                row["Name"] = $"Placeholder {i - 4}";
                row["CreatedOn"] = DateTime.UtcNow;
            }

            // Show a sample row
            var sample = gridJs.GetRow(5);
            Console.WriteLine($"First inserted row name: {sample["Name"]}");
        }
    }
}
```

Εκτελέστε αυτό το πρόγραμμα και θα δείτε την έξοδο της κονσόλας που επιβεβαιώνει τον αριθμό γραμμών και το όνομα της πρώτης placeholder γραμμής. Αυτή είναι η πλήρης απάντηση στο **πώς να εισάγετε γραμμές** στο GridJs, με επαλήθευση και προαιρετική πληρότητα δεδομένων.
