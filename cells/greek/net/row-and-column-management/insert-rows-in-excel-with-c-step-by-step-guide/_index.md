# Εισαγωγή γραμμών στο Excel με C# – Οδηγός βήμα‑βήμα

Κάποτε χρειάστηκε να **εισάγετε γραμμές στο Excel** αλλά δεν ήξερες από πού να ξεκινήσεις; Δεν είσαι μόνος σου—οι περισσότεροι προγραμματιστές συναντούν αυτό το εμπόδιο όταν αυτοματοποιούν τα φύλλα εργασίας για πρώτη φορά. Το καλό νέο είναι ότι με λίγες γραμμές C# μπορείς να εισάγεις γραμμές σε οποιαδήποτε θέση, να κάνεις μαζική εισαγωγή γραμμών, και ακόμη να προσθέσεις 500 γραμμές με μία εντολή χωρίς να επηρεάσει η απόδοση.

Σε αυτό το tutorial θα περάσουμε από ένα πλήρες, εκτελέσιμο παράδειγμα που καλύπτει **πώς να εισάγεις γραμμές**, πώς να **εισάγεις 500 γραμμές**, και τις βέλτιστες πρακτικές για μια **μαζική εισαγωγή γραμμών Excel**. Στο τέλος θα έχεις ένα αυτόνομο script που μπορείς να ενσωματώσεις σε οποιοδήποτε .NET project και να αρχίσεις να το χρησιμοποιείς αμέσως.

## Προαπαιτούμενα

- .NET 6.0 ή νεότερο (ο κώδικας λειτουργεί επίσης με .NET Core και .NET Framework)  
- Το πακέτο NuGet **Aspose.Cells for .NET** (ή οποιαδήποτε συμβατή βιβλιοθήκη που εκθέτει `InsertRows`).  
- Βασική κατανόηση της σύνταξης C#—δεν απαιτούνται προχωρημένες έννοιες.

> **Pro tip:** Αν χρησιμοποιείς διαφορετική βιβλιοθήκη (π.χ., EPPlus ή ClosedXML), το όνομα της μεθόδου μπορεί να διαφέρει, αλλά η λογική παραμένει η ίδια.

## Βήμα 1: Ρύθμιση του έργου και εισαγωγή εξαρτήσεων

Δημιούργησε μια νέα εφαρμογή console (ή ενσωμάτωσε το σε υπάρχον project) και πρόσθεσε το πακέτο Aspose.Cells:

```bash
dotnet new console -n ExcelRowInserter
cd ExcelRowInserter
dotnet add package Aspose.Cells
```

Τώρα άνοιξε το `Program.cs` και εισήγαγε τα namespaces που θα χρειαστούμε:

```csharp
using System;
using Aspose.Cells;
```

## Βήμα 2: Φόρτωση ή δημιουργία βιβλίου εργασίας και λήψη του στόχου φύλλου

Αν ήδη έχεις ένα αρχείο Excel, φόρτωσέ το. Διαφορετικά, θα δημιουργήσουμε ένα νέο βιβλίο εργασίας για σκοπούς επίδειξης.

```csharp
// Step 2: Load an existing workbook or create a new one
Workbook workbook = new Workbook();                 // creates a blank workbook
Worksheet ws = workbook.Worksheets[0];              // reference the first worksheet

// Optional: populate a few rows so we can see the effect of insertion
ws.Cells["A1"].PutValue("Header");
ws.Cells["A2"].PutValue("Row 1");
ws.Cells["A3"].PutValue("Row 2");
ws.Cells["A4"].PutValue("Row 3");
```

> **Γιατί είναι σημαντικό:** Η λήψη αναφοράς στο φύλλο εργασίας (`ws`) είναι το θεμέλιο κάθε αυτοματοποίησης Excel. Χωρίς αυτήν δεν μπορείς να χειριστείς κελιά, γραμμές ή στήλες.

## Βήμα 3: Εισαγωγή γραμμών σε συγκεκριμένη θέση

Για **εισαγωγή γραμμών στη θέση** 1000, χρησιμοποιούμε τη μέθοδο `InsertRows`. Το πρώτο όρισμα είναι ο μηδενικής βάσης δείκτης όπου ξεκινά η εισαγωγή, και το δεύτερο όρισμα είναι ο αριθμός των γραμμών που θα προστεθούν.

```csharp
// Step 3: Insert 500 rows beginning at row 1000 (1‑based index for Excel users)
int startRow = 999;          // zero‑based index, so 999 = Excel row 1000
int rowsToInsert = 500;      // bulk insert rows Excel – this is the count

ws.Cells.InsertRows(startRow, rowsToInsert);
```

> **Τι συμβαίνει στο παρασκήνιο;** Η βιβλιοθήκη μετακινεί όλες τις υπάρχουσες γραμμές προς τα κάτω κατά 500, δημιουργώντας κενές γραμμές έτοιμες για δεδομένα. Η λειτουργία αυτή εκτελείται στη μνήμη, οπότε είναι εξαιρετικά γρήγορη ακόμη και για μεγάλα φύλλα.

## Βήμα 4: Επαλήθευση της εισαγωγής (προαιρετικό αλλά συνιστάται)

Είναι καλή πρακτική να επιβεβαιώνεις ότι οι γραμμές εισήχθησαν στη σωστή θέση. Ένας γρήγορος τρόπος είναι να γράψεις μια τιμή στην πρώτη νεοδημιουργημένη γραμμή:

```csharp
// Step 4: Write a test value into the first inserted row
ws.Cells["A1000"].PutValue("Inserted row start");
```

Αν ανοίξεις το αποθηκευμένο αρχείο, θα δεις το κείμενο “Inserted row start” στη γραμμή 1000 του Excel, επιβεβαιώνοντας ότι η **εισαγωγή 500 γραμμών** ολοκληρώθηκε επιτυχώς.

## Βήμα 5: Αποθήκευση του βιβλίου εργασίας

Τέλος, αποθήκευσε τις αλλαγές στο δίσκο:

```csharp
// Step 5: Save the workbook
string outputPath = "InsertedRowsDemo.xlsx";
workbook.Save(outputPath);
Console.WriteLine($"Workbook saved to {outputPath}");
```

Η εκτέλεση του προγράμματος θα δημιουργήσει το `InsertedRowsDemo.xlsx` με τις νέες γραμμές στη θέση τους.

### Πλήρης κώδικας (έτοιμος για αντιγραφή)

```csharp
using System;
using Aspose.Cells;

namespace ExcelRowInserter
{
    class Program
    {
        static void Main(string[] args)
        {
            // Load or create workbook
            Workbook workbook = new Workbook();
            Worksheet ws = workbook.Worksheets[0];

            // Populate some initial data for context
            ws.Cells["A1"].PutValue("Header");
            ws.Cells["A2"].PutValue("Row 1");
            ws.Cells["A3"].PutValue("Row 2");
            ws.Cells["A4"].PutValue("Row 3");

            // Insert 500 rows at Excel row 1000 (zero‑based index 999)
            int startRow = 999;
            int rowsToInsert = 500;
            ws.Cells.InsertRows(startRow, rowsToInsert);

            // Write a marker into the first newly inserted row
            ws.Cells["A1000"].PutValue("Inserted row start");

            // Save the result
            string outputPath = "InsertedRowsDemo.xlsx";
            workbook.Save(outputPath);
            Console.WriteLine($"Workbook saved to {outputPath}");
        }
    }
}
```

Η εκτέλεση αυτού του script παράγει ένα αρχείο Excel όπου οι γραμμές 1000‑1499 είναι κενές (εκτός από το δείκτη που προσθέσαμε). Μπορείς τώρα να γεμίσεις αυτές τις γραμμές με δεδομένα, να εφαρμόσεις μορφοποίηση ή να εκτελέσεις περαιτέρω αυτοματοποίηση.

## Ακραίες περιπτώσεις & Συχνές ερωτήσεις

### Τι γίνεται αν η αρχική γραμμή υπερβαίνει το τρέχον μέγεθος φύλλου;

Η Aspose.Cells επεκτείνει αυτόματα το φύλλο εργασίας για να φιλοξενήσει την εισαγωγή. Σε άλλες βιβλιοθήκες, ίσως χρειαστεί να καλέσεις μια μέθοδο όπως `ws.Cells.MaxRows = …` πριν την εισαγωγή.

### Μπορώ να εισάγω γραμμές στη μέση ενός πίνακα χωρίς να σπάσω τύπους;

Ναι. Η μέθοδος `InsertRows` μετακινεί τους τύπους προς τα κάτω, διατηρώντας τις αναφορές. Ωστόσο, οι απόλυτες αναφορές (`$A$1`) παραμένουν αμετάβλητες, οπότε έλεγξε προσεκτικά τυχόν κρίσιμους υπολογισμούς.

### Υπάρχει αντίκτυπος στην απόδοση όταν εισάγονται χιλιάδες γραμμές;

Επειδή η λειτουργία εκτελείται στη μνήμη, το κόστος είναι ελάχιστο. Το πραγματικό bottleneck εμφανίζεται συνήθως όταν γράφεις μεγάλα σύνολα δεδομένων σε αυτές τις γραμμές. Σε αυτήν την περίπτωση, γράψε τιμές παρτίδες χρησιμοποιώντας πίνακες ή `PutValue` με ένα εύρος.

### Πώς να εισάγω γραμμές σε *μαζική* λειτουργία χωρίς βρόχο;

Η κλήση `InsertRows` είναι η ίδια η μαζική λειτουργία—δεν χρειάζεται βρόχος `for`. Αν χρειάζεται να εισάγεις γραμμές σε πολλές, μη συνεχόμενες θέσεις, σκέψου να ταξινομήσεις τις θέσεις κατά φθίνουσα σειρά και να καλέσεις `InsertRows` για κάθε μία· έτσι αποφεύγεις προβλήματα μετατόπισης δεικτών.
