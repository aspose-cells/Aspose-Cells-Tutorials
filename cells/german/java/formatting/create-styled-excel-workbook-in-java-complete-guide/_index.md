# Erstellen eines formatierten Excel-Workbooks in Java – Komplettanleitung

Haben Sie jemals **ein formatiertes Excel-Workbook** aus Java erstellen müssen, waren aber von den Formatierungsdetails überfordert? Sie sind nicht allein. Egal, ob Sie Daten aus einer Datenbank ziehen oder einen Bericht exportieren, das richtige Aussehen – Währungssymbole, Datumsformate, fette Überschriften – kann wie eine lästige Aufgabe erscheinen.  

In diesem Tutorial führen wir Sie durch eine praktische Lösung, die **benutzerdefinierte Formatierung zu Excel hinzufügt**, Ihnen **das Datumsformat für Excel‑Spalten setzt** und schließlich **das Workbook als xlsx speichert**, ohne dass Sie ins Schwitzen kommen. Bereit, es in Aktion zu sehen?

## Was Sie lernen werden

- Wie man ein Aspose.Cells `Workbook` von Grund auf erstellt.  
- Wie man ein Style‑Array anwendet, das allgemeine, Währungs‑ und Datumsspalten abdeckt.  
- Der genaue Aufruf, der **ein formatiertes Excel-Workbook** mit eingebetteten Spaltennamen erzeugt.  
- Wie man die Datei mit der **save workbook as xlsx**‑Methode persistiert.  

Keine externen Konfigurationsdateien, keine obskuren XML‑Tricks – nur reiner Java‑Code, den Sie kopieren‑einfügen, ausführen und anpassen können.

### Voraussetzungen

- Java 8+ (der Code funktioniert mit jeder aktuellen JDK).  
- Aspose.Cells für Java‑Bibliothek (Sie können das neueste JAR von Maven Central beziehen).  
- Ein wenig Erfahrung mit Collections (`List`, `Map`).  

Falls Ihnen einer dieser Punkte unbekannt ist, pausieren Sie kurz und installieren Sie das Aspose.Cells‑JAR; der Rest wird sich von selbst ergeben.

---

## Erstellen eines formatierten Excel-Workbooks – Projekt einrichten

Als erstes: ein neues Maven‑ (oder Gradle‑)Projekt anlegen und Aspose.Cells als Abhängigkeit hinzufügen.

```xml
<!-- pom.xml snippet -->
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-cells</artifactId>
    <version>23.12</version> <!-- use the latest version -->
</dependency>
```

> **Pro‑Tipp:** Wenn Sie Gradle verwenden, funktionieren die gleichen Koordinaten mit `implementation 'com.aspose:aspose-cells:23.12'`.

Warum mit einer Maven‑Konfiguration beginnen? Sie zieht automatisch alle transitiven Abhängigkeiten, sodass Sie später keine Zeit damit verschwenden, die richtigen JARs zu suchen.

---

## Benutzerdefinierte Formatierung zu Excel‑Spalten hinzufügen

Jetzt, wo das Projekt bereit ist, definieren wir die gewünschten Stile. In Aspose.Cells können Sie ein Array von Format‑Strings übergeben, das direkt jeder Spalte zugeordnet wird.

```java
// Define column styles: General, Currency, Date
String[] columnStyles = {
    "General",          // Column A – plain text or numbers
    "$#,##0.00",        // Column B – currency with two decimals
    "mm/dd/yyyy"        // Column C – US‑style date
};
```

Der Schritt **benutzerdefinierte Formatierung zu Excel hinzufügen** ist so einfach wie das Erstellen dieses String‑Arrays.  
Jeder Eintrag folgt dem gleichen Muster, das Sie im Excel‑Dialog „Benutzerdefiniert“ verwenden würden, sodass Sie keine neue Syntax lernen müssen.

---

## Datumsformat für Excel‑Spalte setzen

Enthält Ihre Daten Datumswerte, sollten Sie sicherstellen, dass sie einheitlich angezeigt werden. Der dritte Eintrag in `columnStyles` (`"mm/dd/yyyy"`) erledigt genau das – er weist Aspose.Cells an, die dritte Spalte als Datum zu behandeln und im Format Monat/Tag/Jahr anzuzeigen.

```java
// Example: Adding a row with a Date object
Map<String, Object> row = new HashMap<>();
row.put("Product", "Widget");
row.put("Price", 19.99);
row.put("ReleaseDate", new java.util.Date()); // Today’s date
dataRows.add(row);
```

Wenn das Workbook generiert wird, sorgt die Anweisung **set date format excel column** dafür, dass jede Zelle `ReleaseDate` wie `04/27/2026` (oder das heutige Datum) aussieht. Keine Nachbearbeitung nötig.

---

## Daten importieren und die Stile anwenden

Nachdem die Stile definiert sind, ist der nächste Schritt, unsere Daten in das Arbeitsblatt zu laden. Aspose.Cells stellt `importCustomObjects` bereit, das nicht nur die Daten einfügt, sondern auch das zuvor erstellte Style‑Array anwendet.

```java
// Step 1: Simulate fetching rows from a DB
List<Map<String, Object>> dataRows = fetchRows(); // see helper below

// Step 2: Create a fresh workbook and grab the first sheet
Workbook workbook = new Workbook();
Worksheet worksheet = workbook.getWorksheets().get(0);

// Step 3: Import data with custom styles; true = first row has headers
worksheet.getCells().importCustomObjects(
        dataRows,        // data source
        0,               // start row (0‑based)
        0,               // start column (0‑based)
        columnStyles,    // style array from above
        true);           // first row contains column names
```

### Warum `importCustomObjects`?

- **Ein‑Aufruf‑Magie:** Es schreibt die Daten *und* wendet das Format in einem Schritt an.  
- **Header‑Bewusstsein:** Das Übergeben von `true` weist die API an, die Schlüssel der ersten Map als Spaltenüberschriften zu behandeln – ideal für **create styled excel workbook**‑Szenarien, in denen lesbare Spaltentitel gewünscht sind.  
- **Performance:** Unter der Haube bündelt Aspose die Schreibvorgänge, was schneller ist als das zeilenweise Befüllen von Zellen.

> **Achtung:** Die Reihenfolge der Schlüssel in jeder `Map` ist entscheidend. Verwenden Sie ein `LinkedHashMap`, wenn Sie eine deterministische Spaltenreihenfolge benötigen.

## Workbook als XLSX‑Datei speichern

All die Arbeit wäre sinnlos, wenn Sie die Datei nicht persistieren könnten. Die letzte Zeile ist ein Einzeiler, der **save workbook as xlsx** ausführt.

```java
// Step 5: Persist the workbook
String outputPath = "output/DataTableStyled.xlsx";
workbook.save(outputPath);
System.out.println("Workbook saved to " + outputPath);
```

Die `save`‑Methode erkennt die Dateierweiterung automatisch. Durch Angabe von `.xlsx` stellen Sie sicher, dass das moderne Office‑Open‑XML‑Format verwendet wird, das von den meisten nachgelagerten Tools erwartet wird.

## Vollständiges, ausführbares Beispiel

Unten finden Sie die komplette Klasse, die Sie direkt in Ihre IDE kopieren können. Sie enthält eine Mock‑Methode `fetchRows`, die Beispieldaten zurückgibt, sodass Sie keine echte Datenbank benötigen, um zu testen.

```java
import com.aspose.cells.*;
import java.util.*;

public class ImportDataTableWithStyle {
    public static void main(String[] args) throws Exception {
        // ---------- Step 1: Get data ----------
        List<Map<String, Object>> dataRows = fetchRows();

        // ---------- Step 2: Create workbook ----------
        Workbook workbook = new Workbook();
        Worksheet worksheet = workbook.getWorksheets().get(0);

        // ---------- Step 3: Define column styles ----------
        String[] columnStyles = { "General", "$#,##0.00", "mm/dd/yyyy" };

        // ---------- Step 4: Import data with styles ----------
        // true = first row contains column names
        worksheet.getCells().importCustomObjects(dataRows, 0, 0, columnStyles, true);

        // ---------- Step 5: Save as XLSX ----------
        String outPath = "output/DataTableStyled.xlsx";
        workbook.save(outPath);
        System.out.println("Workbook saved to " + outPath);
    }

    // Mock method – replace with real DB call if needed
    private static List<Map<String, Object>> fetchRows() {
        List<Map<String, Object>> rows = new ArrayList<>();

        Map<String, Object> row1 = new LinkedHashMap<>();
        row1.put("Product", "Gadget");
        row1.put("Price", 49.95);
        row1.put("ReleaseDate", new Date()); // today
        rows.add(row1);

        Map<String, Object> row2 = new LinkedHashMap<>();
        row2.put("Product", "Widget");
        row2.put("Price", 19.99);
        Calendar cal = Calendar.getInstance();
        cal.set(2025, Calendar.DECEMBER, 15);
        row2.put("ReleaseDate", cal.getTime());
        rows.add(row2);

        return rows;
    }
}
```

Führen Sie sie aus, öffnen Sie die erzeugte Datei, und Sie sehen das Ergebnis von **create styled excel workbook**, das Sie erwartet haben.

## Häufige Fragen & Sonderfälle

**Was tun, wenn ich mehr als drei Spalten brauche?**  
Erweitern Sie einfach das `columnStyles`‑Array – jeder Eintrag entspricht einer Spalte in der Reihenfolge. Für Spalten, die Sie unverändert lassen wollen, verwenden Sie `"General"`.

**Kann ich Schriftarten oder Farben anwenden?**  
Ja. Statt eines String‑Formats können Sie ein `Style`‑Objekt erstellen, es einem `StyleFlag` zuweisen und ein `Style[]`‑Array an `importCustomObjects` übergeben. Das ist ein tieferes Thema, aber das Prinzip bleibt gleich.

**Wie sieht es mit großen Datenmengen aus?**  
Aspose.Cells verarbeitet Millionen von Zeilen effizient, Sie sollten jedoch den JVM‑Heap erhöhen (`-Xmx2` …).
