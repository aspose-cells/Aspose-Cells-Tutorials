# जावा में स्टाइल्ड Excel वर्कबुक बनाएं – पूर्ण गाइड

क्या आपको कभी जावा से **create styled excel workbook** बनाने की जरूरत पड़ी लेकिन फॉर्मेटिंग विवरणों से जकड़ गए? आप अकेले नहीं हैं। चाहे आप डेटाबेस से डेटा ले रहे हों या रिपोर्ट एक्सपोर्ट कर रहे हों, सही लुक—करेंसी सिम्बॉल, डेट मास्क, बोल्ड हेडर—पाना एक काम जैसा लग सकता है।  

इस ट्यूटोरियल में हम एक व्यावहारिक समाधान के माध्यम से चलेंगे जो **adds custom formatting to excel** करता है, आपको **set date format excel column** सेट करने देता है, और अंत में **save workbook as xlsx** बिना किसी मेहनत के। क्या आप इसे कार्रवाई में देखना चाहते हैं?

## आप क्या सीखेंगे

- कैसे एक Aspose.Cells `Workbook` को शुरू से बनाएं।  
- कैसे एक style array लागू करें जो General, Currency, और Date कॉलम को कवर करता है।  
- वह सटीक कॉल जो **creates styled excel workbook** को कॉलम नामों के साथ बनाता है।  
- कैसे **save workbook as xlsx** मेथड का उपयोग करके फ़ाइल को स्थायी बनाएं।  

कोई बाहरी कॉन्फ़िगरेशन फ़ाइल नहीं, कोई अस्पष्ट XML ट्रिक्स नहीं—सिर्फ साधारण जावा कोड जिसे आप कॉपी‑पेस्ट, रन और अनुकूलित कर सकते हैं।

### आवश्यकताएँ

- Java 8+ (कोड किसी भी हालिया JDK पर काम करता है)।  
- Aspose.Cells for Java लाइब्रेरी (आप Maven Central से नवीनतम JAR प्राप्त कर सकते हैं)।  
- कलेक्शन्स (`List`, `Map`) की थोड़ी सी परिचितता।  

यदि इनमें से कोई भी परिचित नहीं लग रहा है, तो एक क्षण रुकें और Aspose.Cells JAR इंस्टॉल करें; बाकी स्वाभाविक रूप से चलेंगे।

## स्टाइल्ड Excel वर्कबुक बनाना – प्रोजेक्ट सेटअप

सबसे पहले: एक नया Maven (या Gradle) प्रोजेक्ट बनाएं और Aspose.Cells को डिपेंडेंसी के रूप में जोड़ें।

```xml
<!-- pom.xml snippet -->
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-cells</artifactId>
    <version>23.12</version> <!-- use the latest version -->
</dependency>
```

> **Pro tip:** यदि आप Gradle का उपयोग कर रहे हैं, तो वही कोऑर्डिनेट्स `implementation 'com.aspose:aspose-cells:23.12'` के साथ काम करेंगे।

Maven सेटअप से शुरू क्यों करें? यह सभी ट्रांज़िटिव डिपेंडेंसीज़ को स्वचालित रूप से खींचता है, इसलिए आप बाद में सही JARs खोजने में समय बर्बाद नहीं करेंगे।

## Excel कॉलम में कस्टम फॉर्मेटिंग जोड़ें

अब प्रोजेक्ट तैयार है, चलिए उन स्टाइल्स को परिभाषित करते हैं जो हम चाहते हैं। Aspose.Cells में आप फॉर्मेट स्ट्रिंग्स का एक array पास कर सकते हैं जो सीधे प्रत्येक कॉलम से मैप होता है।

```java
// Define column styles: General, Currency, Date
String[] columnStyles = {
    "General",          // Column A – plain text or numbers
    "$#,##0.00",        // Column B – currency with two decimals
    "mm/dd/yyyy"        // Column C – US‑style date
};
```

The **add custom formatting to excel** step is as simple as building this string array.  
प्रत्येक एंट्री वही पैटर्न फॉलो करती है जो आप Excel के “Custom” फॉर्मेट डायलॉग में उपयोग करेंगे, इसलिए आपको नई सिंटैक्स सीखने की जरूरत नहीं है।

---

## Excel कॉलम की डेट फॉर्मेट सेट करें

यदि आपके डेटा में डेट्स शामिल हैं, तो आप चाहते हैं कि वे लगातार दिखें। `columnStyles` में तीसरी एंट्री (`"mm/dd/yyyy"`) ठीक वही करती है—यह Aspose.Cells को बताता है कि तीसरे कॉलम को डेट के रूप में ट्रीट करें और इसे month/day/year क्रम में दिखाएं।

```java
// Example: Adding a row with a Date object
Map<String, Object> row = new HashMap<>();
row.put("Product", "Widget");
row.put("Price", 19.99);
row.put("ReleaseDate", new java.util.Date()); // Today’s date
dataRows.add(row);
```

जब वर्कबुक जेनरेट होती है, तो **set date format excel column** निर्देश सुनिश्चित करता है कि हर `ReleaseDate` सेल `04/27/2026` (या आज की तारीख) जैसा दिखे। कोई पोस्ट‑प्रोसेसिंग आवश्यक नहीं।

---

## डेटा इम्पोर्ट करें और स्टाइल्स लागू करें

स्टाइल्स परिभाषित होने के बाद, अगला कदम हमारे डेटा को वर्कशीट में फीड करना है। Aspose.Cells `importCustomObjects` प्रदान करता है, जो न केवल डेटा डालता है बल्कि हमने अभी बनाई हुई स्टाइल एरे को भी लागू करता है।

```java
// Step 1: Simulate fetching rows from a DB
List<Map<String, Object>> dataRows = fetchRows(); // see helper below

// Step 2: Create a fresh workbook and grab the first sheet
Workbook workbook = new Workbook();
Worksheet worksheet = workbook.getWorksheets().get(0);

// Step 3: Import data with custom styles; true = first row has headers
worksheet.getCells().importCustomObjects(
        dataRows,        // data source
        0,               // start row (0‑based)
        0,               // start column (0‑based)
        columnStyles,    // style array from above
        true);           // first row contains column names
```

### क्यों `importCustomObjects`?

- **Single‑call magic:** यह डेटा *और* फॉर्मेटिंग को एक ही कॉल में लिखता है।  
- **Header awareness:** `true` पास करने से API को पहली मैप की कुंजियों को कॉलम हेडर के रूप में ट्रीट करने को कहता है, जो **create styled excel workbook** परिदृश्यों के लिए आदर्श है जहाँ आप पठनीय कॉलम शीर्षक चाहते हैं।  
- **Performance:** आंतरिक रूप से Aspose लिखने के ऑपरेशन्स को बैच करता है, जिससे यह सेल‑बाय‑सेल लूपिंग से तेज़ हो जाता है।  

> **Watch out:** प्रत्येक `Map` में कुंजियों का क्रम महत्वपूर्ण है। यदि आपको निर्धारित कॉलम क्रम चाहिए तो `LinkedHashMap` का उपयोग करें।

## वर्कबुक को XLSX फ़ाइल के रूप में सहेजें

सारा काम बेकार होगा अगर आप फ़ाइल को स्थायी नहीं कर सकते। अंतिम लाइन एक-लाइनर है जो **save workbook as xlsx** करता है।

```java
// Step 5: Persist the workbook
String outputPath = "output/DataTableStyled.xlsx";
workbook.save(outputPath);
System.out.println("Workbook saved to " + outputPath);
```

`save` मेथड स्वचालित रूप से फ़ाइल एक्सटेंशन का पता लगाता है। `.xlsx` निर्दिष्ट करके, आप आधुनिक Office Open XML फ़ॉर्मेट की गारंटी देते हैं, जो अधिकांश डाउनस्ट्रीम टूल्स की अपेक्षा है।

## परिणाम चलाएँ और सत्यापित करें

`ImportDataTableWithStyle` क्लास को कंपाइल और रन करें। निष्पादन के बाद आपको `output/DataTableStyled.xlsx` पर एक फ़ाइल दिखनी चाहिए। इसे Excel (या LibreOffice) में खोलें और आप देखेंगे:

- कॉलम **A** साधारण टेक्स्ट दिखाता है (`Product`).  
- कॉलम **B** नंबरों को डॉलर साइन और दो दशमलव स्थानों के साथ दर्शाता है (`$19.99`).  
- कॉलम **C** डेट्स को `mm/dd/yyyy` फॉर्मेट में प्रस्तुत करता है (`04/27/2026`).  

यहाँ एक त्वरित स्क्रीनशॉट है (पहुँचयोग्यता के लिए alt टेक्स्ट शामिल है):

यदि नंबर या डेट्स गलत दिख रहे हैं, तो दोबारा जांचें कि `columnStyles` एरे आपके मैप कुंजियों के क्रम से मेल खाता है। याद रखें, **add custom formatting to excel** क्रम‑सेंसिटिव है।

## पूर्ण, चलाने योग्य उदाहरण

नीचे पूर्ण क्लास है जिसे आप सीधे अपने IDE में कॉपी कर सकते हैं। इसमें एक mock `fetchRows` मेथड शामिल है जो सैंपल डेटा रिटर्न करता है, इसलिए परीक्षण के लिए आपको वास्तविक डेटाबेस की आवश्यकता नहीं है।

```java
import com.aspose.cells.*;
import java.util.*;

public class ImportDataTableWithStyle {
    public static void main(String[] args) throws Exception {
        // ---------- Step 1: Get data ----------
        List<Map<String, Object>> dataRows = fetchRows();

        // ---------- Step 2: Create workbook ----------
        Workbook workbook = new Workbook();
        Worksheet worksheet = workbook.getWorksheets().get(0);

        // ---------- Step 3: Define column styles ----------
        String[] columnStyles = { "General", "$#,##0.00", "mm/dd/yyyy" };

        // ---------- Step 4: Import data with styles ----------
        // true = first row contains column names
        worksheet.getCells().importCustomObjects(dataRows, 0, 0, columnStyles, true);

        // ---------- Step 5: Save as XLSX ----------
        String outPath = "output/DataTableStyled.xlsx";
        workbook.save(outPath);
        System.out.println("Workbook saved to " + outPath);
    }

    // Mock method – replace with real DB call if needed
    private static List<Map<String, Object>> fetchRows() {
        List<Map<String, Object>> rows = new ArrayList<>();

        Map<String, Object> row1 = new LinkedHashMap<>();
        row1.put("Product", "Gadget");
        row1.put("Price", 49.95);
        row1.put("ReleaseDate", new Date()); // today
        rows.add(row1);

        Map<String, Object> row2 = new LinkedHashMap<>();
        row2.put("Product", "Widget");
        row2.put("Price", 19.99);
        Calendar cal = Calendar.getInstance();
        cal.set(2025, Calendar.DECEMBER, 15);
        row2.put("ReleaseDate", cal.getTime());
        rows.add(row2);

        return rows;
    }
}
```

इसे रन करें, जेनरेट हुई फ़ाइल खोलें, और आप देखेंगे कि **create styled excel workbook** परिणाम जैसा आप चाहते थे।

## सामान्य प्रश्न और किनारे के मामले

**अगर मुझे तीन से अधिक कॉलम चाहिए?**  
सिर्फ `columnStyles` एरे को विस्तारित करें—प्रत्येक एंट्री क्रम में एक कॉलम से मैप होती है। उन कॉलम्स के लिए जिन्हें आप अनछुए रखना चाहते हैं, `"General"` उपयोग करें।

**क्या मैं फ़ॉन्ट्स या रंग लागू कर सकता हूँ?**  
हाँ। स्ट्रिंग फॉर्मेट के बजाय, आप एक `Style` ऑब्जेक्ट बना सकते हैं, उसे `StyleFlag` को असाइन कर सकते हैं, और `importCustomObjects` को `Style[]` एरे पास कर सकते हैं। यह एक गहरा विषय है, लेकिन सिद्धांत वही रहता है।
