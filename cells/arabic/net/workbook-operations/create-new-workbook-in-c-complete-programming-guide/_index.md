# إنشاء دفتر عمل جديد في C# – دليل برمجة شامل

هل احتجت يوماً إلى **إنشاء دفتر عمل جديد** في C# لكنك لم تكن متأكدًا من أين تبدأ؟ لست وحدك. سواءً كنت تُؤتمت خط أنابيب تقارير أو مجرد تجربة صيغ Excel في الكود، فإن القدرة على إنشاء دفتر عمل، وإدراج صيغ مثل `EXPAND` أو `COT`، ثم **حفظ دفتر العمل إلى ملف** هي مهارة أساسية لأي مطور .NET.

في هذا البرنامج التعليمي سنستعرض مثالًا واقعيًا يفعل ذلك بالضبط: سننشئ دفتر عمل جديد، نستخدم دالة `EXPAND` لتحويل مصفوفة ثابتة إلى عمود ديناميكي، نحسب الظل المماس للزاوية باستخدام دالة `COT`، وأخيرًا **نحفظ دفتر العمل إلى ملف** بصيغة `.xlsx`. في النهاية ستحصل على مقتطف جاهز للتنفيذ، وتفهم *لماذا* كل استدعاء مهم، وتطلع على بعض التنويعات المفيدة للحالات الخاصة.

> **نصيحة احترافية:** كل الشيفرة أدناه تعمل مع أحدث نسخة من Aspose.Cells لـ .NET (حتى مارس 2026). إذا كنت تستخدم إصدارًا أقدم، فإن واجهة البرمجة (API) متشابهة إلى حد كبير، لكن تأكد من مراجعة استيراد المساحات الاسمية (namespaces).

## ما ستحتاجه

- .NET 6.0 أو أحدث (العينة تستهدف .NET 6، لكن .NET 5 يعمل أيضًا)  
- Aspose.Cells لـ .NET مثبت عبر NuGet (`Install-Package Aspose.Cells`)  
- معرفة أساسية بـ C# (أنت قادر على ذلك)  

هذا كل شيء—لا ملفات DLL إضافية، لا تفاعل COM، وبالتأكيد لا حاجة لتثبيت Excel على الجهاز. جاهز؟ لنبدأ.

## الخطوة 1: إنشاء دفتر عمل جديد

أول شيء يجب القيام به هو إنشاء كائن من فئة `Workbook`. فكر فيه كفتح ملف Excel فارغ في الذاكرة. هذا الكائن يحتوي على مجموعة من الأوراق، الأنماط، وكل ما ستحتاجه لاحقًا.

```csharp
using Aspose.Cells;

class ExcelDemo
{
    static void Main()
    {
        // Step 1: Create a new workbook and grab the first worksheet
        Workbook workbook = new Workbook();               // <-- creates an empty .xlsx structure
        Worksheet ws = workbook.Worksheets[0];            // default sheet is named "Sheet1"
```

لماذا نأخذ الورقة الأولى مباشرة؟ معظم الأمثلة السريعة تعمل على ورقة واحدة، ومؤشر `Worksheets[0]` هو أسرع طريقة للحصول على مرجع دون الحاجة للتكرار. إذا احتجت أوراقًا متعددة لاحقًا، يمكنك إضافتها باستخدام `workbook.Worksheets.Add()`.

## الخطوة 2: كيفية استخدام EXPAND لإنشاء نطاقات ديناميكية

`EXPAND` هي دالة Excel حديثة تأخذ مصفوفة وتوسّعها إلى حجم محدد. في شيفرتنا سنوسّع المصفوفة الحرفية `{1,2,3}` إلى **عمود من 5 صفوف** يبدأ من الخلية `A1`. الصياغة داخل السلسلة هي بالضبط ما تكتبه في Excel، لذا يمكنك نسخها ولصقها مباشرةً في خلية لاحقًا إذا رغبت.

```csharp
        // Step 2: Apply EXPAND to turn {1,2,3} into a 5‑row vertical range
        ws.Cells["A1"].Formula = "=EXPAND({1,2,3},5,1)"; // rows=5, cols=1
```

### ما الذي يحدث خلف الكواليس؟

- `{1,2,3}` هي مصفوفة أفقية حرفية.  
- الوسيط الثاني (`5`) يخبر Excel بتوسيع المصفوفة إلى **5 صفوف**.  
- الوسيط الثالث (`1`) يفرض إخراج **عمود واحد**.  

إذا حذفت الوسيط الثالث، سيحاول Excel الحفاظ على الشكل الأصلي، مما قد ينتج عنه كتلة 5×3 بدلاً من عمود واحد. هذه فخ شائع عندما تبدأ بالتجربة مع `EXPAND`.

#### تنويعات قد تحتاجها

| الشكل المطلوب | مثال الصيغة |
|---------------|-------------|
| كتلة 3 صفوف × 2 عمود | `=EXPAND({1,2,3},3,2)` |
| تعبئة للأسفل فقط (نفس العمود) | `=EXPAND({10,20},10,1)` |
| توسيع إلى عدد أعمدة أكبر | `=EXPAND({5},5,4)` |

لا تتردد في استبدال القيم الحرفية أو الأبعاد لتتناسب مع منطق توليد البيانات لديك.

## الخطوة 3: كيفية حساب الظل المماس باستخدام دالة COT

دالة `COT` تُعيد الظل المماس لزاوية معبر عنها بالراديان. في مثالنا نحسب الظل المماس للزاوية 45° (π/4 راديان). النتيجة، `1`، تُوضع في الخلية `B1`.

```csharp
        // Step 3: Use COT to calculate cotangent of 45 degrees (π/4 radians)
        ws.Cells["B1"].Formula = "=COT(PI()/4)"; // PI() returns π, divided by 4 = 45°
```

### لماذا نستخدم COT بدلاً من الحساب اليدوي؟

Excel يعرف بالفعل كيفية التعامل مع التحويل المثلثي، لذا تتجنب أخطاء التقريب العائمة التي قد تظهر إذا حاولت حساب `1 / TAN(angle)`. بالإضافة إلى ذلك، تبقى الصيغة قابلة للقراءة لأي شخص يراجع الجدول لاحقًا.

#### حالة خاصة: زوايا خارج نطاق 0‑360°

إذا أدخلت زاوية أكبر من `2*PI()` (أو سالبة)، سيقوم Excel بلفها تلقائيًا، لكن النتيجة قد تكون مفاجئة. لتكون آمنًا، قد ترغب في تطبيع الزاوية أولًا:

```csharp
        // Normalize angle to 0‑2π range before applying COT
        ws.Cells["C1"].Formula = "=COT(MOD(PI()*3, 2*PI()))";
```

هذا المقتطف يوضح كيفية دمج `MOD` مع `COT` للحصول على حسابات قوية.

## الخطوة 4: كيفية حفظ دفتر العمل إلى ملف (Excel)

الآن بعد أن وضعت الصيغ، الخطوة الأخيرة هي **حفظ دفتر العمل إلى ملف**. يمكنك اختيار أي مسار تفضله—فقط تأكد من وجود المجلد وأن لديك صلاحيات الكتابة.

```csharp
        // Step 4 (optional): Save the workbook so you can inspect the results
        string outputPath = Path.Combine(
            Environment.GetFolderPath(Environment.SpecialFolder.Desktop),
            "output.xlsx");

        workbook.Save(outputPath);
        Console.WriteLine($"Workbook saved to {outputPath}");
    }
}
```

### ما الذي يتم حفظه فعليًا؟

عند فتح `output.xlsx` في Excel، سترى:

| A | B |
|---|---|
| 1 | 1 |
| 2 |   |
| 3 |   |
|   |   |
|   |   |

- العمود **A** يحتوي على المصفوفة الموسعة `{1,2,3}` متبوعًا بخلين فارغين (لأننا طلبنا 5 صفوف).  
- الخلية **B1** تُظهر `1`، وهو الظل المماس للزاوية 45°.  

إذا قمت بتحديث دفتر العمل (اضغط `F9` أو فعّل الحساب التلقائي)، سيقوم Excel بتقييم الصيغ وعرض النتائج. كما يوفر Aspose.Cells طريقة `CalculateFormula` إذا احتجت القيم دون فتح Excel:

```csharp
        workbook.CalculateFormula();
        double cotResult = ws.Cells["B1"].DoubleValue; // should be 1.0
```

## أسئلة شائعة ومشكلات محتملة

| السؤال | الجواب |
|----------|--------|
| **هل أحتاج إلى تمكين الحساب يدويًا؟** | لا. بشكل افتراضي يحفظ Aspose.Cells الصيغ كما هي؛ سيحسبها Excel عند الفتح. استخدم `workbook.CalculateFormula()` للحساب المسبق. |
| **هل يمكنني كتابة صيغ إلى عدة خلايا مرة واحدة؟** | بالتأكيد. استخدم `ws.Cells["D1:D5"].Formula = "=RAND()"` لملء نطاق بأرقام عشوائية. |
| **ماذا لو لم يكن المجلد المستهدف موجودًا؟** | أنشئه أولًا: `Directory.CreateDirectory(Path.GetDirectoryName(outputPath));` |
| **هل تدعم `EXPAND` إصدارات Excel القديمة؟** | ظهرت `EXPAND` مع Excel 365/2019. إذا كنت تحتاج توافقًا مع ملفات أقدم، فكر في استخدام تركيبات `INDEX`/`SEQUENCE` بدلاً منها. |
| **كيف أخفي عرض الصيغة؟** | عيّن `ws.Cells["A1"].FormulaHidden = true;` واحمِ الورقة إذا لا تريد للمستخدمين رؤية الصيغة الأساسية. |

## الخلاصة

أنت الآن تعرف **كيفية إنشاء دفتر عمل جديد** في C#، وتستفيد من قوة دالة `EXPAND` لتوليد مصفوفات ديناميكية، وتحسب الظل المماس باستخدام `COT`، وتقوم **بحفظ دفتر العمل إلى ملف** كوثيقة Excel منظمة. المثال الكامل القابل للتنفيذ موجود في المقتطفات أعلاه—انسخه إلى تطبيق Console، اضغط `F5`، وافتح `output.xlsx` لتشاهد السحر.

### ما الخطوة التالية؟

- **استكشف وظائف المصفوفات الديناميكية الأخرى** مثل `SEQUENCE`، `FILTER`، و `SORT`.  
- **أتمتة إنشاء المخططات** باستخدام واجهة المخططات الغنية في Aspose.Cells.  
- **دمج مع مصادر البيانات** (SQL، CSV) وإدخال تلك القيم في الصيغ برمجيًا.  
- **تعلم كيفية حفظ Excel كملف PDF** أو صيغ أخرى—مثالي لخطوط أنابيب التقارير.

لا تتردد في التجربة: غيّر قيم المصفوفة، عدّل الزاوية، أو اكتب النتيجة في ورقة مختلفة. السماء هي الحد عندما تجمع بين C# ومحرك الصيغ الحديث في Excel.

برمجة سعيدة، ولتُحسب جداولك دائمًا بشكل صحيح!
