# كيفية حساب الظل المقلوب (Cotangent) في Excel باستخدام C# – دليل كامل

هل تساءلت يومًا **كيفية حساب الظل المقلوب** مباشرة داخل ورقة Excel من تطبيق C#؟ ربما تقوم ببناء نموذج مالي، أو حاسبة علمية، أو مجرد أتمتة تقرير، وتحتاج إلى الظل المقلوب لزاوية دون نقل البيانات إلى أداة منفصلة. الخبر السار؟ ببضع أسطر من الشيفرة يمكنك **إنشاء دفتر عمل Excel**، وإدراج صيغة `COT` في خلية، ومشاهدة Excel يقوم بالحساب لك.

في هذا الدرس سنستعرض العملية بالكامل: من تهيئة دفتر العمل، إلى استخدام دالة `EXPAND` لإعادة تشكيل البيانات، إلى **تعيين صيغة الخلية** للظل المقلوب، وأخيرًا **كيفية حفظ Excel** حتى تتمكن من فتحه في الواجهة. في النهاية ستحصل على مقطع C# جاهز للتنفيذ يمكنك نسخه‑اللصق في أي مشروع .NET.

> **ملخص سريع:**  
> • الهدف الأساسي – **كيفية حساب الظل المقلوب** في Excel باستخدام C#.  
> • الأهداف الثانوية – **إنشاء دفتر عمل Excel**، **كيفية استخدام expand**، **تعيين صيغة الخلية**، **كيفية حفظ Excel**.  
> • المتطلبات المسبقة – إشارة إلى مكتبة جداول البيانات (سنستخدم Aspose.Cells، لكن المفاهيم تنطبق على EPPlus، ClosedXML، إلخ).

## الخطوة 2 – استخدام EXPAND لبناء مصفوفة (كيفية استخدام Expand)

دالة `EXPAND` في Excel مفيدة عندما تريد تحويل مصفوفة أحادية البُعد إلى نطاق متعدد الصفوف/الأعمدة. في مثالنا سنولد **مصفوفة 3 × 2** من قائمة بسيطة `{1,2,3}`. هذا يوضح **كيفية استخدام expand** ويظهر أيضًا أن الصيغ يمكنها إرجاع مصفوفات، وليس قيمًا مفردة فقط.

```csharp
        // Place the EXPAND formula in cell A1
        // =EXPAND({1,2,3},3,2) creates a 3‑row, 2‑column matrix
        worksheet.Cells["A1"].Formula = "=EXPAND({1,2,3},3,2)";
```

عند فتح الملف المحفوظ، ستحتوي الخلايا A1:B3 على:

| A | B |
|---|---|
| 1 | 2 |
| 2 | 3 |
| 3 | 0 |

(العمود الثاني يملأ بالأصفار لأن المصفوفة المصدر تحتوي على ثلاثة عناصر فقط.)

> **نصيحة احترافية:** إذا كنت تحتاج إلى شكل مختلف، فقط غير الوسيطين الثاني والثالث من `EXPAND`. الدالة تضيف الخلايا المفقودة تلقائيًا بأصفار.

## الخطوة 3 – تعيين صيغة COT (كيفية حساب الظل المقلوب)

الآن نصل إلى نجمة العرض: **كيفية حساب الظل المقلوب**. يوفر Excel دالة `COT`، التي تتوقع زاوية بالراديان. سنستخدم `PI()/4` (45°) كمثال بسيط؛ النتيجة يجب أن تكون بالضبط `1`.

```csharp
        // Put the cotangent formula in cell B1
        // =COT(PI()/4) evaluates to 1 because cot(45°) = 1
        worksheet.Cells["B1"].Formula = "=COT(PI()/4)";
```

يمكنك استبدال `PI()/4` بأي إشارة إلى خلية أخرى تحتوي على قيمة راديان، أو حتى تحويل من درجات إلى راديان مثل `RADIANS(A2)`.

> **لماذا نستخدم صيغة بدلاً من حساب C#؟**  
> إبقاء الحساب داخل Excel يعني أن النتيجة تتحدث تلقائيًا إذا تغيرت الزاوية المصدر. كما أنه يخفف العبء على تطبيقك عبر الاستفادة من محرك حساب Excel المُحسّن.

## الخطوة 4 – حفظ دفتر العمل (كيفية حفظ Excel)

القطعة الأخيرة من اللغز هي حفظ الملف حتى تتمكن من فتحه في Excel أو مشاركته لاحقًا. هنا يصبح **كيفية حفظ Excel** أمرًا ملموسًا.

```csharp
        // Define the output path – adjust as needed

        // Optional: let the user know we’re done
```

> **حالة حافة:** إذا لم يكن الدليل موجودًا، سيُطلق `Save` استثناء. احرص على تغليف الاستدعاء داخل `try/catch` أو تأكد من إنشاء المجلد مسبقًا.

هذا هو البرنامج الكامل القابل للتنفيذ. قم بتجميعه وتشغيله، ثم افتح `CotangentDemo.xlsx`. سترى المصفوفة الموسعة في `A1:B3` وقيمة الظل المقلوب `1` في `B1`.

## مثال كامل يعمل – جميع الخطوات مجمعة

فيما يلي الشيفرة الكاملة مع كل جزء موصول معًا. انسخ‑الصقها في مشروع وحدة تحكم جديد واضغط **F5**.

```csharp
using System;
using Aspose.Cells;

namespace ExcelCotangentDemo
{
    class Program
    {
        static void Main()
        {
            // Step 1 – create a new workbook and get its first worksheet
            Workbook workbook = new Workbook();
            Worksheet worksheet = workbook.Worksheets[0];

            // Step 2 – use EXPAND to generate a 3×2 matrix from a 1‑D array
            worksheet.Cells["A1"].Formula = "=EXPAND({1,2,3},3,2)";

            // Step 3 – set a COT formula that calculates cotangent of 45°
            worksheet.Cells["B1"].Formula = "=COT(PI()/4)";

            // Step 4 – save the workbook to view the results
            string outputPath = @"C:\Temp\CotangentDemo.xlsx";
            workbook.Save(outputPath);

            Console.WriteLine($"Workbook successfully saved at: {outputPath}");
        }
    }
}
```

### النتيجة المتوقعة عند فتح الملف

| A | B |
|---|---|
| 1 | 1 |
| 2 | 0 |
| 3 | 0 |

- **A1‑B3**: المصفوفة التي أنشأتها `EXPAND`.  
- **B1**: نتيجة `COT(PI()/4)` – بالضبط **1**.

## الأسئلة المتكررة (FAQs)

### 1. هل يمكنني حساب الظل المقلوب للزوايا المخزنة في خلايا أخرى؟
بالطبع. استبدل الثابت `PI()/4` بإشارة، مثل `=COT(RADIANS(C2))` حيث تحتوي الخلية C2 على الزاوية بالدرجات.

### 2. ماذا لو أردت النتيجة بالدرجات بدلًا من الراديان؟
استخدم `DEGREES(ATAN(1/yourValue))` لتحويل الظل العكسي مرة أخرى إلى درجات، أو ببساطة ضع تحويل الزاوية داخل `RADIANS` كما هو موضح أعلاه.

### 3. هل تقوم Aspose.Cells بتقييم الصيغ تلقائيًا؟
نعم. عندما **تحفظ** دفتر العمل، تقوم المكتبة بحساب جميع الصيغ بشكل افتراضي. إذا احتجت القيم في الشيفرة قبل الحفظ، استدعِ `workbook.CalculateFormula()`.

### 4. كيف يختلف هذا عن استخدام EPPlus أو ClosedXML؟
سطح API مشابه — أنشئ `Workbook`، وصل إلى `Worksheets`، عيّن `Formula`. الاختلاف الرئيسي هو الترخيص وبعض الميزات المتقدمة. المفاهيم الأساسية (إنشاء، تعيين صيغ، حفظ) تبقى نفسها.

### 5. ماذا لو أردت إرجاع النتيجة إلى C#؟
بعد استدعاء `workbook.CalculateFormula()`، يمكنك قراءة خاصية `Value` للخلية:

```csharp
double cotValue = worksheet.Cells["B1"].DoubleValue; // should be 1.0
```

## نصائح ومخاطر قد تواجهها

- **الأصفار المتبقية في EXPAND:** إذا كانت المصفوفة المصدر أقصر من الحجم المطلوب، يملأ Excel الخلايا الناقصة بأصفار. هذا سلوك متوقع، لكن كن واعيًا إذا كنت تعتمد على قيم غير صفرية.  
- **فاصل الصيغة حسب اللغة:** بعض إصدارات Excel تستخدم الفاصلة المنقوطة (`;`) كفاصل للمعاملات. المكتبة دائمًا تتوقع الفواصل العادية (`,`) لذا لا تحتاج للقلق بشأن الإعدادات الإقليمية.  
- **أذونات الملفات:** عند التشغيل تحت IIS أو حساب خدمة، تأكد من أن العملية تملك صلاحية كتابة على المجلد المستهدف.  
- **توافق الإصدارات:** تم تقديم دالة `EXPAND` في Excel 365/2021. إذا كنت تحتاج إلى توافق مع إصدارات أقدم، سيتعين عليك محاكاة السلوك باستخدام أعمدة مساعدة.
