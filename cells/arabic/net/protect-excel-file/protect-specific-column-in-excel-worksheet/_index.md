# حماية عمود محدد في ورقة عمل Excel

## مقدمة

في عالم حيث أصبحت إدارة البيانات معقدة بشكل متزايد، فإن معرفة كيفية حماية أقسام معينة من مستنداتك يمكن أن تحمي المعلومات المهمة من التغييرات غير المرغوب فيها. سواء كنت طالبًا يدير درجاتك، أو مدير مشروع يتتبع الميزانيات، أو محللًا يتعامل مع بيانات حساسة، فمن الأهمية بمكان الحفاظ على أمان المعلومات المهمة مع السماح للآخرين باستخدام جدول البيانات. سيوضح هذا الدليل كيفية حماية أعمدة معينة في ورقة عمل Excel باستخدام Aspose.Cells لـ .NET.

## المتطلبات الأساسية 

قبل الغوص في الكود، هناك بعض المتطلبات الأساسية التي يجب عليك الاهتمام بها:

1. Visual Studio: تأكد من تثبيت Microsoft Visual Studio (يفضل 2017 أو إصدار أحدث). سيعمل هذا كبيئة تطوير لك. 
2.  مكتبة Aspose.Cells: يجب أن يكون لديك مكتبة Aspose.Cells التي تم تنزيلها والإشارة إليها في مشروعك. يمكنك[تحميل المكتبة هنا](https://releases.aspose.com/cells/net/) إذا لم تكن قد فعلت ذلك بالفعل.
3. الفهم الأساسي للغة C#: في حين أن أمثلة التعليمات البرمجية واضحة، فإن الحصول على معرفة أساسية بلغة C# سيساعدك على إجراء التعديلات حسب الضرورة.
4. .NET Framework: تأكد من أن مشروعك يستهدف .NET Framework حيث يتم دعم Aspose.Cells.

الآن، دعونا ننتقل إلى الجزء الممتع - البرمجة!

## استيراد الحزم

للبدء، تحتاج إلى استيراد مساحات الأسماء الضرورية المتعلقة بـ Aspose.Cells. في الجزء العلوي من ملف C#، قم بتضمين السطر التالي:

```csharp
using System.IO;
using Aspose.Cells;
```

تتمتع هذه المكتبة بالقوة وتسمح لك بإجراء عدد لا يحصى من العمليات، بما في ذلك حماية بياناتك داخل ملفات Excel، وهو ما نهدف إلى تحقيقه اليوم.

دعنا نقسم هذا إلى عدة خطوات واضحة وموجزة. ستقوم بحماية أعمدة محددة، مما يسمح ببقاء بقية ورقة العمل قابلة للتحرير.

## الخطوة 1: إعداد دليل البيانات

أولاً، عليك تحديد المسار للدليل الذي سيتم حفظ ملف Excel فيه. ويتضمن هذا إنشاء دليل إذا لم يكن موجودًا بالفعل. وإليك كيفية القيام بذلك:

```csharp
// قم بتحديد المسار إلى دليل المستندات.
string dataDir = "YOUR DOCUMENT DIRECTORY";
// إنشاء الدليل إذا لم يكن موجودًا بالفعل.
bool IsExists = System.IO.Directory.Exists(dataDir);
if (!IsExists)
    System.IO.Directory.CreateDirectory(dataDir);
```

يقوم مقتطف التعليمات البرمجية بإنشاء دليل في المسار المحدد إذا لم يكن موجودًا بالفعل، مما يضمن وجود موقع آمن لملف الإخراج الخاص بك.

## الخطوة 2: إنشاء مصنف جديد

بعد ذلك، نحتاج إلى إنشاء مصنف جديد. يتيح لك Aspose.Cells إنشاء ملفات Excel ومعالجتها بسهولة. وإليك كيفية القيام بذلك:

```csharp
// إنشاء مصنف جديد.
Workbook wb = new Workbook();
```

 من خلال إنشاء مثيل جديد`Workbook`الهدف هو أن تبدأ بلوحة فارغة، وجاهز لتخصيص جدول البيانات الخاص بك.

## الخطوة 3: الوصول إلى ورقة العمل الأولى

بعد إنشاء المصنف، ستحتاج إلى الوصول إلى ورقة العمل الأولى التي ستقوم فيها بإجراء العمليات الخاصة بك:

```csharp
// إنشاء كائن ورقة عمل والحصول على الورقة الأولى.
Worksheet sheet = wb.Worksheets[0];
```

 ال`Worksheet` يسمح لك الكائن بالتعامل مع ورقة العمل المحددة في المصنف. في هذه الحالة، نستخدم الورقة الأولى.

## الخطوة 4: إلغاء قفل جميع الأعمدة

لتعيين أعمدة معينة كمحمية، يجب عليك إلغاء قفل جميع الأعمدة في ورقة العمل أولاً. هذه الخطوة تحضرها للتعديلات:

```csharp
// تعريف كائن النمط.
Style style;
// تعريف كائن علم النمط.
StyleFlag flag;
// قم بالمرور على جميع الأعمدة في ورقة العمل وإلغاء قفلها.
for (int i = 0; i <= 255; i++)
{
    style = sheet.Cells.Columns[(byte)i].Style;
    style.IsLocked = false;
    flag = new StyleFlag();
    flag.Locked = true;
    sheet.Cells.Columns[(byte)i].ApplyStyle(style, flag);
}
```

 يتكرر هذا الكود خلال كل من الأعمدة الـ 256 الأولى. ويقوم بإلغاء قفل كل عمود عن طريق تعديل إعدادات النمط.`StyleFlag` يضمن إمكانية تطبيق الخاصية المقفلة لاحقًا.

## الخطوة 5: قفل العمود المطلوب

الآن، ستحتاج إلى قفل العمود الأول على وجه التحديد، مع ترك جميع الأعمدة الأخرى قابلة للتعديل. إليك كيفية القيام بذلك:

```csharp
// احصل على نمط العمود الأول.
style = sheet.Cells.Columns[0].Style;
// قفله.
style.IsLocked = true;
//إنشاء العلم.
flag = new StyleFlag();
// ضبط إعداد القفل.
flag.Locked = true;
// قم بتطبيق النمط على العمود الأول.
sheet.Cells.Columns[0].ApplyStyle(style, flag);
```

هنا، يقوم الكود بجلب نمط العمود الأول، وتعيينه على القفل، ثم تطبيق هذا النمط. والنتيجة هي أن المستخدمين يمكنهم تحرير بقية الصفحة ولكن لن يتمكنوا من تعديل العمود الأول.

## الخطوة 6: حماية ورقة العمل

تتضمن الخطوة التالية تمكين الحماية لورقة العمل بأكملها. وهنا ستدخل أقفال الأعمدة حيز التنفيذ:

```csharp
// حماية الورقة.
sheet.Protect(ProtectionType.All);
```

 ال`Protect` تضمن الطريقة تأمين جميع العناصر القابلة للتنفيذ في الورقة، باستثناء المناطق التي سمحت بها على وجه التحديد (مثل الأعمدة غير المؤمنة).

## الخطوة 7: احفظ المصنف

بمجرد تكوين كل شيء وتجهيزه، حان الوقت لحفظ المصنف الخاص بك، مع التأكد من تسجيل جميع التغييرات:

```csharp
// احفظ ملف Excel.
wb.Save(dataDir + "output.out.xls", SaveFormat.Excel97To2003);
```

 يحفظ هذا الكود المصنف الخاص بك بتنسيق Excel 97-2003 في المسار المحدد. تأكد من استبدال`dataDir` مع مسار الدليل الفعلي الخاص بك.

## خاتمة

باتباع الخطوات الموضحة أعلاه، تكون قد نجحت في حماية أعمدة معينة في ورقة عمل Excel مع الاحتفاظ بأجزاء أخرى قابلة للتحرير. إن استخدام Aspose.Cells لـ .NET يفتح عالمًا من الاحتمالات عندما يتعلق الأمر بالتلاعب بملفات Excel. تعد هذه القدرة على حماية المعلومات الحساسة أمرًا حيويًا بشكل خاص في بيئات العمل المشتركة.
