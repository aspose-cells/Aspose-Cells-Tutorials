# تخصيص صيغ الخلايا باستخدام Aspose.Cells .NET
في عالمنا اليوم الذي يعتمد على البيانات، يُعدّ تخصيص صيغ جداول البيانات وتوطينها أمرًا بالغ الأهمية للشركات العاملة في مناطق مختلفة. يستكشف هذا البرنامج التعليمي كيفية استخدام Aspose.Cells .NET لتخصيص إعدادات عولمة صيغ الخلايا، وهي ميزة فعّالة للمطورين الذين يعملون على تطبيقات متعددة اللغات.

**ما سوف تتعلمه:**
- كيفية إنشاء إعدادات العولمة المخصصة في Aspose.Cells
- تطبيق هذه الإعدادات لتعديل أسماء الوظائف القياسية داخل الصيغ
- دمج هذه الوظيفة في مشاريع .NET الخاصة بك
قبل أن نبدأ في التنفيذ، تأكد من أنك مجهز بالأدوات والمعرفة اللازمة.

### دليل التنفيذ
#### إعدادات العولمة المخصصة لصيغ الخلايا
في هذا القسم، سننشئ إعدادات عولمة مخصصة بتجاوز أسماء دوال محددة في الصيغ. يتيح لنا هذا استخدام إصدارات محلية من دوال مثل SUM وAVERAGE ضمن جداول بيانات Excel.

**الخطوة 1: تحديد فئة العولمة المخصصة**
نبدأ بإنشاء فئة ترث من `GlobalizationSettings`. إليك كيفية تجاوز أسماء الوظائف:

```csharp
using Aspose.Cells;

class GS : GlobalizationSettings
{
    public override string GetLocalFunctionName(string standardName)
    {
        if (standardName == "SUM")
        {
            return "UserFormulaLocal_SUM";
        }
        
        if (standardName == "AVERAGE")
        {
            return "UserFormulaLocal_AVERAGE";
        }

        return standardName; // تأكد من إرجاع الاسم الأصلي للوظائف غير القابلة للاستبدال
    }
}
```

**الخطوة 2: تطبيق الإعدادات المخصصة على مصنف**
بعد ذلك، سنقوم بتطبيق هذه الإعدادات داخل مثيل المصنف.

```csharp
using Aspose.Cells;

public class RunWorkbookWithCustomGlobalizationSettings
{
    public static void Execute()
    {
        Workbook wb = new Workbook();
        
        // تعيين إعدادات العولمة المخصصة
        wb.Settings.GlobalizationSettings = new GS();

        Worksheet ws = wb.Worksheets[0];
        Cell cell = ws.Cells["C4"];

        // استخدام دالة SUM المخصصة
        cell.Formula = "SUM(A1:A2)";
        string formulaLocalSum = cell.FormulaLocal;
        
        Console.WriteLine("Formula Local (SUM): " + formulaLocalSum);

        // استخدام دالة AVERAGE المخصصة
        cell.Formula = "=AVERAGE(B1:B2, B5)";
        string formulaLocalAverage = cell.FormulaLocal;
        
        Console.WriteLine("Formula Local (AVERAGE): " + formulaLocalAverage);
    }
}
```
**توضيح:**
- نحن نتجاوز `GetLocalFunctionName` لربط أسماء الوظائف القياسية بإصداراتنا المترجمة.
- يتم تحديث إعدادات المصنف باستخدام فئتنا المخصصة، مما يؤثر على كافة الصيغ الموجودة في المصنف.

#### التطبيقات العملية
1. **دعم متعدد اللغات:** قم بتحديد أسماء الوظائف للمستخدمين في مناطق مختلفة دون تغيير منطق الصيغة الأساسية.
2. **أدوات إعداد التقارير المخصصة:** قم بإعداد التقارير وفقًا للمصطلحات والمعايير الخاصة بالصناعة.
3. **التكامل مع أنظمة تخطيط موارد المؤسسات:** قم بمحاذاة وظائف Excel مع اتفاقيات التسمية الداخلية المستخدمة في أنظمة تخطيط موارد المؤسسة.
